# Squid-N 動作達成ロードマップ

作成日: 2026-07-04 ／ 最終更新: 2026-07-04（全 Phase 実装完了）
目的: **「GUI 単独で空モデルから解析・設計・保存まで完結する」**状態を最短で達成し、
あわせてソルバーの安定性(異常入力での無応答・サイレント失敗・panic)を解消する。

> 本書は網羅調査(UI / ソルバー / 仕様対実装)の結果に基づく。
> 各 Phase は本ブランチ上で実装済み。実装内容の詳細と残課題は `docs/申し送り.md` を参照。

## 0. 現状診断(実装前の要約)

### 0.1 UI 導線の致命的断絶(GUI 単独で解析に到達できなかった)

| # | 問題 | 根拠 | 状態 |
|---|------|------|------|
| A1 | **材料を追加する UI が存在しない**。`squid-n-edit` に `AddMaterial` コマンド自体がない。部材への材料割当 UI もない → 設計検定が恒久的に空 | `app.rs` materials_panel は一覧のみ | ✅ 解消 |
| A2 | **荷重ケースを新規作成する UI がない**。`AddLoadCase` コマンドもない → 新規モデルでは荷重入力フォームすら出ず、線形静的解析が実行不能 | `tables/loads.rs` | ✅ 解消 |
| A3 | **ファイル保存/読込が未接続**。`squid-n-io` に SCZ 形式実装済みだがアプリから呼ばれない。終了=全データ消失 | `io/scz.rs` vs `main.rs` | ✅ 解消 |
| A4 | **階(Story)定義 UI がない**。地震静的・プッシュオーバー・偏心率は stories+seismic_weight+diaphragm 必須 → 地震解析は**エラーなしでゼロ変位を返す**(サイレント失敗) | 旧 `analysis.rs` | ✅ 解消 |
| A5 | 時刻歴グラフは `dummy_time_history()` のダミー正弦波固定。ソルバー結果と非接続 | 旧 `time_history_view.rs` | ✅ 解消 |
| A6 | CMQ 図は `app.beam_loads` を参照するが書き込むコードが皆無 → 常時「データなし」 | `viewer/mod.rs:760` | ⏸ 未対応(申し送り) |
| A7 | 部材・断面・節点荷重の削除ボタンがない(節点のみ削除可) | `tables/*.rs` | ✅ 解消 |
| A8 | 解析パラメータが全て固定(固有値モード数=3、地震方向=X、AiMode固定、Z/地盤種別/C0入力なし) | 旧 `app.rs` | ✅ 解消 |

- レポートタブ → ✅ CSV プレビュー/エクスポート/コピー

### 0.3 ソルバー安定性の問題

| # | 問題 | 状態 |
