# 床（スラブ）の設計と実装計画

現状、床の分配エンジン（`squid-n-load::floor`）・入力UI（`app/tables/slabs.rs`）・
解析接続（`refresh_beam_loads`／`sync_slab_loads_action`）は実装済みだが、
以下の本質的ギャップがある。本ドキュメントはその設計と段階実装をまとめる。

## 現状のギャップ

1. **DL/LL が区別されない**: `AreaLoad{kind:String, value}` は自由文字列で、床荷重は
   すべて kind=Dead の 1 ケース「床荷重(自動)」へ合算される。積載荷重（Live/LiveSeismic）
   として分離されず、①令85条1項の 3 欄（床用/骨組用/地震用）の使い分け、②荷重組合せでの
   DL/LL 係数分け、③令85条2項の柱積載低減（API はあるが未配線）ができない。
2. **用途別積載荷重テーブルが無い**: 令別表第1（住宅・事務室・教室…）のプリセットが無い。
3. **小梁の入力手段が無い**: エンジン（二段階伝達）とモデル（`JoistLine`）はあるが、UI は
   常に `joists: Vec::new()`。小梁は仮想ラインで実部材でないため断面検定・たわみ検定不可。
4. **入力が境界4節点固定**: モデル/エンジンは多角形対応なのに UI から作れない。スラブ自重の
   自動算定も無い。

## 用語・単位

- 内部単位系は N-mm-s（面荷重は N/mm²）。令別表第1 は N/m² で規定 → ×1e-6 で N/mm² に変換。
- **床用（floor）**: 床スラブ・小梁の設計用。**骨組用（frame）**: 大梁・柱・基礎の設計用。
  **地震用（seismic）**: 地震力（地震用重量）算定用。（令85条1項・令別表第1）

| 用途 | 床用 | 骨組用 | 地震用 |

| 自動車車庫・通路（Garage） | 5400 | 3900 | 2000 |

## 設計

### モデル（squid-n-core）

- `LoadPurpose { Floor, Frame, Seismic }` を追加。
- `SlabUsage`（上表の列挙 ＋ `Custom{floor,frame,seismic}`）を追加。
  `SlabUsage::live_load(purpose) -> f64`（N/mm²）を提供。
- `Slab` に `#[serde(default)] usage: Option<SlabUsage>` を追加（後方互換）。
- ヘルパ:
  - `Slab::dead_intensity()`: 既存 `loads` の合算（仕上げ等の固定荷重。DL）。
  - `Slab::live_intensity(purpose)`: `usage` から積載を返す（None は 0）。
  - `Slab::intensity(purpose)`: 長期・地震用の合成（DL + LL(purpose)）。
- 既存 `loads: Vec<AreaLoad>` は **DL（固定荷重）** として扱い後方互換とする。

### 荷重ケース同期（app）

現状 1 本（「床荷重(自動)」kind=Dead、DL のみ）を、用途に応じ分離する:
- 「床荷重(自動)」 kind=Dead: DL を分配。
- 「床積載(自動)」 kind=Live: LL(骨組用) を分配（長期骨組解析用）。
- 地震用重量集計は DL + LL(地震用) を用いる（`story_gen` へ接続）。

### 令85条2項（柱積載低減）

`live_load.rs` の `column_live_load_reduction`／`floors_supported_by_column` を、
`LoadCfg.live_load_reduction` が有効な場合に長期軸力算定へ配線する（Phase A-4）。
低減率の算定（`column_live_load_factors`）は実装済み・参考表示済み。設計軸力への
適用は per-case 軸力分解が必要なため Phase A-4 で行う。

### 入力UI（app）

- スラブ追加/一覧に **用途（SlabUsage）** の選択を追加。積載値は用途から自動表示（Custom は手入力）。
- 境界を可変長（3〜N 節点）に拡張（エンジンは対応済み）。
- （将来）3D ビューでの節点クリック指定、小梁入力、スラブ自重自動算定。

## 段階実装（推奨順）

- **A-1（完了）**: モデル層。`LoadPurpose`／`SlabUsage`／令別表第1 テーブル、
  `Slab.usage` と各ヘルパ、単体テスト。
- **A-2（完了）**: 荷重ケース同期の DL/LL 分離。用途選択 UI。
  - `distribute_slab_w(model, slab, w)` を追加し、DL（`dead_intensity`）と
    LL（`live_intensity(Frame)`）を別々に分配可能に。
  - `sync_slab_loads_action` が「床荷重(自動)」(Dead) と「床積載(自動)」(Live) の
    2 ケースへ冪等同期（`SyncSlabLoadsToCase` に `kind` を追加）。
  - スラブ一覧・追加フォームに用途（`SlabUsage`）選択を追加（`SetSlabUsage`／
    `AddSlab.usage`）。積載値プレビュー付き。
- **A-3（完了）**: 地震用重量への LL(地震用) 接続。
  - `sync_slab_loads_action` が「床地震用積載(自動)」(LiveSeismic) も同期
    （`live_intensity(Seismic)`）。`gravity_cases_for_seismic_weight` が
    LiveSeismic を優先採用するため、地震用重量には地震用値（骨組用より小さい）が
    算入される（令85条1項・令88条）。A-2 で Live(骨組用) にフォールバックして
    地震用重量が過大になる問題も同時に解消。
- **A-4**: 令85条2項（柱積載低減）の設計軸力への適用。
  `column_live_load_factors`（低減率の算定）は実装済みだが、現状は設計タブの
  参考表示のみ。実際の長期軸力へ適用するには、柱の長期軸力を荷重ケース別
  （DL 分と LL 分）に分解し、LL 分にのみ柱ごとの低減率を乗じる必要がある
  （設計検定パイプラインの `MemberDemand` 生成に per-case 軸力分解を追加）。
  影響範囲が広く独立性が高いため別フェーズとする。
- **B-1（完了）**: 小梁の入力UI（`JoistLine` 追加/削除）。
  - `SetSlabJoists`（`joists` 全置換・対称逆操作）を追加。
  - スラブタブに小梁入力セクションを追加。対象スラブを選び、支持2節点＋負担幅
    `spacing` で小梁を追加/削除。架かる方向 `dir` は支持2節点の平面ベクトルから
    自動算定。一覧表に小梁本数の列を追加。
  - 分配は既存の二段階伝達（`distribute_rect_with_joists`）を利用（矩形スラブ・
    三角/台形または一方向のとき有効）。小梁は仮想ライン（実部材ではない）で、
    実部材化は Phase B-2。
- **C（完了）**: 可変長境界・節点クリック入力。
  - スラブ追加フォームの境界を固定4節点から可変長（3〜N 節点）へ拡張。頂点スロットを
    +/− ボタンで増減。`SlabDraft.nodes` を `Vec<Option<NodeId>>` に変更。
  - 3D ビューアに「スラブ作成モード」を追加。境界節点を外周順にクリックで選び、
    3節点以上で「確定」→ `AddSlab`（梁・壁作成モードと排他）。
- **D（完了）**: ST-Bridge `StbSlab`/`StbSecSlab_RC` 往復。
  - モデルに `Slab.thickness: Option<f64>`（mm、serde default。`None` は建物一律の
    `Model::slab_thickness`）を追加。
  - import: `StbSlab`（`StbNodeIdOrder` の境界節点ループ。子要素 `StbNodeId` 形式にも
    対応）と `StbSecSlab_RC`（厚さ）をパースして `Slab` を生成。断面参照から厚さを解決。
    未対応リストから `StbSlab`/`StbSecSlab_RC` を除外。境界節点が解決できない・3頂点
    未満のスラブはスキップして報告。
  - export: 各スラブに `StbSecSlab_RC`（厚さ）＋`StbSlab`（境界ループ）を書き出し。
    断面 id は既存断面（柱・梁。標準モードの分割含む）と衝突しない base から採番。
  - 荷重・用途・分配法は ST-Bridge に対応物が無いため往復対象外（境界と厚さのみ往復）。
- **B-2（完了）**: 小梁の実部材化コマンド（梁要素生成）。
  - `MaterializeSlabJoists`（対象スラブの各小梁に、支持2節点を両端に持つ実 `Beam` を
    生成。断面未割当・両端ピン・末尾追加。冪等）と対称逆操作
    `PopTailMembers`/`PushTailMembers` を追加。
  - 分配エンジンに `LoadTarget::Span([n0,n1])` を追加。小梁の支持2節点間に実 `Beam` が
    存在する場合、点反力ではなく等分布荷重（`w·spacing`）を小梁自身へ載せ、小梁が FEM で
    支持へ伝達する（`beam_between` で自動切替。二重計上しない）。app 側は Span を Edge と
    同様に実部材の分布荷重へ変換。
  - これにより小梁が応力解析に参加し、断面検定・たわみ検定の対象になる。スラブタブに
    「小梁を実部材化」ボタン（未実部材化本数を表示）を追加。
  - 注: 支持2節点は既存節点（B-1 の入力仕様）。spacing 間隔での中間節点自動生成は将来課題。
