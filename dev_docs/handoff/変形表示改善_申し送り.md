# 変形表示改善（応力図重ね・床/二次部材の追従）申し送り

作成日: 2026-07-22
対象ブランチ: `claude/displacement-display-fix-pay8wd`
対象コード: `crates/squid-n-app/src/viewer/mod.rs`、`app/mod.rs`

## 1. 何を変えたか

GUI ビューアの変形表示について、次の 4 点を改善した。

### 1.1 N/Q/M 図への変形図の重ね表示

従来、変形図（`ViewMode::Deformed`）と応力図（N/Q/M）は独立した表示モードで、
応力と変形を同時に確認できなかった。N/Q/M 図のオプション行に「変形表示」
トグル（`App::overlay_deform`）を追加し、ON にすると節点座標に変位を加味して
描画する。応力図は変形後の節点座標 `coords3` を基準に材軸へ張り出すため、
図も変形後の材軸に自動で追従する。独立した「変形」モードは従来どおり残る。

### 1.2 床（スラブ・小梁）・二次部材の表示切替

ビューア上部のコントロール行に「床・二次部材」トグル
（`App::show_floor_secondary`、既定 ON）を追加した。OFF でスラブ面・小梁
（`slab.joists`）・二次部材（`model.secondary_members`）の描画をまとめて
非表示にする。変形図で解析対象外の要素を隠したい場合に使う。
CMQ 図は従来どおり設定によらず常に非表示。

### 1.3 床・二次部材の変形追従（未参照節点の変位補間）

スラブ境界・小梁支持点・二次部材の節点は主架構要素（`model.elements`）に
接続せず、拘束（剛床・剛リンク・MPC）のマスターでもないため、`DofMap` で
全自由度が不活性になり（`squid-n-core/src/dof.rs`、
`test_unreferenced_node_is_inactive` 参照）、解析結果の変位が常に `[0.0; 6]` の
まま残る。このため変形図で床・二次部材だけが原位置に残っていた。

対策として `interpolate_unreferenced_disp`（`viewer/mod.rs`）を追加した。
解析自由度を持たない節点（判定は `DofMap::build` の structural 判定と同じ規則:
主架構要素が接続する節点と拘束マスター節点を除外）について、最寄りの主架構
2 節点要素（線材）の線分上へ節点座標を射影（射影パラメータ t は [0,1] に
クランプ）し、両端節点の変位を t で線形補間して埋める。剛床代表節点
（階自動生成が重心に置く仮想節点）は要素に接続しないが正しい解析変位を
持つため、拘束マスターの除外がないと補間で上書きされる点に注意
（回帰テスト「剛床マスター節点の変位は補間で上書きされない」で担保）。ST-Bridge 取り込みモデルでは二次部材の支持点が
大梁のスパン中間へ節点共有なしで載るのが典型（`squid-n-load/src/secondary.rs`
の変換と同じ前提）のため、射影補間で表示上の追従ができる。

- あくまで**描画専用の近似**であり、解析結果（`StaticOnce::disp`）は変更しない。
- 補間は毎フレーム、変位表示時のみ実行（未参照節点数 × 線材数 の総当たり探索。
  通常規模のモデルでは問題ないが、超大規模モデルで描画が重い場合は
  空間分割などの高速化余地がある）。

### 1.4 変形倍率の自動計算（倍率スライダーの廃止）

従来の「倍率」スライダー（`App::deform_scale`、既定 100・範囲 1〜10000）は
「最大変位＝モデルサイズ 10%」への正規化にさらに乗算されるため、既定値では
建物サイズの 10 倍もの誇張表示になっていた。スライダーを廃止し、
**最大変位がバウンディングボックス対角長の 10% で表示される倍率を自動計算**
する方式に一本化した（`deform_scale_actual = model_size * 0.1 / max_disp`）。

実効倍率はビュー左上に「変形倍率 ×N（自動）」として注記する（実変位の
表示時のみ。モード形は固有ベクトルの規模が任意で倍率に物理的な意味がない
ため表示しない）。`App::deform_scale` フィールドは削除した（viewer 以外に
参照箇所がないことを確認済み）。

## 2. テスト

`viewer/mod.rs` の `#[cfg(test)]` に `interpolate_unreferenced_disp` の単体テストを
追加した（既存慣習に合わせ日本語テスト名）:

- 主架構に接続する節点の変位は補間で変更されない
- 大梁スパン中間の未参照節点は両端変位の線形補間になる（t=0.25）
- 梁軸から外れた未参照節点も最寄り線分の射影位置で補間される
- 主架構の線材が無ければ未参照節点の変位はゼロのまま
- 剛床マスター節点の変位は補間で上書きされない

## 3. 残課題・改善余地

- **N/Q/M 重ね表示の荷重ケース不一致（既知の制約）**: 重ね表示の変位は
  `App::current_static()`（ナビゲータの結果ケース選択に追従）だが、N/Q/M 図の
  応力値 `results.member_forces` は直近に実行した単一解析の結果に固定され、
  ナビゲータでの表示切替に追従しない。複数ケースを解析してナビゲータで
  切り替えた場合、応力図と変形が別ケースになりうる。根本対応は
  `member_forces` を結果ケースごとに保持してナビゲータ選択に追従させる
  改修（応力図表示全体の課題で、本変更より前からの既存挙動）。

- 補間ソースは 2 節点線材のみ。壁（面要素）にしか近くない未参照節点は
  最寄り線材からの補間になるため、壁支配の部位では追従精度が落ちる。
  必要なら壁ポリゴンへの射影補間を追加する。
- 小梁のたわみ（支持点間の曲げ変形）は表現されない（両端の支持点変位に
  よる剛体的な追従のみ）。小梁格子解析（`floor_grillage`）の結果を使った
  たわみ表示は将来課題。
- 未参照節点の探索は総当たり。超大規模モデルでは空間インデックス化を検討。
