# 検定比の3Dモデル上での可視化 申し送り

作成日: 2026-07-23
対象コード: `crates/squid-n-app/src/viewer/check_ratio.rs`（新規）、
`viewer/mod.rs`、`docs/result_view.md`

## 1. 何を実装したか

3D ビューアの表示モードに「検定比」を追加した（`ViewMode::CheckRatio`）。
`app.results.checks`（部材検定）・`app.results.joint_checks`（節点単位の検定：
柱梁接合部・パネルゾーン・耐震壁など）を集計し、部材・節点を検定比に応じて
着色する。詳細な表示仕様は利用者向けドキュメント `docs/result_view.md`
「検定比の表示」節を参照。

## 2. 設計判断

### 2.1 色規約は検定表（`design_view.rs`）と共通の `theme::status_color` を採用

検定表の検定比セルは既に `theme::status_color(ratio)`（≤0.8 緑／≤1.0 黄／
>1.0 赤=NG の3色）で着色されている。3D 可視化でも同じ関数をそのまま使い、
検定表と3Dビューで「同じ検定比なら同じ色」になるよう一貫させた。

**コンター（連続値グラデーション）は採らなかった**。N/Q/M 図のコンター表示
（`diagram.rs` の `contour_color`／`theme::ColorMap`）は値の相対的な大小を
連続色で見せることに意味があるが、検定比は 0.8・1.0 という**絶対的な判定
基準**を持つ値であり、「OK/注意/NG」の3値分類を一目で読み取れることの方が
重要と判断した。連続色にすると 0.79 と 0.81 が僅かな色差になり、判定基準
をまたいだことが視認しにくくなる。

### 2.2 集計関数を `egui::Painter` から独立させた

`max_ratio_by_elem` / `max_ratio_by_node`（`check_ratio.rs` 内、非公開）は
`(id, ratio, ok)` の軽量なタプル列を受け取り `HashMap<Id, (最大検定比, 全位置OKか)>`
を返す純粋関数として切り出した。`CheckResult`（`basis`/`detail` の
`String` を持つ）をそのまま受けると集計ロジックのテストに不要な文字列構築が
必要になるため、呼び出し側（`draw_check_ratio`）で `cr.ratio` / `cr.ok` へ
分解してから渡す設計にした。`diagram.rs` の `diagram_fill_polygons` /
`contour_color` を `egui::Painter` から切り離してユニットテストしている
既存の方針に倣っている。

内部実装は部材用・節点用で完全に同じ集計ロジック（最大値の採用・NG の
伝播）のため、`max_ratio_by_key<K, I>` というキー型ジェネリックな共通実装
1つに集約し、`max_ratio_by_elem` / `max_ratio_by_node` は型を固定した薄い
ラッパーにしている。

### 2.3 NodeId・ElemId のインデックス解決

部材の両端座標は `elem.nodes[i].index()` を直接 `pts` の添字として使う
（`ElemId`/`NodeId` の内部値が生成順にそのまま採番される前提。既存の
部材線描画・N/Q/M図が同じ前提で実装されている）。

一方、節点単位の検定（`joint_checks`）の対象節点を `pts` へ写像する際は、
念のため `app.model.nodes` を `enumerate()` して実際の配列位置を求め、
その位置の `node.id` が検定対象の `NodeId` と一致するかで突き合わせている
（`NodeId` の内部値が常に配列添字と一致するとは限らない前提に立った、より
安全な実装）。現状のモデル生成では両者は一致するはずだが、将来モデルの
節点配列が並び替え・フィルタされるようになっても壊れない書き方にした。

### 2.4 描画順序

部材線・壁の基本描画（灰色の部材線・青の壁）の後、選択ハイライト・支持
条件シンボルの前に検定比の着色を描く。検定比の色が部材線の上に重なり、
かつ選択中の部材（赤の太線）・支持条件シンボル（矢印・円弧）は検定比の色の
さらに上に重なって視認できるようにするため。

## 3. 残課題・注意

- 検定比図は「検定比」表示モード専用で、他モード（変形図・N/Q/M 図等）とは
  同時に重ねて表示できない（`viewer_panel` の `if mode == ViewMode::CheckRatio`
  分岐）。応力図に検定比を重畳する要望が出た場合は、現状の排他的なモード
  切替から「オーバーレイ」方式への変更を検討する必要がある。
- 節点検定の凡例集計（`max=`／`NG N件`）は部材検定・節点検定の両方を
  合算した値。`ViewMode::CheckRatio` の doc コメントは「部材検定の最大検定比
  で着色」とだけ書いているが、実装は節点検定（接合部等）の丸印も同じ凡例に
  含めている。将来、部材と節点で凡例を分けたいという要望が出た場合は
  `draw_legend` の集計元を分離する。
- 検定なしの部材・節点は色を一切変更しない（既存のグレー/青のまま）ため、
  「検定を実施していない」ことと「検定して OK だった」ことは色だけでは
  区別できない設計にしている（前者は無着色、後者は緑）。凡例に「未検定:
  グレー」の注記を入れることで対応した。
