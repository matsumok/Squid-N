# 応力図（N/Q/M 図）描画改善 申し送り

作成日: 2026-07-22
対象ブランチ: `claude/fix-axial-moment-diagrams-cfj9eu`
対象コード: `crates/squid-n-app/src/viewer/diagram.rs`（新規）、`viewer/mod.rs`、`app/mod.rs`

## 1. 何を直したか

GUI ビューアの N/Q/M 図に次の 2 つの描画不具合があり、修正した。

### 1.1 N 図のスパイク描画（画面全体に伸びるぐちゃぐちゃの塗り）

旧実装は「材軸 i 端 → 各張り出し点 → 材軸 j 端」を 1 つの閉じた
`egui::Shape::convex_polygon`（塗り＋輪郭 stroke）で描いていた。最大値 60px
正規化に対して値が相対的に極小の部材では図形が材軸上にほぼ潰れ、閉路の
折り返し点で epaint のマイター結合が発散し、部材軸方向へ画面外まで伸びる
スパイクになる。この問題は CMQ 図では対策済み（`paint_diagram_polygon` の
ドキュメントコメント参照）だったが、N/Q/M 図が未対策のまま残っていた。

対策（CMQ 図と同じ方針）:
- 輪郭は閉じない `egui::Shape::line` の折れ線として描く。
- 張り出しピークが `MIN_DIAGRAM_PX = 0.5` px 未満の潰れた図形はスキップ。
- スクリーン距離 `MIN_SEGMENT_PX = 0.25` px 未満の連続点は間引く。

### 1.2 M 図の塗りつぶし領域の誤り

`convex_polygon` は凸多角形前提（頂点 0 からの扇状三角形分割）のため、
M 図のような非凸・符号反転（両端 hogging＋中央 sagging）を含む閉多角形を
渡すと「実線と材軸の間」ではない誤った領域が塗られていた。

対策: 隣接サンプル区間ごとに「材軸上 2 点＋張り出し点 2 点」の台形を個別に
塗る方式へ変更（`diagram_fill_polygons`）。符号反転区間は線形補間のゼロ交差
`xc = x0 + (x1-x0)·v0/(v0-v1)` で片側符号のみの三角形 2 つに分割する。
台形・三角形は必ず凸なので扇状分割の問題が出ない。

## 2. 追加した機能: コンター表示

N/Q/M 図の塗りを「単色（従来）／値に応じた色分け（コンター）」で切替可能に
した（`App::diagram_contour`、ビューア上部の「コンター」トグル）。

- カラーマップは UI（コンター ON 時のコンボボックス）から選択可能:
  Viridis（既定）・Plasma・Turbo・Jet・青-白-赤（発散型）。既定は TONMANUAL §3
  「カラーマップ（連続値）」の規定（Viridis）に従い、`theme::ColorMap` の
  `#[default]` で維持している。当初は発散型・青-白-赤の固定実装だったが、
  ユーザー要望（jet/rainbow 等の一般的な配色→トンマナ準拠で Viridis 化→
  選択式に）を受けて段階的に現在の形になった。LUT と補間（`lut_sample`）は
  theme.rs（配色の単一情報源）に集約し、`contour_color` は
  `t∈[−1,1] → ColorMap::sample((t+1)/2)` の薄い写像（−max=LUT始端 →
  +max=LUT終端）。Turbo の中間アンカーは公知の 5 次多項式近似から算出した値
  （端点のみ Google 参照テーブルの正確な値）。
- 階調を滑らかにするため、コンター時は各サンプル区間を 8 分割
  （`CONTOUR_SUBDIV`）して細分クワッドごとに着色する。
- コンター時は塗り alpha 160（単色時 60）・輪郭は `GRAY_600` 1.0px。
- 左上凡例の下にカラーバー（−max / 0 / +max ラベル付き）を描く。

## 3. テスト

純粋関数（`diagram_fill_polygons` / `contour_color`）を egui::Painter 非依存で
切り出し、`diagram.rs` の `#[cfg(test)]` で単体テストしている（符号反転分割・
片側符号不変条件・ソート保険・カラーマップ端点とクランプ・単調性）。

## 4. 残課題・注意

- `draw_cmq_diagram`（CMQ 図）は従来の `paint_diagram_polygon` 方式のまま。
  塗りは C/Q 図が単調な台形状のため実害は小さいが、M（中央モーメント）成分は
  非凸形状になり得るので、将来 `diagram_fill_polygons` 方式への統一を検討。
- コンターの代表値はポリゴン頂点（材軸上の点を除く）の平均で、区間中点値と
  等価。三角形（ゼロ交差分割後）では頂点値そのままになるが視認上問題ない。
- epaint のマイター結合発散は egui 側の挙動であり、閉路ポリゴン＋stroke の
  組合せを応力図系の描画で使わないこと（本件と CMQ 図の再発防止規約）。
