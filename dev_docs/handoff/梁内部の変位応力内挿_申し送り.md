# 梁要素内部の変位・応力内挿 申し送り

作成日: 2026-07-23
対象ブランチ: `claude/beam-element-displacement-interpolation-8y5ius`
対象コード: `crates/squid-n-app/src/viewer/mod.rs`、
`crates/squid-n-solver/src/statics/analysis/mod.rs`、
`crates/squid-n-solver/src/statics/analysis/{seismic,wind}.rs`、
`crates/squid-n-solver/src/statics/linear/mod.rs`

利用者向け計算根拠は `docs/calc_basis/04_要素剛性/10_梁要素内部の変位と断面力.md`（新規）に記載した。

## 1. 変形図の梁たわみを Hermite 3 次で内挿（線形 → Hermite）

### 変更前

変形図（`ViewMode::Deformed`）・モード形（`ViewMode::Mode`）で、各要素は変形後の
2 節点位置を `painter.line_segment` で**直線**結びしていた。端部の回転自由度が
描画に反映されず、梁の曲げたわみ（部材中間のふくらみ）が全く表れなかった。

### 変更後

梁要素（`ElementKind::Beam`）に限り、端部の並進・回転（節点変位 6 成分）から
要素内部の変位場を **Hermite 3 次形状関数**で内挿し、`DEFORM_CURVE_SEGMENTS = 12`
分割の折れ線（`egui::Shape::line`）として曲線描画する。

- 実装関数 `beam_deformed_polyline`（`viewer/mod.rs`）。要素ローカル系で
  軸方向は線形 (1−ξ, ξ)、曲げ 2 面は Hermite（`N1..N4`）で内挿し、グローバルへ
  戻して未変形材軸上の各点へ加える。形状関数・符号規約は等価節点力
  （`member_load.rs`）と一致させた（局所 z 面は θy の符号反転）。
- ξ=0,1 で回転項が消え、端部は節点変位（＝節点マーカー位置）に厳密一致する。
- **表示専用**であり解析結果（`StaticOnce::disp`・内力）は変更しない。要素は
  せん断変形を含む Timoshenko 梁だが、内部変位の内挿は Euler–Bernoulli の
  Hermite 曲線で近似する（変形形状の可視化として標準的）。
- 梁以外（ブレース＝軸材、壁＝面要素、二次部材、ゼロ長梁）は従来どおり直線。
  未変形表示・その他モードも従来どおり直線。

## 2. 応力（内力）回復の非対称を解消（Analysis 経路にも部材荷重を重ね合わせ）

### 変更前の不具合

部材中間荷重の固定端内力の重ね合わせ（`superpose_member_loads`）が、
一度きりの `linear::linear_static_once` 経路にはあったが、**分解済み K を再利用する
`Analysis` ファサード経路**（`linear_static` / `linear_combination` / 地震 / 風）には
無かった。アプリ本体・MCP の主経路は `Analysis` 側を使うため、等分布荷重を受ける
梁でも内力図の M が端部間で線形になり（放物線分が欠落）、中央曲げが過小評価
されていた。

### 修正

`superpose_member_loads` を `pub(crate)` にして共有し、`Analysis::recover_member_forces`
でも各断面へ重ね合わせるようにした。これで両経路が一致し、等分布荷重で
M が放物線・Q が線形の正しい分布になる。

- `linear_static(lc)`: 当該荷重ケースの `member` をそのまま重ね合わせ。
- `linear_combination(combo)`: 各項の `member` を組合せ係数で倍して集約
  （`scale_member_load`）。荷重ベクトル f と同じ線形結合を内力回復にも適用する。
- 地震 (`seismic.rs`)・風 (`wind.rs`): 節点荷重のみのため空スライスを渡す。

後方互換のための分岐は設けず、共有関数に一本化した（最も素直な形）。

## 3. テスト

- `analysis/tests.rs`
  - `analysis_linear_static_superposes_member_load`: 単純梁 UDL で
    `Analysis::linear_static` の中央 Mz = wL²/8・端部 ~0 を検証。
  - `analysis_linear_combination_scales_member_load`: 1.5×UDL の組合せで
    中央 Mz = 1.5·wL²/8 を検証。
- 既存の `linear/tests.rs`（`linear_static_once` 経路の wL²/8・PL/4 検証）は
  変更なしで継続。

## 4. 残課題・改善余地

- 変形図の梁たわみ内挿は部材中間荷重を考慮しない（端部 6 自由度のみからの
  Hermite）。UDL 下の梁でも表示上のたわみ曲線は 3 次までで、荷重による
  4 次のふくらみは表現しない（内力図は 2 で放物線化済み）。必要なら
  固定端たわみの重ね合わせで表示を精緻化できる。
- `linear_static_once` 経路と `Analysis` 経路の二重実装は依然として残る
  （内力回復の重ね合わせは共有関数に一本化したが、組立・求解の入口が別）。
  将来的に入口を統合する余地がある。
