# テーブルUI グリッド操作（スプレッドシート的編集）実装仕様

テーブル入力に「Excel と相互にコピー/ペーストできる」「矩形選択・キーボード移動で
編集できる」操作感を与える**グリッド操作レイヤ**の実装仕様。
特定のテーブル専用の作り込みではなく、**アダプタ（§3.4）を差し替えるだけで
アプリ内の各テーブル（節点・部材・荷重など 12 種）へ順次展開できる汎用レイヤ**
として設計する。使い捨てプロトタイプ（egui 実機）で操作感・実装方式を
検証済みであり、本書はその検証結果と設計判断をまとめたものである。

- 対象読者: 本機能の実装者／レビュアー
- 関連文書: [UI設計.md](UI設計.md)（テーブルUI全般）、
  [../handoff/TONMANUAL.md](../handoff/TONMANUAL.md)（配色・寸法の規約）

---

## 0. 本仕様について

### 0.1 目的

Squid-N のテーブル（節点・部材・荷重など 12 種）はセル単位の `TextEdit` 入力のみで、
大量入力・一括修正の効率が悪い。実務では Excel/スプレッドシートで座標表や荷重表を
組み立ててから投入する流れが一般的なため、次を可能にする:

1. Excel からのブロック貼り付け（新規行の一括投入を含む）
2. テーブルから Excel への範囲コピー（TSV）
3. 矩形選択・行/列/全選択・キーボードでのセル移動と編集

### 0.2 スコープ（第1弾）

- 矩形範囲選択（マウスドラッグ / Shift+クリック / Shift+矢印）
- 行・列・全選択（行ヘッダ / 列ヘッダ / 左上コーナー / Ctrl+A）
- Ctrl+C（TSV コピー）・Ctrl+V（TSV ペースト）・Delete（選択範囲クリア）
- 矢印 / Tab / Enter によるセル移動、2モード式編集（F2 / ダブルクリック / 文字キー / Backspace）
- 表外クリック・Esc による選択解除
- **新規行プレースホルダ**（最下部の常設空行。空テーブルへの一括投入と手入力での行追加の受け口。§4.5）
- **行削除**（右クリックメニュー／Ctrl+Delete。§4.6）
- 適用第1弾は**節点テーブル**（全列 f64・行追加が座標のみで完結するため）

### 0.3 非スコープ（将来拡張。§10）

- フィルハンドル（ドラッグでの連続値埋め）・Ctrl+X（切り取り）
- 列ソート・行の並び替え
- コンボボックス／チェックボックス列を含むテーブルへの適用（API 安定後に判断）

### 0.4 検証状況

操作仕様・描画仕様・egui 実装方式は、使い捨てプロトタイプ
（`examples/grid_prototype.rs`、フォーク側ブランチ `claude/grid-prototype` に保存）で
実機検証済み。§7 の「egui 実装上の要点」はすべて実際に踏んだ問題と
その対処である。本仕様は spec とプロトタイプ双方への敵対的レビュー
（空テーブル・スクロールバー・フォーカス残留などの障害シナリオ）を経ている。

## 1. 用語

| 用語 | 定義 |
| --- | --- |
| グリッド操作レイヤ | テーブルUIに矩形選択・クリップボード・キーボードナビゲーションを与える汎用レイヤ。ドメイン知識を持たない |
| データ列 | グリッド操作レイヤが選択・編集・クリップボードの対象とする列。列番号はデータ列の 0 始まり |
| 行ヘッダ列 | ID など行の識別を示す列。**データ列ではなく、グリッドの座標系の外**。クリックは「行全体の選択」を意味する（Excel の行番号ヘッダと同じ） |
| 選択モード | セルがラベル描画され、矩形選択・コピー/ペースト・セル移動を受け付ける状態。テーブルの既定状態 |
| 編集モード | 単一セルだけが `TextEdit` 化され、キー入力がセル内テキスト編集に向かう状態 |
| アクティブセル | 選択の起点となるセル（Excel の枠付きセル）。編集・通常移動の基準。内部状態では `anchor` |
| 矩形選択 | アクティブセル（anchor）と選択端（cursor）で定まる連続セル範囲 |
| はみ出し行 | ペーストブロックのうち表の末尾行を超える部分。行追加に対応するテーブルでは確認なしで自動的に行として追加される（undo 1 回で戻る） |
| 新規行プレースホルダ | 行追加に対応するテーブルの最下部に常設される空行（行ヘッダは「＋」）。まだデータではないが、選択・貼り付け起点・直接入力の受け口になる（Airtable / Access の新規行と同じパターン） |

## 2. 既存クレートを採用しない判断

egui 本体・egui_extras には「セル範囲選択＋表としてのコピー/ペースト」の機能はない
（egui にあるのは `Event::Copy` / `Event::Paste(String)` / `Context::copy_text` という
プリミティブのみ）。サードパーティは調査の上、以下の理由で採用せず自作とした:

| 候補 | 不採用理由 |
| --- | --- |
| egui-data-table (kang-sw) | undo/redo を内部に持ち**無効化できず**（履歴上限 0 指定は既定 100 に置換される仕様）、「モデル編集は必ず squid-n-edit コマンド経由（undo・MCP 共通経路）」という本プロジェクトの原則と両立しない。`DataTable<R>` が行データを所有するため、MCP・3D ビュー等からのモデル変更との双方向同期層が別途必要になる |
| egui_tabular (romixlab) | crates.io 未公開・姉妹リポジトリへの path 依存を含み、依存として引けない。undo 未実装 |

ただし両者は設計の先行事例として有用であり、本仕様の一部（編集中はクリップボード
イベントを表に流さない、書込み可否をアダプタに委ねる等）は同じ結論に到達している。

必要なのは「選択＋クリップボード＋ナビゲーション」の薄い相互作用レイヤだけであり、
既存の `TableBuilder` ベースのテーブル実装を保ったまま被せられる。

## 3. アーキテクチャ

### 3.1 二層構成とモジュール配置

```
crates/squid-n-app/src/grid/
├── mod.rs        // 公開面
├── core.rs       // ① 純ロジック（egui 非依存・ヘッドレステスト可能）
└── widget.rs     // ② egui グルー（描画・入力処理、gui feature 限定）
```

- **① 純ロジック（grid_core）**: 選択状態機械・TSV 変換・タイル展開・ペースト検証。
  egui に依存せず、単体テストで網羅する。テーブル横断で共有する中核。
- **② egui グルー**: セル描画（選択/編集モード）・行/列/コーナーヘッダ・
  イベントルーティング・コンテキストメニュー・フィードバックフラッシュ。
  薄く保ち、ロジックを持ち込まない。

グリッド操作レイヤ自体はテーブル固有の知識を持たず、テーブルごとの差は
すべてアダプタ（§3.4）に隔離する。これにより第 2 弾以降のテーブル展開は
「アダプタ実装の追加」だけで済む。

### 3.2 座標系: データ列のみ

グリッドの座標系（`CellRef.col`）は**データ列のみ**を数える。行ヘッダ列（ID）は
グリッドの関知外で、テーブル側が描画し、クリックを `select_row` 呼び出しに変換する。

この分離により:
- 「編集不可の ID 列に貼り付けようとして全体拒否」という不正カテゴリが**構造的に消滅**
- `min_col` のような特別扱いフラグが不要
- ペースト検証の不正は「列はみ出し」「値パース失敗」の 2 種だけになる

### 3.3 grid_core の型と API（プロトタイプで検証済みの形）

```rust
/// セル参照（0 始まりの行・データ列）
pub struct CellRef { pub row: usize, pub col: usize }

/// 矩形選択範囲（両端含む）
pub struct SelRect { pub r0: usize, pub r1: usize, pub c0: usize, pub c1: usize }

/// 選択／編集モードの状態機械
pub struct GridState {
    pub rows: usize,
    pub cols: usize,
    pub anchor: CellRef,          // アクティブセル（選択の起点・編集対象）
    pub cursor: CellRef,          // 選択端（Shift 拡張・ドラッグで動く側）
    pub editing: Option<CellRef>, // Some の間は編集モード
    pub active: bool,             // false の間は「選択なし」（§4.4）
}

impl GridState {
    pub fn new(rows: usize, cols: usize) -> Self;       // active=false で開始
    pub fn rect(&self) -> SelRect;
    pub fn click(&mut self, cell: CellRef, extend: bool);
    pub fn drag_to(&mut self, cell: CellRef);
    pub fn move_cursor(&mut self, dr: isize, dc: isize, extend: bool);
    pub fn select_row(&mut self, row: usize, extend: bool);
    pub fn select_col(&mut self, col: usize, extend: bool);
    pub fn select_all(&mut self);
    pub fn begin_edit(&mut self);                        // anchor で編集開始
    pub fn end_edit(&mut self);
    pub fn deactivate(&mut self);                        // 選択解除（anchor は保持）
    pub fn clamp_selection(&mut self);                   // 行数変更後の是正
}
```

**不変条件（テストで担保すること）**:

- `rows == 0 || cols == 0` の間は `active` になれない。選択系メソッド
  （click / select_row / select_col / select_all / move_cursor）はすべて no-op。
  これを破ると空グリッドでの Ctrl+A → Ctrl+C が範囲外アクセスでパニックする。
- `select_row` / `select_col` の extend（Shift+クリック・ドラッグ継続）は
  **Excel 準拠**で「起点行〜対象行 × 全列」（列は「起点列〜対象列 × 全行」）に
  広げる。アクティブセルは先頭列（行選択時）／先頭行（列選択時）に寄る。

```rust

/// クリップボード変換
pub fn rect_to_tsv(rect: SelRect, cell_text: impl Fn(usize, usize) -> String) -> String;
pub fn parse_tsv(text: &str) -> Vec<Vec<String>>;        // CRLF・末尾改行を吸収
pub fn tile_block(block: &[Vec<String>], sel_rows: usize, sel_cols: usize)
    -> Vec<Vec<String>>;                                 // Excel 互換タイル展開（§5.2.3）

/// ペースト検証（all-or-nothing）。詳細は §5.2
pub struct PastePlan {
    /// (行, データ列, セル文字列)。行は末尾を超えうる。
    /// 値の型変換はアダプタが適用時に行う（検証は validate 済みなので失敗しない）
    pub set: Vec<(usize, usize, String)>,
    pub extra_rows: usize,             // はみ出し行数（自動追加の対象。§5.2.5）
    pub skipped_empty: usize,          // 空セル（既存値維持）の数
    pub anchor: CellRef,
    pub block_rows: usize,
    pub block_cols: usize,
}

/// validate = アダプタの `validate_cell`。plan_paste が全セルを検証し、
/// 1 件でも Err ならペースト全体を拒否する（all-or-nothing）。
/// 検証を全部通ってから初めて適用する、という順序を変えないこと。
pub fn plan_paste(
    block: &[Vec<String>],
    anchor: CellRef,
    rows: usize,
    cols: usize,
    validate: impl Fn(usize, usize, &str) -> Result<(), String>,
) -> Result<PastePlan, Vec<String>>; // Err = 不正理由の一覧
```

> プロトタイプは全列 f64 のため validate を `parse::<f64>` に固定しているが、
> 本実装では上記のとおりアダプタ委譲とする（文字列列・列挙列への拡張のため）。

### 3.4 テーブルアダプタ（GridAdapter）

汎用レイヤとテーブルの境界。ドメイン知識（セルの型・行追加の可否と方法）は
すべてこちら側に置く。

```rust
pub trait GridAdapter {
    fn rows(&self) -> usize;
    fn cols(&self) -> usize;
    /// コピー・編集開始用のセル文字列。**内部値の正準文字列**を返す
    /// （表示用に丸めた文字列を返さない。§5.1）
    fn cell_text(&self, row: usize, col: usize) -> String;

    /// 行ヘッダ（ID 列）の表示文字列。既定は 1 始まりの行番号
    /// （T4 実装時に追加: 節点は ID＝配列位置のため 0 始まり ID を表示する）
    fn row_label(&self, row: usize) -> String { (row + 1).to_string() }

    /// ペースト・編集確定の 1 セル分の検証。Err はセル単位の不正理由。
    /// plan_paste（§3.3）がこれを全セルに適用する
    fn validate_cell(&self, row: usize, col: usize, text: &str) -> Result<(), String>;

    /// 検証済みセル群の適用。squid-n-edit の複合コマンド 1 個に落とす（§3.5）。
    /// append_rows > 0 なら先に行を追加する（追加行の貼り付け対象外の列は
    /// アダプタの既定値）。呼び出し規約:
    /// - 通常のペースト: cells 全部 + append_rows = はみ出し行数（自動追加。§5.2.5）
    /// - 行追加非対応テーブル: widget が cells を row < rows() にフィルタして渡し、
    ///   append_rows = 0（はみ出し分は切り捨て）
    /// - 新規行プレースホルダでの編集確定: cells = その 1 セル、append_rows = 1
    fn apply_block(&mut self, cells: &[(usize, usize, String)], append_rows: usize);

    /// 選択範囲クリア（Delete）。クリアの意味（0 埋め・既定値・禁止）は
    /// アダプタが決める。cells は選択が跨ぐ**実データ行**のセルのみ
    /// （プレースホルダは渡されない）。クリアしたセル数を返す
    /// （クリア非対応のテーブルは 0 を返し、widget が理由をログする）
    fn clear_cells(&mut self, cells: &[(usize, usize)]) -> usize;

    /// はみ出し行の追加・新規行プレースホルダに対応するか
    /// （節点 = true、非対応テーブル = false）
    fn can_append_rows(&self) -> bool;

    /// 行削除に対応するか
    fn can_delete_rows(&self) -> bool;

    /// 1 行分の削除可否検証。参照中（部材が接続された節点等）なら
    /// Err に「なぜ削除できないか（参照元）」を返す
    fn validate_row_deletion(&self, row: usize) -> Result<(), String>;

    /// 検証済みの行削除。squid-n-edit の複合コマンド 1 個に落とす。
    /// ID＝配列位置の繰り上げと undo の整合のため、行番号の**降順**で
    /// DeleteNode を並べること
    fn delete_rows(&mut self, rows: &[usize]);
}
```

- `can_append_rows() == false` のテーブルでは、プレースホルダ行を表示せず、
  はみ出し分は確認なしで切り捨ててイベントログに通知する。
- 選択範囲クリア（Delete）の意味（0 埋め・既定値・禁止）もアダプタが決める
  （`clear_cells`。T3 実装時に追加: widget から呼ぶ受け口が trait に必要なため）。

### 3.5 squid-n-edit 統合: 複合コマンド

ペースト・範囲クリアは複数セル（＋行追加）の変更になるため、
**undo 1 回で丸ごと戻せる**よう squid-n-edit に複合コマンドを追加する:

```rust
/// 子コマンド列を順に適用し、逆操作を「逆順の逆コマンド列」として返す
pub struct CompositeCommand {
    pub label: String,
    pub children: Vec<Box<dyn EditCommand>>,
}
impl EditCommand for CompositeCommand {
    fn apply(&self, model: &mut Model) -> Box<dyn EditCommand> {
        let inverses: Vec<_> = self.children.iter().map(|c| c.apply(model)).collect();
        Box::new(CompositeCommand {
            label: self.label.clone(),
            children: inverses.into_iter().rev().collect(),
        })
    }
    fn label(&self) -> &str { &self.label }
}
```

既存の `EditCommand::apply` が逆コマンドを返す設計（squid-n-edit）のため、
この追加は小さく、MCP 経由の編集とも自然に共存する。
節点テーブルのペーストは `AddNode` × extra_rows ＋ `SetNodeCoord` × セル数を
1 個の `CompositeCommand`（label 例:「節点座標の貼り付け」）にまとめる。

## 4. 操作仕様

### 4.1 選択モデル（Excel 準拠）

- 選択は `anchor`（アクティブセル）と `cursor`（選択端）の対で表す矩形。
- **アクティブセルの枠は常に選択の起点に留まり**、ドラッグ・Shift+矢印・
  Shift+クリックでは `cursor` 側だけが動く。
- 通常移動（矢印など）は **anchor 基準**で移動し、選択を単一セルに畳む。
- 編集開始で選択は anchor の単一セルに畳まれる。

### 4.2 マウス

| 操作 | 動作 |
| --- | --- |
| セルをクリック | そのセルを単一選択（アクティブセル移動） |
| セルをドラッグ | 矩形選択（起点＝アクティブセル） |
| Shift+クリック | アクティブセルを動かさず選択端だけ拡張 |
| セルをダブルクリック | そのセルで編集モード開始（現在値を初期値に） |
| 行ヘッダをクリック / ドラッグ / Shift+クリック | 行全体を選択 / 行範囲を拡張（Shift+クリックは Excel 準拠で「起点行〜対象行 × 全列」） |
| 列ヘッダをクリック / ドラッグ / Shift+クリック | 列全体を選択 / 列範囲を拡張（同上、「起点列〜対象列 × 全行」） |
| 左上コーナーをクリック | 全選択（**実データ行のみ**。プレースホルダは含まない） |
| 表外をクリック | 選択を完全解除（§4.4）。「表外」＝**テーブル領域（スクロールバー含む外形矩形）の外**。スクロールバー操作では解除されない |

### 4.3 キーボード

| キー | 選択モード | 編集モード |
| --- | --- | --- |
| 矢印 | セル移動（+Shift で選択拡張） | （TextEdit のカーソル移動） |
| Tab / Shift+Tab | 右 / 左へ移動 | 確定して右へ |
| Enter / Shift+Enter | 下 / 上へ移動 | 確定して下へ |
| F2 | アクティブセルを編集 | — |
| 文字キー | その文字を初期値として編集開始（Excel 同様） | 文字入力 |
| Esc | 選択解除 | 編集キャンセル（値を戻す） |
| Delete | 選択範囲をクリア（アダプタ定義の既定値へ） | 文字削除 |
| Backspace | **アクティブセルを空の状態で編集開始**（Excel 準拠。範囲クリアは Delete のみ） | 文字削除 |
| Ctrl+C | 選択範囲を TSV コピー | テキストコピー |
| Ctrl+V | TSV ペースト | テキスト貼り付け |
| Ctrl+A | 全選択（実データ行のみ） | テキスト全選択 |
| Ctrl+Delete | 行削除（§4.6） | — |

編集確定時の値が不正（数値列に非数値など）の場合は変更せず、理由をイベントログへ。
**空文字列のまま確定した場合は「変更なし」**（Backspace → Enter は no-op。
本アプリのセルは空にできない型が大半のため、空確定をクリアとは解釈しない）。

### 4.4 選択解除と復帰（Excel からの意図的な差分）

Excel には「選択なし」状態が存在しないが、本アプリではテーブルが 3D ビューや
インスペクタと同居するため、**表外クリックまたは Esc で選択を完全解除**する
（他パネル作業中に青い選択表示が残るのはノイズになるため）。

- 解除中は選択描画なし。Ctrl+C / Ctrl+V / Delete / 文字キー編集開始は no-op。
  **ペーストは無言で無視**する（貼り付け先が定まっていない操作に
  フィードバックは出さない）。コピーはログに理由を出す。
- `anchor` は解除中も保持し、**矢印キーまたはセルクリックで直前位置から復帰**する。
- 編集モード中・コンテキストメニュー表示中の表外クリックは解除の対象外
  （編集中のクリックは「確定」が先。メニュー中の除外は §4.6）。
- テーブル表示直後は「選択なし」から始まる。

### 4.5 新規行プレースホルダ

行追加に対応するテーブル（`can_append_rows() == true`）は、最下部に常設の
空行（プレースホルダ）を 1 行表示する。行ヘッダは番号ではなく「＋」。

- **グリッド座標系上は「最終行＝実データ行数」の位置にある通常の行**として
  選択・ナビゲーションの対象になる（`GridState.rows = 実データ行数 + 1`）。
- **直接入力**: プレースホルダのセルで編集を確定すると行が 1 行追加され、
  入力した列に値が入る（他列はアダプタの既定値）。Enter 確定なら次の
  プレースホルダ（1 行下）にアクティブセルが移り、連続入力できる。
- **貼り付け起点**: プレースホルダを起点に貼ると全行がはみ出し扱いになり、
  自動で行が追加される（§5.2.5）。**空のテーブルもプレースホルダ 1 行
  だけの状態から始まる**ため、「空の表に Excel から一括投入」がこの経路で成立する
  （空テーブル特有の例外処理は存在しない）。
- コピー対象になった場合は空文字列（Excel の空行コピーと同じ）。
  Delete のクリア対象にはならない。全選択（コーナー / Ctrl+A）には含まれない。
- 既存の「追加フォーム」は当面併存させ、プレースホルダが十分機能することを
  確認した後の整理（置き換え）は別途判断する。

### 4.6 行の削除

`can_delete_rows() == true` のテーブルで有効。操作は
**右クリックメニュー（行ヘッダ・セルのどちらでも）**を主経路とし、
キーボードは **Ctrl+Delete**（Delete＝中身のクリアの「強い版」という段階的な割当）。

> Excel の Ctrl+マイナスは採用しない。egui が既定で Ctrl+マイナス/プラスを
> UI 全体ズームに使っており、**Alt 併用（Ctrl+Alt+マイナス）でも egui の
> ズーム判定は Alt の有無を区別せず発火する**（プロトタイプで実証）。
> 共存にはキーボードズームの無効化（`Options::zoom_with_keyboard = false`）が
> 必要だが、これはテーブル機能ではなく**アプリ全体の決定**であり、
> 表の都合で行わない（将来アプリ側でズーム無効化が決まれば
> Ctrl+マイナス併用を再検討してよい）。

- **メニュー項目は対象行数を明示する**: 複数行が対象なら
  「選択した N 行を削除」、単一行なら「この行を削除」。
  「複数行選択したのに 1 行しか消えない／逆」の驚きをラベルで防ぐ。
- **Delete キーとは役割を分離する**: Delete は常に「中身のクリア」であり、
  行自体は消えない（Excel と同じ。同じキーの意味が状態で変わると誤削除を招く）。
- **対象は選択が跨ぐ実データ行すべて**（r0〜r1。セル選択でもよい。
  Google Sheets と同じ）。新規行プレースホルダは対象外。
- 選択外の行ヘッダ・セルを右クリックした場合は、その行（セル）を選択してから
  メニューを出す（Excel と同じ）。
- **メニュー表示中のクリックは「表外クリック＝選択解除」（§4.4）の判定から
  除外する**。メニュー項目を押した瞬間に選択が消え、削除対象が失われる
  誤動作を防ぐ。
- **参照中の行を含む場合は全体拒否**（ペーストと同じ all-or-nothing）:
  1 行でも削除不可（部材が接続された節点等）なら何も消さず、対象範囲を
  赤フラッシュし、どの行がなぜ削除できないか（参照元）を赤字ログに列挙する。
- 削除は複合コマンド 1 個（DeleteNode を行番号の降順に並べる）で、
  **undo 1 回で全行が元の位置に戻る**（ID の繰り上げも復元される）。
- 削除後は削除位置に続く行を行選択状態にする（Excel と同じ）。
  最終行まで消えた場合は新規行プレースホルダが選択される。

## 5. クリップボード仕様

### 5.1 コピー（Ctrl+C）

- 選択範囲のセル文字列を、セル間 Tab・行間 LF の **素の TSV** にして
  システムクリップボードへ（`Context::copy_text`）。
- セル文字列は**内部値の正準文字列**（f64 の Display 等）を使う。
  表示用に丸めた文字列を使うと、丸め表示の列を将来作った際にコピーで
  精度が落ちるため（0.847 が「0.85」でコピーされる事故の予防）。
- エスケープは行わない（対象列は数値中心で、Excel 自体も貼り付け時に
  エスケープを解さないため）。セル内にタブ・改行を含みうる文字列列へ
  対応する際に再検討する（§10）。
- 行・列・全選択のコピーも同じ経路（選択矩形をそのまま TSV 化）。
- 行ヘッダ列（ID）は選択に含まれないためコピーにも含まれない。

### 5.2 ペースト（Ctrl+V）

処理順序: パース → タイル展開 → 検証（all-or-nothing）→ 適用（はみ出し行は自動追加）。

#### 5.2.1 パースとサイズ上限

- `Event::Paste(String)` で受けたテキストを Tab / LF で分割。
- Excel の CRLF（`\r\n`）と末尾改行は吸収する（`\r` 除去・末尾空行を破棄）。
- **ブロックが 100,000 セル（行数×最大列数）を超える場合は全体拒否**し、
  イベントログにサイズと上限を報告する。Excel の「列全体コピー」
  （104 万行）の誤ペーストで UI がフリーズするのを防ぐ暴発ガードであり、
  実務のモデル規模（数千〜数万セル）には影響しない。

#### 5.2.2 タイル展開（Excel 互換）

選択範囲の行数・列数が**ともに**コピー元ブロックの整数倍（かつどちらかが拡大）
のときだけ、ブロックを繰り返して選択範囲サイズへ展開する。

| コピー元 | 選択 | 挙動 |
| --- | --- | --- |
| 1×1 | 3×2 | 3×2 に複製（単一値での範囲埋め） |
| 2×1 | 4×1 | 2 回繰り返し |
| 2×1 | 3×1 | 展開せず、起点から 2 セルだけ貼る（選択サイズは無視） |
| 3×2 | 2×2 | 展開せず、起点から 3×2 を貼る（選択の外へはみ出す） |

割り切れない場合に「選択の外へはみ出して貼る」のも Excel の単一矩形選択と同じ。
はみ出した先には §5.2.4（列）・§5.2.5（行）の規則がそのまま適用される。

**サイズ不一致の通知**: 選択が複数セルで、かつタイル展開にもならない
サイズ不一致の貼り付けは、成功させた上で「選択範囲 R×C とコピー元 R'×C' の
サイズが一致しません」をログに出し、**実際に貼り付けられた範囲を注意色で
フラッシュ**する（§6.4）。貼り付け先の選択ミスに気づかせるための措置で、
動作自体は Excel 互換のまま変えない。

#### 5.2.3 検証は all-or-nothing

1 セルでも不正（値パース失敗・列はみ出し）があれば**ペースト全体を適用しない**。
拒否時はどのセルがなぜ不正かをイベントログに列挙する（例:
「ブロック2行1列目『abc』: 数値として解釈できません」。件数が多い場合は先頭
10 件＋残数）。予測可能性を優先した設計判断であり、「貼れた分だけ貼る」動作は
採らない。誤操作は undo 1 回（§3.5）で救済する。

#### 5.2.4 空セルと列はみ出し

- **空セルは「変更なし」マーカー**として扱い、既存値を維持する。不正には数えない
  （Excel の空白混じり範囲を貼れるようにするため。意図的な削除は Delete に分離済み）。
- 貼り付け先がデータ列の範囲を右に超える場合は不正（全体拒否）。

#### 5.2.5 はみ出し行（行の自動追加）

貼り付けが表の末尾行を超える場合、アダプタが行追加に対応
（`can_append_rows() == true`、例: 節点）なら**確認なしで自動的に行を追加して
貼り付ける**（Google Sheets の下端超え貼り付けと同じ挙動）。

無確認でも安全といえる根拠（すべて本仕様の他項で担保済み）:

- 適用は複合コマンド 1 個なので **undo 1 回で行追加ごと戻る**（§5.2.6）
- 貼り付け範囲が選択状態になるため**増えた行が見える**（§5.3）
- 適用結果（追加行数を含む）が**ログに明記**される（§5.2.6）
- **サイズ上限**（§5.2.1）で巨大ペーストの暴発は事前に止まる

補足:

- 非対応テーブルでは確認なしで切り捨て、イベントログに通知する。
- 追加された行の貼り付け対象外の列は、アダプタが定める既定値とする。
- 「はみ出し」は本仕様の内部用語であり、利用者向けの文言には使わない。

#### 5.2.6 適用と undo

適用は §3.5 の複合コマンド 1 個として実行し、undo 1 回で行追加も含めて元に戻る。
適用結果（適用セル数・空セルスキップ数・追加行数・切り捨てセル数）を
イベントログに 1 行で報告する。

### 5.3 貼り付け後の選択

貼り付けたブロックの矩形（タイル展開後・空セル含む見かけの矩形）を選択状態にする
（Excel と同じ）。アクティブセルは貼り付け起点（左上）。どこに何が入ったかが
一目で分かり、続けてコピー・Delete・再貼り付けができる。

## 6. 見た目仕様（スプレッドシート様式・TONMANUAL 準拠）

プロトタイプでの試行の結果、「セルごとに独立した入力欄風の枠」は
(1) 隣接セルの罫線が二重に見えて騒がしい、(2) スプレッドシートに見えないため
可能な操作（ドラッグ選択・コピペ）が伝わらない——という理由で不採用とし、
**スプレッドシート様式**に確定した。

### 6.1 セルと罫線

- 本体は**白地＋共有罫線**。ストライプは使わない。
- 罫線は gray-200 の 1px。**各セルが右辺と下辺のみを描き**、隣接セルと 1 本の線を
  共有する（二重線の防止。egui_extras のセル間スペーシングの跨ぎ方は §7.5）。
- 表の**上外周はヘッダ行が、左外周は行ヘッダ列が担当**して描く
  （右・下辺方式だけでは格子の上辺・左辺が欠けるため）。
- 角丸・セル間の隙間・セル個別の枠は使わない。
- 新規行プレースホルダのセルは、ごく薄いグレー地（gray-100 半透明）で
  「まだデータではない」ことを示す。

### 6.2 選択の描画（Excel 式）

- 選択範囲の**外周だけ**を blue-500 の 2px 枠で囲う（各セルが自分の担当辺
  ＝選択矩形の境界に接する辺だけを描く）。単一セル選択ではこの枠が
  そのままカーソル枠になる。
- 範囲の内側は blue-300 の半透明（α≈70/255）で塗る。
- **アクティブセル（anchor）だけは塗らず白抜き**にし、入力が向かう先を示す。
- 文字色は gray-700（薄い塗りの上でもそのまま可読）。

### 6.3 行・列・コーナーヘッダ

- ヘッダ地は gray-100、罫線は本体と共有。
- 選択範囲に含まれる行・列のヘッダは blue-300 半透明（α≈80/255）でハイライト
  （Excel の行番号・列記号ヘッダと同じ合図。セル本体の選択より弱く、主従を付ける）。
- 左上コーナーは「ID」見出しを兼ね、クリックで全選択。

### 6.4 失敗・注意フィードバック（フラッシュ）

操作の失敗・注意すべき結果は、イベントログだけでは見逃されるため、
**対象の場所を表の上でフラッシュ**して示す。色は結果の意味で使い分ける
（TONMANUAL §3 の意味色。**同じ色が正反対の結果を指さない**ようにする）:

- **赤（ERROR_RED）＝ 何も適用されていない**: 貼り付けの全体拒否時は
  貼り付け先になるはずだった範囲（ブロックの足跡のうち表内に収まる部分）、
  編集確定の失敗時は当該セル、サイズ上限超過（§5.2.1）では選択範囲、
  行削除の全体拒否（§4.6）では削除対象だった行範囲。
- **黄（注意色）＝ 適用されたが確認せよ**: サイズ不一致の貼り付け（§5.2.2）で、
  実際に貼り付けられた範囲。

いずれも半透明で約 1 秒フェードアウト（TONMANUAL §8: モーションは控えめ・
短く。アニメーション中のみ再描画を要求する）。
イベントログの**エラー行は ERROR_RED の赤字**で描画し、
フラッシュ → ログ詳細への視線誘導をつなげる。

### 6.5 寸法・フォント

- 行高は固定 px にせず `theme::table_row_height`（Body 行高＋余白）から導出する
  （TONMANUAL §4 の導出原則）。ヘッダ行も同じ高さ。
- セル文字は TextStyle::Body、左寄せ・上下中央、左パディング 4px。
- 編集モードの `TextEdit` は選択モードのセルと**同じ矩形・同じ左余白**に固定し、
  モード切替で文字が 1px も動かないようにする（§7.4）。

## 7. egui 実装上の要点（プロトタイプで実証済み）

実装時に必ず踏む egui 固有の問題と、検証済みの対処。

### 7.1 イベントルーティング（キー衝突の解消）

- グローバル入力（`ctx.input().events`）の処理は**選択モードのときだけ**行う。
  編集モード中は一切処理しない
  （`TextEdit` が Ctrl+C/V・矢印・文字入力を消費する。この分離が 2 モード式の core）。
- `Event::Copy` / `Event::Cut` / `Event::Paste(text)` を使う（生の Ctrl+C 判定をしない。
  OS ごとの修飾キー差は egui が吸収する）。コピーは `ctx.copy_text(tsv)`。
- 文字キーでの編集開始は `Event::Text(t)` で拾い、制御文字を除去してから初期値にする。

### 7.2 ドラッグ選択は hovered() に頼れない

egui はドラッグ中、**ドラッグ元以外のウィジェットの `hovered()` を false にする**ため、
`resp.hovered()` ベースでは範囲選択が追従しない。対処:

- 各セルで `pointer.interact_pos()` とセル矩形の**包含判定**を直接行う。
- 「ドラッグ起点がグリッド上か」をフラグ（`drag_selecting`。行/列ヘッダ起点は
  それぞれ `drag_rows` / `drag_cols`）で持ち、スクロールバー等のドラッグが
  選択に化けるのを防ぐ。ポインタが離れたらフラグを解除。

### 7.2b 手動ポインタ判定はポップアップを貫通する（押下は hovered() を使う）

§7.2 の対処（ポインタ座標×矩形の包含判定）を**押下（クリック開始）にまで
使ってはならない**。生座標の判定は egui のレイヤ（重なり）を無視するため、
表の上に浮いたコンテキストメニューやダイアログへのクリックが**真下のセルに
貫通**し、選択が意図せず単一セルに潰れる（実害: 複数行選択 → 右クリック →
「N 行を削除」を押した瞬間に選択が潰れ、1 行しか消えないバグ）。

使い分けの規約:

- **押下（クリック・ドラッグ開始）**: `Response.hovered()`（レイヤ考慮あり）
- **ドラッグ継続の追従のみ**: ポインタ座標×矩形（§7.2。ドラッグ中は
  ポップアップが開いていないため貫通の心配がない）

### 7.3 表外クリックの検出

テーブル描画全体を `ui.scope(...)` で囲み、その `response.rect`
（**スクロールバーを含む**テーブル外形矩形）を保持する。フレーム末尾で
`primary_pressed && ポインタが外形矩形の外 && 編集中でない &&
コンテキストメニュー非表示` なら選択解除とする。

**セル単位の押下フラグ方式にしないこと**: 最初のプロトタイプは
「セル・ヘッダ上の押下」を検出フラグで持つ方式だったが、スクロールバーの
つまみを掴んだだけで「表外」と判定され選択が消える欠陥があった。
「表外」の判定は領域（矩形）で行う。

### 7.4 編集モードの TextEdit は `ui.put` で固定

`ui.add(TextEdit)` は TextEdit 固有のマージンで数 px ずれ、モード切替時に
文字が動いて見える。**セル矩形へ `ui.put(rect, TextEdit…)`** で置き、
`.vertical_align(Center)`・`.margin(symmetric(4, 0))`・`.font(Body)` で
ラベル描画と幾何を一致させる。フォーカスは編集開始フレームで
`request_focus()`（1 回だけ）。確定・キャンセルは
`lost_focus` ＋ そのフレームの Enter / Tab / Esc 押下で分岐する。

**確定後は egui フォーカスを明示的に放棄すること**
（`memory_mut(|m| m.surrender_focus(...))`）。Tab 確定時に egui の
フォーカス移動が次のフォーカス可能ウィジェット（ボタン等）へ乗り、
直後の Enter がそのボタンを押す事故（例: 「解析実行」誤爆）を防ぐ。

### 7.5 共有罫線とスペーシング

egui_extras のテーブルセルはセル間スペーシング分だけ矩形が離れているため、
セル矩形をそのまま使うと罫線・塗りが途切れる。
`rect.expand2(item_spacing * 0.5)` した矩形に対して罫線・塗りを描き、
隣接セルと連続させる。

### 7.6 egui 0.34 の API 形

upstream の egui 0.34 では `eframe::App::update` ではなく
**`App::ui(&mut self, ui, frame)`**、パネルは `egui::Panel::top/bottom(...)` の
**`show_inside(ui, …)`** を使う（`TopBottomPanel::show` 系は deprecated）。
既存 `app/mod.rs` の実装形に合わせること。

### 7.7 その他

- セルの当たり判定は `ui.interact(rect, id, Sense::click_and_drag())`。
  id は `ui.id().with(("cell", row, col))` などで一意化する。
- 行数が増減したフレームでは `GridState.rows` を同期し `clamp_selection()` を呼ぶ。
- 行削除のコンテキストメニューは `Response::context_menu`。メニューが開いている
  フレームはクロージャ内でフラグを立て、表外クリック判定（§7.3）から除外する。
  メニューを閉じるのは `ui.close()`（`close_menu` は 0.34 で deprecated）。

## 9. テスト方針

### 9.1 grid_core 単体テスト（egui 非依存）

- 選択状態機械: click/drag/move（extend 有無）/select_row/col/all/deactivate と
  復帰・clamp（行数減少時）
- **空グリッド不変条件**: rows==0 で全選択系メソッドが no-op（active にならない）。
  空グリッドで rect_to_tsv 相当の経路に入らないこと
- select_row/col の extend が全列（全行）に広がること（Excel 準拠、§3.3）
- parse_tsv: CRLF・末尾改行・空セル
- tile_block: 整数倍（行のみ・列のみ・両方）/ 割り切れない / 1×1 / 空ブロック
- plan_paste: 正常 / 空セルスキップ / 列はみ出し拒否 / 検証失敗拒否（複数件の列挙）/
  はみ出し行数の算定 / サイズ上限超過の拒否
- rect_to_tsv: 単一セル / 矩形 / 行・列・全選択

### 9.2 ヘッドレス UI テスト（app/tests.rs）

- ペースト適用が複合コマンド 1 個になり、undo 1 回で行追加ごと戻る
- はみ出し行の自動追加（確認なしで行が増え、ログに行数が出る）
- 全体拒否時にモデルが一切変化しない
- Delete クリアの適用と undo
- **空モデル（節点 0）への貼り付け**がプレースホルダ経由で成立し、
  自動で N 行追加 → undo 1 回で空に戻る
- プレースホルダでの編集確定が行追加＋値設定の複合コマンド 1 個になる
- Backspace がアクティブセルの編集開始（空バッファ）であり範囲クリアしない
- 行削除が複合コマンド 1 個になり、undo 1 回で ID 繰り上げごと復元される
- 参照中の節点を含む行削除が全体拒否され、モデルが一切変化しない

## 10. 既知の制約と将来拡張

- **表外ドラッグの自動スクロール**は未実装（選択ドラッグを表の外へ出しても追従しない）。
- **IME（日本語入力）は未対応**: 文字キーでの編集開始は `Event::Text` 前提で、
  IME の変換合成（`Event::Ime`）を扱っていない。数値列のみの第1弾では実害が
  ないが、文字列列（部材名・断面名等）へ展開する際は、選択モード中の IME 起動
  （未確定文字での編集開始・変換の維持）を設計してから着手すること。
- **新規行プレースホルダによる追加フォームの置き換え**: プレースホルダが
  十分機能するなら、テーブル上部の追加フォームは冗長になる。運用を見て判断。
- **不正値ポリシーのアダプタ委譲**: 現在は全テーブル一律 all-or-nothing。
  先行事例（egui-data-table）は Abort/SkipCell/SkipRow をアダプタ選択制にしており、
  将来テーブルごとに方針を変えたくなった場合は `validate_cell` の戻り値を
  拡張する形で移行できる。
- **TSV エスケープ**: セル内タブ・改行を含む文字列列に対応する場合に再検討
  （Excel 互換を最優先するなら素の TSV が正）。
- フィルハンドル・Ctrl+X・行挿入ペースト（Excel の「コピーしたセルの挿入」相当）・
  読み取り専用**データ列**（算出値列）のサポート。

## 11. 決定記録（要約）

実装中に迷いやすい点の「なぜ」を残す。

| 論点 | 決定 | 主な理由 |
| --- | --- | --- |
| 既存クレート採用 | 自作 | undo 二重化がアーキテクチャ原則と衝突（§2） |
| 編集モデル | 2 モード式（Excel 方式） | TextEdit とのキー衝突が構造的に解消。描画コストも減 |
| アクティブセルの位置 | 選択の起点に固定 | Excel と同じ。拡張中に枠が動くのは誤解を招く |
| 行ヘッダ列 | グリッド座標系の外 | 「選択できるのに編集不可」の煩わしさを構造から排除 |
| 不正値の扱い | all-or-nothing | 予測可能性優先。救済は undo 1 回 |
| 空セル | 変更なしスキップ | 空白混じりの Excel 範囲を貼れるように。削除は Delete に分離 |
| はみ出し行 | アダプタのオプトインで**自動追加**（確認なし） | 当初は確認ダイアログとしたが、プロトタイプ試用で「主要フロー（一括投入）に毎回割り込む」と判明し撤回。undo 1 回・選択表示・ログ・サイズ上限が安全網で、Google Sheets とも同挙動 |
| 割り切れない選択への貼り付け | 起点から 1 回貼る | Excel の単一矩形選択と同じ |
| 選択解除 | 表外クリック・Esc で完全解除 | テーブルが他パネルと同居するため（Excel からの意図的差分） |
| 「表外」の定義 | テーブル領域（スクロールバー含む）の外 | セル単位判定だとスクロールバー操作で選択が消える |
| 見た目 | スプレッドシート様式（共有罫線・白地） | 個別枠は二重線で騒がしく、可能な操作が伝わらない |
| 行高 | `table_row_height` から導出 | TONMANUAL §4（固定 px 禁止） |
| 空テーブルへの投入 | 新規行プレースホルダ（Airtable 型） | 「何もない所へ貼る」違和感を構造から解消し、空表でも貼り付け起点が存在する。手入力の行追加も自然になる |
| Backspace | Delete と分離（アクティブセルを空で編集開始） | Excel 準拠。範囲クリアの誤爆も減る |
| ヘッダの Shift+クリック | Excel 準拠（全行 / 全列に拡張） | ドラッグでは全行になるのに Shift+クリックだけ矩形なのは不整合 |
| ペーストのサイズ上限 | 100,000 セルで全体拒否 | Excel の列全体コピー誤ペーストによるフリーズ防止。実務規模には影響しない |
| コピーの文字列 | 内部値の正準文字列（表示の丸めを使わない） | 丸め表示列を将来作った際の精度低下を予防 |
| 失敗フィードバック | 対象範囲の赤フラッシュ＋ログ赤字（§6.4） | 全体拒否がログだけでは見逃され、貼り付け先の選択ミスに気づけない |
| サイズ不一致の貼り付け | Excel 互換の成功＋貼り付け範囲を黄フラッシュ | 動作は変えず選択ミスに気づかせる。赤は「未適用」専用のため注意色で区別 |
| 選択なしのペースト | 無言で無視 | 貼り付け先が定まっていない操作に通知はノイズ |
| 行削除の操作 | 右クリック（ヘッダ・セル）＋ Ctrl+Delete。Delete とは分離 | Excel の Ctrl+マイナスは egui の UI ズームと競合（Alt 併用でも発火）。ズーム無効化はアプリ全体の決定のため表の都合では行わず、ズームに触れない Ctrl+Delete を採用。Delete の意味が状態で変わると誤削除を招く |
| 行削除の対象 | 選択が跨ぐ実データ行すべて | Google Sheets と同じ。削除は明示操作に限るので対象は広くても安全 |
| 参照中の行を含む削除 | 全体拒否（赤フラッシュ＋参照元をログ） | ペーストと同じ all-or-nothing。「3行のつもりが2行だけ消えた」を作らない |
