# 床（スラブ）の設計と実装計画

現状、床の分配エンジン（`squid-n-load::floor`）・入力UI（`app/tables/slabs.rs`）・
解析接続（`refresh_beam_loads`／`sync_slab_loads_action`）は実装済みだが、
以下の本質的ギャップがある。本ドキュメントはその設計と段階実装をまとめる。

## 現状のギャップ

1. **DL/LL が区別されない**: `AreaLoad{kind:String, value}` は自由文字列で、床荷重は
   すべて kind=Dead の 1 ケース「床荷重(自動)」へ合算される。積載荷重（Live/LiveSeismic）
   として分離されず、①令85条1項の 3 欄（床用/骨組用/地震用）の使い分け、②荷重組合せでの
   DL/LL 係数分け、③令85条2項の柱積載低減（API はあるが未配線）ができない。
2. **用途別積載荷重テーブルが無い**: 令別表第1（住宅・事務室・教室…）のプリセットが無い。
3. **小梁の入力手段が無い**: エンジン（二段階伝達）とモデル（`JoistLine`）はあるが、UI は
   常に `joists: Vec::new()`。小梁は仮想ラインで実部材でないため断面検定・たわみ検定不可。
4. **入力が境界4節点固定**: モデル/エンジンは多角形対応なのに UI から作れない。スラブ自重の
   自動算定も無い。

## 用語・単位

- 内部単位系は N-mm-s（面荷重は N/mm²）。令別表第1 は N/m² で規定 → ×1e-6 で N/mm² に変換。
- **床用（floor）**: 床スラブ・小梁の設計用。**骨組用（frame）**: 大梁・柱・基礎の設計用。
  **地震用（seismic）**: 地震力（地震用重量）算定用。（令85条1項・令別表第1）

## 用途別積載荷重テーブル（積載荷重 N/m²。floor / frame / seismic）

## 設計

### モデル（squid-n-core）

- `LoadPurpose { Floor, Frame, Seismic }` を追加。
- `SlabUsage`（上表の列挙 ＋ `Custom{floor,frame,seismic}`）を追加。
  `SlabUsage::live_load(purpose) -> f64`（N/mm²）を提供。
- `Slab` に `#[serde(default)] usage: Option<SlabUsage>` を追加（後方互換）。
- ヘルパ:
  - `Slab::dead_intensity()`: 既存 `loads` の合算（仕上げ等の固定荷重。DL）。
  - `Slab::live_intensity(purpose)`: `usage` から積載を返す（None は 0）。
  - `Slab::intensity(purpose)`: 長期・地震用の合成（DL + LL(purpose)）。
- 既存 `loads: Vec<AreaLoad>` は **DL（固定荷重）** として扱い後方互換とする。

### 荷重ケース同期（app）

現状 1 本（「床荷重(自動)」kind=Dead、DL のみ）を、用途に応じ分離する:
- 「床荷重(自動)」 kind=Dead: DL を分配。
- 「床積載(自動)」 kind=Live: LL(骨組用) を分配（長期骨組解析用）。
- 地震用重量集計は DL + LL(地震用) を用いる（`story_gen` へ接続）。

### 令85条2項（柱積載低減）

`live_load.rs` の `column_live_load_reduction`／`floors_supported_by_column` を、
`LoadCfg.live_load_reduction` が有効な場合に長期軸力算定へ配線する（Phase A-4）。
低減率の算定（`column_live_load_factors`）は実装済み・参考表示済み。設計軸力への
適用は per-case 軸力分解が必要なため Phase A-4 で行う。

### 入力UI（app）

- スラブ追加/一覧に **用途（SlabUsage）** の選択を追加。積載値は用途から自動表示（Custom は手入力）。
- 境界を可変長（3〜N 節点）に拡張（エンジンは対応済み）。
- （将来）3D ビューでの節点クリック指定、小梁入力、スラブ自重自動算定。

  - スラブ一覧・追加フォームに用途（`SlabUsage`）選択を追加（`SetSlabUsage`／
    `AddSlab.usage`）。積載値プレビュー付き。
- **A-3（完了）**: 地震用重量への LL(地震用) 接続。
  - `sync_slab_loads_action` が「床地震用積載(自動)」(LiveSeismic) も同期

  影響範囲が広く独立性が高いため別フェーズとする。

- **F（実装済み: F-1〜F-4）**: 交差小梁（十字梁）と二段階サブストラクチャ解法。
  実装は `squid-n-app/src/floor_grillage.rs`（`build_slab_grillage`/`solve_grillage`/
  `joist_design_forces`/`compute_reactions`）と `app/actions.rs::floor_design_checks`。
  - **F-1**: 床格子サブモデルの求解＋反力抽出（`reaction = K·u − F_ext`）。
  - **F-2**: 小梁からの格子構築（内部交点検出・座標一致による節点共有・交点分割）。
  - **F-3**: 交差小梁を格子 FEM 部材力で設計（M・Q・たわみ）。非交差は既存の単純梁へ。
  - **F-4**: ①ピン受け/架けと②剛接十字の区別。`JoistLine.pinned_onto: Option<usize>`
    （同一スラブ内の受け梁インデックス）で指定。UI（小梁一覧の「剛接十字／受け:小梁c」
    セレクタ）から `SetSlabJoists` で反映。
    - **②剛接十字**: 交点を**共有節点**とし二方向曲げを連続（交差材のねじり剛性 GJ で
      たわみを抑制）。
    - **①ピン受け/架け**: 架け梁側に受け梁と**座標一致する別節点**を設け、受け梁節点と
      **鉛直 Uz のみ**を `Constraint::RigidLink` で結合（曲げは伝えず鉛直せん断のみ）。
      架け梁は自身は連続（機構を生じない）。
    - **支点は向き依存**: 小梁は大梁へ両端ピン（主曲げ回転のみ解放）で取り付き、ねじり
      （自軸回転）は接合部で拘束（＝支点で固定）。これによりねじり剛体回転〔機構〕を
      防ぎ、かつ剛接十字のねじり連続（たわみ抑制）を成立させる。
  - **F-3b（実装済み・一部申し送り）**: 抽出した大梁接続点反力を本体の「床荷重(自動)」
    /「床積載(自動)」/「床地震用積載(自動)」へ大梁接続点の集中荷重として反映する
    （`app/actions.rs::slab_grillage_node_reactions`＋`slab_beam_loads`）。交差小梁スラブ
    （軸平行・全仮想）では、平行小梁モデルの小梁点反力（`LoadTarget::Node`）を格子の
    支点反力で**置換**する。格子の各小梁は平行モデルと同じ `w·spacing` を負担するため
    支点反力総和は `w·Σ spacing·L` と厳密一致し**総和保存**、交点での荷重分担のみ高精度化。
    実部材化小梁を含むスラブは本体 FEM と二重計上になるため対象外。**格子置換は
    `distribute_slab_w` が小梁二段階伝達を採るスラブ（内側・辺支持なし・矩形・分配法が
    三角/一方向）に厳密に限定する**（`squid_n_load::floor::uses_joist_distribution`。隅・
    片持ち・辺支持・非矩形・面積分配のスラブは小梁が使われず全面積を境界へ分配するため、
    格子反力を上乗せすると二重計上／隅集中荷重の取りこぼしになる。敵対的レビュー指摘）。
    - **申し送り**: 境界大梁の残り負担（`remainder = width_total − Σspacing`）は既存の
      平行モデルの式のままで、交差小梁の二方向トリビュタリ配分は未対応（F-3b では
      小梁点反力の分担のみ高精度化し、境界配分の総和は従来と不変）。二方向スラブ→
      格子の面荷重トリビュタリ厳密化は今後の課題。
  - 以下は当初の設計メモ（確定事項）。

- **F（当初設計メモ）**: 交差小梁（十字梁）と二段階サブストラクチャ解法。
  - 動機: 十字状に交差する小梁を正しく扱う。現状は 1 枚の床の小梁を「すべて平行」と
    仮定しており（`distribute_rect_with_joists` が `joists[0].dir` を全小梁の共通軸として
    使い、`spacing` を平行ストリップとして合算）、交差小梁は誤分配・独立単純梁設計になる。
  - **接合タイプは 2 種を区別する**（結果が変わる）:
    - **①ピン接合の受け／架け（一方向）**: 短い小梁が長い小梁（受け梁）にピンで載る。
      受け／架けは**かけ方（フレーミング）で決まる**（スパン長では決めない。小梁は両端
      ピンが基本）。架け梁反力 → 受け梁へ鉛直点荷重のみ。たわみは大きめ。
    - **②剛接の十字梁（二方向・格子）**: たわみを抑えるため交点で曲げ連続。両梁が二方向で
      荷重を分担し、ねじりも生じる。たわみは小さい。独立単純梁では表せない。
  - **解法（二段階サブストラクチャ）**: 本体架構と同時に解くと大梁が受け取る床荷重が解に
    埋もれて分離できないため、**床を独立した小さな FEM（格子）として先に解く**:
    1. 床サブモデルを組む: 小梁要素＋交点節点＋大梁接続点（**鉛直剛支持**＝大梁は沈まない
       床/大梁分離の標準仮定）。交点ジョイントを①ピン=端部解放／②剛接=曲げ連続で切替。
    2. 床サブモデルだけを既存の線形静的ソルバ（`squid-n-solver`）で解く。床荷重は既存の
       分配で小梁 UDL に。
    3. 大梁接続点の**支点反力＝大梁へ渡す荷重（CMQ）**を抽出（荷重伝達が明示的で透明）。
    4. その反力を大梁の集中荷重として既存の「床荷重(自動)」経路で本体へ反映。
    5. 小梁・床設計は床サブモデルの**部材力（M・Q・たわみ）**で行う。
  - **本体架構は一切分割しない**（受け取るのは反力のみ）。分割するのは小梁どうし（②剛接で
    交点に中間節点を作り両小梁を接続）。①②は同じ格子サブモデルでジョイントの解放/連続を
    切り替えるだけで統一できる。
  - 実装段階（予定）: (1) モデル/入力（交点・受け梁参照・接合タイプ、UI 指定＋交差の幾何
    検出）、(2) 床サブモデル構築＋求解＋反力抽出→大梁 CMQ、(3) 小梁・床設計への接続。
  - 前提: 大梁接続点は鉛直剛支持と仮定（非連成）。大梁のたわみを床へ反映する連成は Phase G。
