# Tunny Dashboard トンマナガイド (TONMANUAL)

Tunny Dashboard のトーン＆マナー（デザイン／ライティング規約）を定義する。本書は **他ライブラリ・関連プロダクトと UI／配色／文言の一貫性を揃えるための共有仕様** であり、特定の実装に依存しない値・原則のみを記載する。

- プロダクト性格: 最適化結果（多目的最適化・感度分析・意思決定）を分析するためのデスクトップ分析ツール
- 設計原則: **明るく・クリーンで・データを主役にする**。UI クロームは控えめなブルー基調、コンテンツ（チャート・テーブル）は白背景で最大限のコントラストを確保する。

---

## §1. デザイン原則

1. **データファースト** — UI 要素はデータの邪魔をしない。キャンバス／チャートは白背景 (`#FFFFFF`)、装飾は最小限。
2. **静かなクローム** — ツールバーやパネルは淡いブルー／グレーで沈ませ、ユーザーの注意をコンテンツへ向ける。
3. **一貫したブルー** — ブランドのアクセントは blue-500 (`#3B82F6`)。インタラクティブ要素・選択・リンクはこのブルー系で統一する。
4. **意味のある色** — チャートの色は装飾ではなく状態・カテゴリを表す（パレート＝赤、通常＝青 など）。
5. **ライトテーマ基準** — ライトテーマを正とする。

---

## §2. カラーパレット

ベースは Tailwind 系のブルー／グレースケール。

### プライマリ（ブランドブルー）

| 役割 | HEX | 用途 |
| --- | --- | --- |
| blue-200 | `#BFDBFE` | ナビゲーション／ツールバー背景 |
| blue-300 | `#93C5FD` | 選択ハイライト、ボタンホバー、ヘッダー |
| blue-400 | `#60A5FA` | アナウンスバー背景 |
| **blue-500** | **`#3B82F6`** | **メインアクセント。アクティブボタン・枠線・インジケーター** |
| blue-600 | `#2563EB` | アクセントのホバー濃色 |

### セカンダリ（グレースケール）

| 役割 | HEX | 用途 |
| --- | --- | --- |
| gray-900 | `#111827` | 見出し・主要テキスト |
| gray-700 | `#374151` | ナビゲーションテキスト・サブテキスト |
| gray-600 | `#4B5563` | 本文テキスト |
| gray-300 | `#D1D5DB` | キャンバスのドットグリッド |
| gray-200 | `#E5E7EB` | ボーダー・入力欄枠線・ホバー背景・テーブルストライプ |
| gray-100 | `#F3F4F6` | パネル背景・ウィジェット背景・入力欄背景 |

### アクション／セマンティック

| 役割 | HEX | 用途 |
| --- | --- | --- |
| green-500 | `#22C55E` | 重要操作（実行・確定）アクション |
| green-600 | `#16A34A` | アクションのホバー |
| error red | `#EA4335` | エラー表示（ブランド対象外の固定色） |

### 配色の原則

- **背景の階層**: 白（コンテンツ） < gray-100（パネル／ツール領域） < blue-200（ナビゲーション）。手前のクロームほど青く、奥のコンテンツほど白い。
- **テキスト on ブルー背景**: ホバー／アクティブ時は白。通常時は gray-700。
- **コントラスト確保**: 選択ハイライトは背景 blue-200 より一段濃い blue-300 を使い、状態を視認可能にする（同系色の潰れを避ける）。

## §3. データビジュアライゼーション配色

チャート用の色は装飾ではなく **意味（状態・カテゴリ）** を担う。Google マテリアル系の 4 色を基調とする。

| カテゴリ | 色 | HEX | 意味 |
| --- | --- | --- | --- |
| データ点・通常 trial | 青 | `#4285F4` | デフォルトの試行点・バー・線・リンク |
| パレート解・重要・負方向 | 赤 | `#EA4335` | パレートフロント、高スコア、負のバー、pruned trial |
| 強調・ベスト・中間 | 黄 | `#FBBC04` | ベスト値、中スコア、予測フロント |
| 良好・収束・実行中 | 緑 | `#34A853` | 収束線、低スコア、running trial |
| ハイライト試行点 | 紫 | `#7C4DFF` | 選択中の試行点・等高線 |

### グレーアウト規約

- 選択／フィルタ外の点・線は **色相を捨てて半透明グレー** にする。選択中の点を引き立て、元のカテゴリ色と混同させない。
- 実行不可能解（制約違反）も同様にグレーで沈める。

### カラーマップ（連続値）

- 標準セット: Viridis / Plasma / Jet / Turbo / Inferno / Coolwarm / Spectral / Cividis / BlueYellow。
- **デフォルトは Viridis**（知覚均等で色覚多様性に配慮）。

## §3-2. 3D ビュー規約

3D 散布図（多目的パレート 3D / クラスタリング 3D / 意思決定 3D）は配色・カメラ・操作を全ビューで統一する。

### 背景・グリッド・軸の配色

| 要素 | HEX / 値 | 備考 |
| --- | --- | --- |
| 3D 背景 | `#F0F2F5` | **2D チャートの白とは異なり淡いグレー**。立体感と奥行きを出すため意図的に白を避ける |
| グリッド線 | ダーク半透明・線幅 0.5 | XY/XZ/YZ の 3 面、各軸 4 分割 |
| X 軸 | `#D26464`（赤系・線幅 1.5） | |
| Y 軸 | `#50AA50`（緑系・線幅 1.5） | |
| Z 軸 | `#6464C8`（青系・線幅 1.5） | |

> 3D 軸色は彩度を落とした専用色で、§3 のチャート 4 色（青/赤/黄/緑）とは別系統。**赤=X / 緑=Y / 青=Z** の対応を全 3D ビューで固定する。

### 軸ラベル

- 正方向の端: `軸名 (最大値)` を 11px、軸色で表示。
- 負方向の端: `最小値` を 10px、軸色の淡色（約 70% 明度）で表示。
- データ範囲 `[min, max]` は常にクリップ空間 `[-1, 1]` に正規化してから描画する。範囲ゼロの軸は中央 0 に潰す。

### カメラ

| 項目 | 既定値 | 範囲 |
| --- | --- | --- |
| 回転 | 正面 | — |
| ズーム | 3.0 | 0.5 – 10.0 |
| パン | (0, 0) | — |

- **アークボール（クォータニオン）回転**を採用し、ジンバルロックを避ける。
- 投影は正射影ベース。

### インタラクション規約

| 操作 | 動作 | パラメータ |
| --- | --- | --- |
| 左ドラッグ | 回転 | 感度 0.005 /px |
| スクロール | ズーム | 係数 0.01、0.5–10.0 にクランプ |

全 3D ビューでこの操作割り当てを統一する（マウス操作の意味をビュー間で変えない）。

### 深度・重なり

- サーフェスや軸を面と正しく前後表示する場合は、軸を細分化して点・面と同じ深度ソートに混ぜる。

## §4. タイポグラフィ

- **欧文**: Inter（Variable Font）— 第一フォント
- **和文**: Noto Sans JP（Variable Font）— 日本語グリフのフォールバック
- **等幅**: Inter をフォールバックに（数値・コード表示）

フォントは同梱し、環境非依存で同一の見た目を保証する。

### サイズスケール

| 用途 | サイズ(px) |
| --- | --- |
| 見出し | heading（既定見出しサイズ） |
| ツールバーボタン | 13 |
| ツールバーラベル／コンボボックス | 12 |
| セル内・補助ラベル | 11 |
| 密集アイコンタイル | 9 |

新規 UI は **9 / 11 / 12 / 13 + 見出し** のスケールに収め、独自サイズの乱立を避ける。

## §5. レイアウト & スペーシング

### 構造

```
┌─────────────────────────────────────────────┐
│ Toolbar (上部固定, blue-200, 高さ ≥32)        │  2段: 操作行 + カラーマップ/言語行
├──────┬───────────────────────────────┬───────┤
│ Left │   Central Canvas (white)      │ Right │  左右はホバーで開くオーバーレイ
│ panel│   Grid / Canvas view          │ panel │
│(gray)│                               │(gray) │
└──────┴───────────────────────────────┴───────┘
```

### 寸法・余白

| 項目 | 値 |
| --- | --- |
| ツールバー最小高 | 32px / 内側余白 横8・縦4 |
| 左パネル幅 | 120–600px |
| 右パネル幅 | 100–400px |
| パネル内側余白 | 8px |
| ホバー開閉トリガ | 端から 20px 以内で開く |
| エッジインジケーター | 幅 18 × 高 60、角丸 4 |
| パネル開閉アニメ | 0.20 秒 |

### スペーシング指針

- 標準の項目間隔を基本とし、密集領域のみ明示的に縮める（アイコンタイル間 4×3、リスト行間 1 など）。
- パネルはスライドイン＋影でフロート。中央キャンバスは常にフル幅で、パネルは上に重なる（コンテンツ幅を削らない）。

## §6. コンポーネント規約

### 角丸（Corner Radius）

| 要素 | 半径 |
| --- | --- |
| ボタン背景 | 4px |
| エッジインジケーター | 4px |
| カード／アイコンタイル | 6px |

→ **小要素=4px、カード=6px** を基準にする。

### 影（Elevation）

オーバーレイパネルなどフローティング UI のみ影を持つ:

```
offset: (±4, 0)   blur: 12   spread: 0   color: 黒 @ alpha 50
```

中央コンテンツやセルはフラット（影なし）。浮遊する一時 UI（パネル・モーダル）だけに影を与える。

### ボタン

- ツールバーボタン: 透明背景、パディング 横10・縦5、テキスト gray-700。
  - ホバー: 背景 blue-300、テキスト白。
  - 無効: テキストを淡色化（約 40%）、透明背景。
- アクティブ状態: 背景 blue-500、テキスト白。

### 入力欄・コンボボックス

- 背景 gray-100、枠線 gray-200、テキスト gray-700。

### テーブル

- 行ストライプは gray-200（白背景上で淡すぎる色は避け、視認できる濃さにする）。

### パネル背景

- 左右パネル・セルツールバー = gray-100。チャートセル本体 = 白。

---

## §7. アイコン & インジケーター

- システム絵文字を機能アイコンとして使用（例: `📄 Licenses`、`🌐 Help Language`）。意味が直感的な絵文字に限り、装飾目的では使わない。
- パネル開閉の方向は `›` / `‹`（16px・白）で示す。
- ウィジェットピッカーは SVG アイコンタイル（6px 角丸、9px ラベル）。アイコンで種別を一目で示す。
- ホバー近接フィードバック: 端に近いほどインジケーターが明るく補間される。

---

## §8. モーション

- **控えめ・短い**: パネル開閉は 0.20 秒。アニメーション中のみ再描画し、定常状態では無駄に再描画しない。
- スライドイン（位置補間）＋フェード（アルファ補間）を組み合わせる。
- 画像／PNG キャプチャ中は装飾インジケーターを描画しない（成果物への写り込み防止）。

---

## §9. ボイス & トーン（ライティング）

### 言語ポリシー

- **アプリ UI ラベルは英語**（例: "Open", "Grid View", "Target Study:"）。
- ヘルプ／理論ドキュメントは英語・日本語の両方を用意する。

### 文体

- UI ラベルは **簡潔・名詞/動詞句**。冠詞や冗長語を避ける（"Open"、"Generate HTML Report"）。
- ヘルプ文書は丁寧かつ実務的。専門用語（最適化・感度分析・意思決定分析）は読者を実務者と想定し、過度に噛み砕かない。
- エラーは事実ベースで簡潔に。原因と次の操作が分かる表現を優先する。

## §10. 統一チェックリスト

新しい UI／チャートを追加・移植するときは以下を確認する。

- [ ] 色は本書のパレット（§2・§3）の値を使ったか（独自色を散らさない）
- [ ] 背景階層（白＜gray-100＜blue-200）に沿っているか
- [ ] フォントサイズはスケール（9/11/12/13/見出し）内か
- [ ] 角丸は 4px（小要素）/ 6px（カード）か
- [ ] 影はフローティング UI のみか（コンテンツはフラット）
- [ ] チャート色が状態・カテゴリの意味と一致しているか／選択外はグレーアウトか
- [ ] 3D ビューは背景グレー・赤=X/緑=Y/青=Z・アークボール操作に揃っているか
- [ ] UI ラベルは英語・簡潔か
- [ ] 連続値カラーマップの既定は Viridis か
