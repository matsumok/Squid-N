## 1. 修正済み（高確度・再現テスト付き）

### 1.1 プッシュオーバー: ステップ変位が「最後の Newton 修正量」しか累積されない（非安全側）

- **ファイル:** `crates/squid-n-solver/src/nonlinear/pushover/driver.rs`
  （荷重制御フェーズ・変位制御フェーズ）
- **不具合:** 各荷重ステップの Newton 反復で `last_du_free = du_free`（上書き）とし、
  収束時に `total_disp += last_du_free` としていた。要素内部状態は全反復の修正量を
  累積するのに対し、`total_disp`（荷重−変位曲線の変位軸・層間変位・段階耐力喪失の
  部材変形角の基）は**最後の 1 反復分しか受け取らない**。塑性域（1 ステップに複数
  反復を要する）で途中の修正量が脱落し、変位軸が**過小評価**される。`base_shear` は
  内力から求めるため正しく、力軸だけが正・変位軸だけが過小という気付きにくい形。
  段階的耐力喪失判定が遅れる（**非安全側**）。弾性ステップは 1 反復で収束するため
  既存テストは検知できなかった。
- **検証:** 荷重制御終点（λ=1）の頂部変位は物理的にステップ数非依存のはず。
  修正前は 20/40/80 ステップで 80.24/82.35/83.40mm とステップ依存（下方から漸近）、
  修正後は全ステップで 84.4516mm と一致。回帰テスト
  `test_pushover_load_control_endpoint_is_mesh_independent` を追加。
- **修正:** ステップ内の全 `du_free` を累積する `step_du_free` を導入。弧長法フェーズは
  既に累積量（`step_result.du`）を用いており不変。

### 1.2 剛域変換の回転連成 4 項の符号が逆（剛体運動不変性の破れ）

- **ファイル:** `crates/squid-n-element/src/frame/beam/stiffness.rs`
  （`apply_rigid_zone_transform`）
- **不具合:** 剛域腕の運動学は `u_flex = u_node + θ×r`（i端 r=+li·ex, j端 r=−lj·ex）で、
  `uy_i'=uy_i+li·rz_i, uz_i'=uz_i−li·ry_i, uy_j'=uy_j−lj·rz_j, uz_j'=uz_j+lj·ry_j`。
  実装は 4 項すべて符号が逆（`u'=u−θ×r`）だった。剛体回転に対し可撓部へ偽の
  ひずみが入り、材端モーメント・せん断が生じる。`geometric_stiffness`（P-Δ）と
  集中ヒンジ（`concentrated`）も同ルーチンを共有するため影響。既存の剛域テストは
  対角・ねじり項のみ検査しており検知できなかった（opt-in・剛域有効時のみ発現）。
- **検証:** 局所 z（および y）軸まわりの剛体回転 `u_rigid` に対し `K·u_rigid≈0` を検査。
  修正前は代表剛性 1.75e5 に対し偽の材端力 1.89e11、修正後は機械精度。回帰テスト
  `test_rigid_zone_preserves_rigid_body_rotation` を追加。
- **修正:** 4 項の符号を反転。ドキュメントコメントも運動学に合わせて訂正。

### 1.3 MITC4 横せん断 B 行列が逆ヤコビアンの転置を適用（歪んだ四辺形で誤り）

- **ファイル:** `crates/squid-n-element/src/shell/bmatrix.rs`（`shear_b_mitc4`）
- **不具合:** タイイング点での前方射影は `e_cov = J·γ`（`jacobian`、行が共変基底。
  コード内で確認）なので、逆変換は `γ = J⁻¹·e_cov`。しかし最終射影で
  `jit`(=J⁻ᵀ) を**行アクセス**（＝`J⁻ᵀ·e_cov`）していた。正しくは**列アクセス**
  （＝`J⁻¹·e_cov`、`dshape_cart` のデカルト微分と同一規約）。ヤコビアンが非対称な
  歪んだ四辺形でのみ顕在化（矩形では `jit` が対角のため一致）し、`local_stiffness` の
  せん断寄与と `recover_resultants` の qx/qy が誤る。既存の膜・曲げパッチテストは
  せん断を検査せず、板収束テストは矩形メッシュのため検知できなかった。
- **検証:** 平行四辺形（`jac=[[50,0],[25,40]]`、非対称）で一定横せん断場
  （w=0, θx=0, θy=a → γ_xz=a, γ_yz=0）を課すと、修正前は
  γ_xz=0.0006875（正 0.001）, γ_yz=0.000625（正 0）とパッチテスト破れ、修正後は厳密再現。
  回帰テスト `test_mitc4_constant_shear_patch_skewed` を追加。矩形メッシュの
  板収束テスト（`test_ss_plate_convergence`・`test_clamped_plate_convergence`）は不変。
- **修正:** 最終射影を列アクセス（`J⁻¹·e_cov`）へ。

## 2. 第2ラウンドの追加修正・据え置き項目

各項の見出しに現状（【修正済み】／【一部修正】／【据え置き】）を付す。据え置き項目は、
修正を急ぐとかえって剛性式・履歴則・規準値表を壊すリスクが高いため、再現の道筋を添えて
記録に留める。修正済み項目のコミット単位・回帰テストは本レポート冒頭の結論および各
コミットメッセージを参照。優先度順。

### 2.1 【据え置き・要再定式化】FiberBeam/MultiSpring がせん断ばねを曲げと**並列**に加算（横剛性が約 1000 倍過大）

- **ファイル:** `crates/squid-n-element/src/frame/fiber/mod.rs`
  （`tangent_stiffness`/`internal_force` でのせん断ばね加算）、
  `crates/squid-n-element/src/springs/shear_spring.rs`。
- **内容:** ファイバ断面は 3 次エルミート（Euler-Bernoulli）B 行列で並進含む曲げ剛性
  （3EI/L³）を既に持つ。そこへ `k=G·As/L` を同じ並進自由度(1-7, 2-8)間に**並列**で
  加算している。せん断可撓は**直列**（軟化）で合成すべきで、並列加算は逆に剛化する。
  弾性片持ち柱で正しい先端横剛性 ≈3.80e2 N/mm に対し測定値 ≈4.38e5 N/mm（約 1150 倍）。
  `as_y=0` の既存テストではばねが消えるため未検知。カタログ断面は `as_y` を持つため
  実モデルで発現しうる。
- **なぜ未修正か:** 正しい直列合成は内部自由度の静縮約か柔性ベースの再定式化を要し、
  非線形分布塑性要素での接線整合を含めて慎重な検証が必要。設計意図（§6.3/R26 の
  「独立せん断ばね」）を踏まえた実装は保守側で行うのが安全。弾性 `BeamElement` は
  Timoshenko の (1+φ) で正しく扱えており、ファイバ経路のみの問題。
- **推奨:** 弾性 Timoshenko 梁の 3EI/L³/(1+φ) と一致することを合否基準に、直列化
  （または (1+φ) 相当の柔性加算）で再実装。

### 2.2 【据え置き・要設計判断】集中ヒンジ `internal_force` が `K_tangent·u` で復元（降伏後にモーメント過小）

- **ファイル:** `crates/squid-n-element/src/frame/concentrated/mod.rs`。
- **内容:** 非線形回転ばねの抵抗力を、現在の**接線**ばね係数で組んだ縮約行列と
  総変位の積で復元している。降伏後に接線→~0 となると縮約でヒンジ拘束が落ち、材端
  モーメントがピン値（≈0）へ向かい、真の頭打ちモーメント（≈My）を下回る（非安全側）。
- **なぜ未修正か:** ソルバがこの `internal_force` を非線形釣合いにどう用いるか
  （別経路の応力復元があるか）の契約確認が必要。medium 確度。

### 2.3 【修正済み】拘束縮約が連鎖拘束を合成しない（スレーブ兼マスターで拘束無効化）

- **ファイル:** `crates/squid-n-solver/src/common/constraint.rs`（独立判定・再割当）。
- **内容:** ある自由度が一方の拘束のスレーブかつ他方のマスターである場合（剛床スレーブが
  剛リンクのマスター等）、被参照の従属自由度に新規独立番号を与えてしまい、連鎖拘束が
  無言で破れ余分な自由度が残る。medium-high（論理）／連鎖生成頻度は medium。

### 2.4 【修正済み】集中質量スティックモデルの減衰が接線剛性比例（ドキュメントは初期剛性比例）

- **ファイル:** `crates/squid-n-solver/src/dynamic/lumped_mass/time_history.rs`。
- **内容:** docstring は初期剛性比例減衰 `C=(2h/ω1)·K_init` と明記し `a1` を初期振動数で
  一度だけ算定するが、減衰力・有効剛性を**接線** `kt` で組むため実質 `C=a1·K_tangent`。
  降伏後に減衰が過小となり非弾性応答を過大評価。`damping.rs` の `StiffnessKind` が
  Initial/Tangent を区別しており、ここは Initial 意図と読める。要・意図確認。

### 2.5 【修正済み】HHT-α が α によらず β=0.25, γ=0.5 固定

- **ファイル:** `crates/squid-n-solver/src/dynamic/timehistory/hht.rs`。
- **内容:** 真の HHT-α は `γ=1/2−α, β=(1−α)²/4`（α∈[−1/3,0]、K・C は (1+α) 形で整合済み）。
  現状は α≠0 でも Newmark 平均加速度パラメータ固定のため、2 次精度・意図した高振動数
  数値減衰が得られない。ただしコメントに「固定」と明記され設計判断の可能性があるため、
  意図確認の上で `γ=0.5−alpha; β=(1.0−alpha).powi(2)/4.0` とするのが標準。α=0 は現状と一致。

### 2.6 【一部修正】材料履歴則（スリップのピンチ反転・TakedaDegrading の劣化カウント＝修正済み／Masing 微小再載荷＝据え置き）

- **ファイル:** `crates/squid-n-material/src/hysteresis/material.rs`。
- **内容:**
  - **スリップ再載荷** `reload_line`：`pinch_m = target.1`（全復元力）としているため
    原点付近が急・第2区間が水平となり、「原点付近剛性を slip_factor 倍に低下」という
    ドキュメント意図と**逆**（ピンチが効かない）。
  - **TakedaDegrading**：スケルトン上で**毎ステップ**ピークカウントを増分するため、
    単調載荷を細かく刻むほど劣化が進むステップ数依存（非物理）。劣化は反転/サイクル
    事象に紐付けるべき。
  - **Masing 微小再載荷**：再結合閾値が最大経験点でなく現反転点 `tr` 基準のため、
    ピークからの部分除荷→同方向再載荷でスケルトンへ不連続にスナップし得る。
- **なぜ未修正か:** いずれも「正しい挙動」が原典モデルの規定に依存し、insta スナップショット
  更新を伴う。正しいパラメトリック定義を確定してから修正すべき。

### 2.7 【据え置き・要原典照合】RC 造 Ds 表 `RcWall` D ランク値の整合性（保守側）

- **ファイル:** `crates/squid-n-design-jp/src/secondary/holding_capacity.rs`（`ds_value`）。
- **内容:** `RcWall` 行 FA/FB/FC=0.35/0.40/0.45 は告示 1792 の 0.3<βu≤0.7 列に一致するが、
  FD=0.55 のみ βu>0.7 列の値で行内不整合（同列の D ランクは 0.50）。Ds 過大＝保有耐力
  要求 Qun 過大の**保守側**だが告示値と不一致。ただし Frame/Wall の 2 値 enum を βu 3 列へ
  写す簡略化のため、`RcWall` が中列か上列かは設計判断。原典照合の上で確定が必要
  （`member_rank.rs` の RankCriteria 既定値も要・原典照合の placeholder）。

### 2.8 【一部修正】ST-Bridge 取り込み・編集の堅牢性（重複 id・undo 順序＝修正済み／梁往復並べ替え・壁孤立断面＝据え置き）

- **ファイル:** `crates/squid-n-io/src/stbridge/{import/mod.rs,export.rs}`、
  `crates/squid-n-edit/src/node_member.rs`。
- **内容:**
  - **重複ファイル id**（例 `StbNode id="0"` が 2 つ）で `build_index` は重複排除する一方、
    `raw_nodes` 等はそのまま push するため `id != index` となり、参照解決が別実体を指す
    （無言のジオメトリ破損）。id 一意性の検証が無い。
  - **傾斜 45° 超の Beam** を `StbColumn` として下→上へ節点並べ替え出力する一方
    `ref_vector` は不変のため、往復で i/j 反転＝局所軸（強弱軸/材端力符号）が変わり得る。
  - **DeleteMember の undo** が縮む配列のインデックスで記録し前方順に再挿入するため、
    同一要素を参照する複数の部材荷重の順序が入れ替わる（結果は加算的で数値影響なし）。
  - **壁の厚さ Section** が往復ごとに未参照ゼロ断面として増殖（数値影響なし・リスト肥大）。

## 3. 確認して問題なしと判断した主な領域

数値ソルバ（LU/Cholesky/PCG/Auto、疎組立・縮約 TᵀKT/Tᵀf、固有値、Newmark/HHT の
有効荷重導出、Rayleigh 係数、レインフロー）、要素（弾性 Timoshenko 12×12・せん断係数、
ファイバ B 行列とひずみ符号、`member_load` の等価節点力、パネルゾーン、膜/曲げ B 行列・
Gauss 補外、トラス/節点ばね）、断面性能（H/箱/パイプ/溝形/T/山形/平鋼/丸鋼/リップ溝形/
非対称組立 H/SRC/CFT の A・Iy・Iz・J・Zp・せん断面積）、材料（Bilinear/MP/Concrete/
TsujiYamada/SteelBuckling）、荷重（Ai 分布・累積重量 Qi・PH/地下、Rt/Tc/Z、荷重組合せ、
風・積雪、積載低減、偏心率 D 値法・Fe/Fs/Fes）、終局検定（塑性トラス+アーチ Qsu・付着
Qbu・柱 Nuc/Nut・接合部 Vju/Qdu・CFT）、`.scz` 保存（ハッシュ検証・原子的・zip-bomb 上限）、
決定性/モデルハッシュ。いずれも手計算・理論解・規準例題と整合。
