## 1. 今回修正した差異

### 1.1 RC/SRC 断面の as_y/as_z に鉄筋断面積が入っていた（重大バグ）

- **箇所:** `squid-n-core/src/section_shape.rs` `to_section()`
- **マニュアル:** RC のせん断変形用断面積は `As = B·D/κ`（κ=1.2）。
- **不具合:** `Section.as_y/as_z` は全消費側（ビーム要素の φ 項・せん断ばね・
  鋼材検定・UI 表示）で「せん断断面積」の意味なのに、RC/SRC の生成側だけが
  主筋総断面積を格納していた。RC 柱梁のせん断剛性が約 2 桁過小評価される。
- **修正:** RC 矩形/円形は `A/1.2`、SRC は `rcAs + sAs·(ngs−1)`（ngs は暫定的に
  N_S_EQ で代用）を格納。壁（RcWall）も `A/1.2`。

### 1.2 鋼断面パラメトリック生成の as_y/as_z が未設定（5A/6 フォールバック）

- **マニュアル:** S 造は `As = Aw/κ`（κ=1.0、Aw=ウェブ断面積）。
- **不具合:** `SteelH` 等は as=0 のまま生成され、要素側で矩形近似 `5A/6` に
  フォールバック → H 形で約 2 倍のせん断剛性過大。
- **修正:** 強軸側（as_z↔Iy）=ウェブ（`tw·H`、設計検定側 `steel::shear_area`
  と同一規約）、弱軸側（as_y↔Iz）=フランジ。角形鋼管は 2 面の板、円形鋼管は
  `A/2`。CFT は鋼管部のみ（コンクリート無視の既存方針を踏襲）。

### 1.3 SRC の軸剛性用断面積に鉄骨等価分が未累加

- **マニュアル:** `An = rcAn + sAn·(ns−1)`。
- **不具合:** `Section.area` が質量算定と兼用のため、コンクリート全断面のみと
  していた（コード内に意図の明記あり）。軸剛性 EA が大幅過小。
- **修正:** `SectionShape::calc_axial_stiffness_area()` を新設し、
  `BeamElement` は剛性用 `a`（SRC は等価換算累加）と質量用 `a_mass`
  （幾何断面）を分離。質量への影響なし。

### 1.4 RC/SRC 矩形ねじり定数のアスペクト比 10 以上での打ち切り

- **マニュアル:** `J = (B³D/16)[16/3 − 3.36(B/D)(1 − (1/12)(B/D)⁴)]`（全域適用）。
- **不具合:** 実装は r=h/b≥10 で β=1/3 に切替え、r=10 で約 +6.7% の乖離
  （r<10 では別系近似で差 ±0.12% 以内）。
- **修正:** マニュアル式をそのまま全域に適用（`rect_torsion_j()`）。

### 1.5 剛域自動算定が構造種別を無視（S 造にも D/4 ルールを適用）

- **マニュアル:** S 造・CFT は剛域長 0。混合構造は非対称ルール
  （RC/SRC 部材は RC/SRC 直交材の最大せいフェイスから D/4 内側、
  S/CFT 部材は RC/SRC 直交材のフェイス**まで**、直交材がすべて S なら 0）。
- **不具合:** `auto_rigid_zones` が全梁要素に一律 `λ = D_orth/2 − D_self/4` を
  適用。S 造ラーメンに本来 0 の剛域が入り剛性過大。また直交せい探索に
  壁・シェル等の要素種別も混入し得た（「耐震壁周辺の剛域は考慮しない」に反する）。
- **修正:** `Section.shape`（無ければ Material.fc/fy）による種別判定を導入し、
  上記ルールを実装。直交せい探索は Beam 要素のみに限定。
  種別不明（shape なし・fc/fy なし）は従来挙動（RC 扱い）を維持。

### 1.6 PanelZone::new の dc/db 取り違え

- **不具合:** `evaluate()` は dc=柱せい・db=梁せいの前提（テスト・数値例照合
  と一致）だが、`new()` は水平材（梁）の depth を dc に、鉛直材（柱）を db に
  代入していた。現状 PanelZone 要素の生成経路が無く実害は未発現だが、
  将来接続した時点で全結果が誤る。
- **修正:** 対応を入れ替え、回帰テストを追加。

### 1.7 カタログの as_y/as_z doc コメント（表記のみ）

- CSV 実データ・要素側ペアリング（as_z↔Iy）と逆の「強軸/弱軸」表記だったため
  訂正。数値への影響なし。

### 1.8 追実装（照合後の残差異対応）

- **ブレース剛性 KB=E·A/L・引張専用の active-set 反復:**
  `ElementKind::Brace { tension_only }` と軸剛性のみのトラス要素を新設。
  引張専用ブレースは要素側では特別扱いせず、線形応力解析の active-set 反復
  （`solve_tension_only_iterative`、計算条件 `tension_only_iteration`）で
  圧縮側に入ったブレースを無効化して扱う。
- **SRC のヤング係数比 ns を材料から算定:**
  要素材料（コンクリート）の Ec から ns=Es/Ec、ngs=ns·(1+νc)/(1+νs) を求め、
  An/Ie/As の累加とねじりの複合換算 J=cJ+(sG/cG)·sJ を要素生成時に適用
  （`src_equivalent_props`）。材料から算定できない場合は従来の N_S_EQ=15 に
  フォールバック。
- **CFT の充填コンクリート剛性:** `Material.fc` から
  Ec=3.35e4·(γ/24)²·(Fc/60)^(1/3)（γ=23）を求め、鋼管基準の 1/n 換算で
  A/I/As/J に累加（`cft_equivalent_props`）。fc 未設定時は従来（鋼管のみ）。
- **RC 耐震壁の判定・開口式:** 開口周比 r0=√(h0·l0/(h·l))、
  開口低減率 r=1−1.25·r0、複数開口の等価開口置換
  （l0′·h0′=Σli·hi, l0′:h0′=lw:hw）、判定（t≥120mm・r0≤0.4・スリット無し）を
  `squid-n-design-jp::wall_opening` に実装。その後 main 側で壁属性
  `Model::wall_attrs`（開口合計面積・三方スリット）が追加されたため配線済み:
  - 剛性側（factory）: 開口付き壁要素（暫定等価梁）のせん断剛性に
    r=1−1.25·√(開口面積/壁面積) を乗じる
  - 検定側（joint_wiring）: 耐震壁判定で不成立（薄壁・大開口・スリット）の
    壁は耐震壁せん断検定から除外し、開口は等価開口寸法に復元して
    RC 規準 18 条の γ 式へ供給（02 章の r 式とは別規定のまま独立に配線）
  - 個別開口寸法のデータモデル: `WallOpening { width, height, offset }` を
    `WallAttr.openings` として追加（serde 後方互換）。単一開口は実寸法、
    複数開口は等価開口で統合して検定へ供給し、面積のみの旧入力（
    `opening_area`）は擬似等価開口として従来どおり評価。剛性・自重の
    開口面積は `total_opening_area()`（個別開口優先）に一本化。
    壁属性テーブル UI から「幅x高さ[@x,z]」書式で入力可能。
  - 複数開口の取り扱い 3 モード（`Model::multi_opening_mode`、建物一律）:
    「等価開口とする」（既定）／「包絡する」（位置 offset を持つ開口全体の
    外接矩形に置換）／「包絡開口・等価開口自動判定」（包絡可能な開口対を
    反復包絡し、残りを等価開口で評価）。剛性の開口低減・耐震壁判定・検定の
    開口供給に適用（自重控除は常に実開口面積和）。
    自動判定の「包絡可能」条件は原典の判定図（ユーザー提供 PDF
    「耐震壁(壁エレメントモデル)の剛性」）で確認した
    **l < 1.5·h または l < 1m**（l: 開口間距離、h: 包絡開口とした場合の高さ）を
    実装（当初の暫定基準を置換。specs/原典照合リスト.md で照合済み ☑）。
- **スラブ協力幅による剛性増大（RC 規準 8 条・v1）:**
  建物一律の床厚（`Model.slab_thickness`）と梁に隣接するスラブから
  ba=(0.5−0.6·a/l)·a（a≥l/2 は 0.1·l）で協力幅を求め、スラブ考慮の中立軸による
  T 形断面の Ie/I0 を強軸剛性に乗じる。連続梁 λ・吹抜け補正・二重スラブ/
  片持ちスラブの扱いは未対応（v1 の制約としてコード内に明記）。

### 1.8.1 壁エレメントモデル本体の実装（原典 PDF 提供後）

- **壁エレメントモデル（4 節点 24 自由度）:** 参照実装マニュアル計算編 02 のモデル
  （鉛直の壁柱を両端ピンの剛梁ではさむ。剛梁は剛域変換で表現し実要素は
  置かない）を `WallPanelElement` として実装。四隅節点の並進のみが壁柱端の
  並進（平均）と回転（剛梁の剛体回転）へ伝達され、四隅の回転自由度は
  ピン（剛性ゼロ）。壁柱の断面性能は軸=t·lw、面内曲げ=t·lw³/12（側柱の
  ローカル I 不算入）、せん断=A/κ×開口低減率 r。
  従来の暫定等価梁は 4 節点未満・断面/材料未設定時のフォールバックとして残置。
- **上下大梁の剛性倍率:** 壁の上下辺を構成する水平大梁の剛性
  （EA・EI・GJ・せん断断面積）に既定 100 倍を乗じる
  （`WALL_GIRDER_STIFF_FACTOR`。剛性計算条件 UI からの変更は将来対応）。
- **引張専用ブレースの active-set 反復:** 線形応力解析で圧縮側に入った
  引張専用ブレースの軸剛性を残留比 1e-6 まで縮小して無効化し、収束するまで
  再解析する真の引張専用解析を実装（計算条件 `tension_only_iteration`）。
  active な引張ブレースのみが N=EA/L·δ を負担する。
- **追実装（同日）:** 壁柱の鉄筋剛性考慮（壁筋比 ps による (1+(n−1)·ps) 近似）、
  I 形断面の形状係数 κ と側柱断面のせん断剛性への算入（ξ・η の記号定義は
  原典ページに明示がなく仮定＝原典照合リスト登録済み）、側柱の面内両端ピン化
  （面内曲げ面のみ端部回転を静的縮約する `InPlaneReleasedColumn`。
  RC 耐震壁の側柱に自動適用され、面内のモーメント・せん断を負担しない）。
- **フレーム内雑壁のモデル化（追実装）:** 耐震壁判定（t≥120mm・r0≤0.4・
  スリット無し）が不成立の壁は、壁要素の剛性を実質ゼロ（質量のみ保持）とし、
  周辺の柱（袖壁）・梁（腰壁/垂壁）の断面性能へ平行軸合成で算入する。
  壁の長さはマニュアルどおり構造階高・軸間距離の 1/2 位置における
  包絡開口（位置付き個別開口の外接矩形）までのフェイス距離
  （開口が 1/2 位置を跨がない・開口位置不明の場合は折半則＝実装上の補完）。
  上下大梁の 100 倍剛性は耐震壁成立時のみに限定し、側柱の面内ピン化も
  成立壁のみに適用（不成立壁の柱は袖壁付き柱として通常挙動）。
  三方スリット壁は縁切りのため剛性算入せず自重のみ評価。

### 2026-07-13 追補2（既知ギャップの実装）

- **S 造合成梁の断面性能を実装**: スラブが取り付く水平な H 形鋼梁について、
  スラブを考慮した換算断面剛性 I（スラブ上端基準の図心 g による）と鉄骨単独
  剛性 sI の平均 (I+sI)/2 を強軸剛性に採用する（各種合成構造設計指針の
  完全合成梁剛性の平均法。`composite_beam_stiffness_factor`）。協力幅は
  RC 梁と共通の RC 規準 8 条（内法距離 a）による。簡略化: デッキ高さ Hd=0、
  スラブコンクリートは Fc21 標準仮定（モデルにスラブ材料が無いため）、
  頭付きスタッド等の合成条件は判定しない。

## 2026-07-20 追補3（弾性要素の内力トライアル追従化・重大バグ）

### 不具合の内容（追補1・2で「別課題」とした準ニュートン問題の根本対処）

調査の結果、問題は当初の認識より深刻だった。非線形ドライバ（プッシュオーバー・
非線形時刻歴・弧長法）は例外なく「反復中 `update_state(du, commit=false)` →
収束時 `commit_state()`」の規律で要素状態を管理するが、凍結型の弾性要素は
`update_state(du, false)` が du を捨て、`commit_state` も既定の空実装のため、
**Newton 反復中だけでなく収束・commit 後も含めて `committed_disp` が初期値 0 の
まま一切更新されず、これらの要素は非線形解析で復元力を全く負担していなかった**
（釣合いはトライアル追従要素＝ファイバー系のみで成立し、弾性要素は接線剛性
としてのみ寄与）。収束も準ニュートン（幾何級数）に劣化していた。

凍結型だった要素:
1. `BeamElement`（frame/beam/behavior.rs）
2. `TrussElement`（frame/truss.rs）
3. `NodalSpringElement`（springs/spring.rs）
4. `InPlaneReleasedColumn`（wall/side_column/column.rs、内包 BeamElement 経由）
5. `ConcentratedSpringBeam` の弾性梁部分（frame/concentrated/mod.rs。
   端部バネはトライアル追従で正しかったが、梁本体の軸・せん断・非解放曲げの
   弾性復元力が凍結。剛床上の梁の既定要素であり影響大）

### 修正

各要素に `trial_disp` を追加し、`update_state` は commit の真偽に関わらず
トライアルへ蓄積、`internal_force` はトライアルを参照、`commit_state`＝確定、
`revert_state`＝巻き戻し、`snapshot_state`/`restore_state`＝ロールバック用の
完全な状態保存を実装した（FiberBeam 等の既存トライアル追従型と同じ規約）。
`ConcentratedSpringBeam` は N-M 相関の軸力評価もトライアル基準へ変更し、
スナップショットに弾性部変位を含めた。回帰テスト
`test_beam_trial_displacement_tracking`（beam/tests.rs）で規律全体を固定。

これにより Newton 反復は弾性支配の状態でほぼ 1〜2 回で収束するようになる
（追補2 の反復上限 50 は塑性進行時の余裕として維持）。

### 残課題（別カテゴリ・未着手）

`ShellElement`・`PanelZone`・`WallPanelElement` は「凍結」ではなく
**`internal_force` が恒常的にゼロを返す**実装であり、非線形解析で復元力を
負担しない点は同様の懸念がある（特に壁エレメントはプッシュオーバーの主要
耐震要素）。トライアル追従＋`internal_force = K·u` 化は機械的に可能だが、
壁付きモデルの非線形結果が大きく変わるため、参照値との照合を伴う独立した
検証作業として別途実施すべきである。

### 影響

非線形解析（プッシュオーバー・非線形時刻歴）で弾性要素（梁・トラス・
節点バネ・集中ばね梁の弾性部）が正しく復元力を負担するようになり、
釣合い状態・ベースシア配分・N-M 相関の軸力評価が変わる。既存の非線形
解析結果は再実行が必要。線形解析（linear_static 等）は `internal_force`
系を使わないため影響なし。
