## 1. 不一致・バグ疑い

### 1.1 【高】スラブ荷重分配の結果が応力解析へ渡っていない

`crates/squid-n-app/src/app.rs:1348-1373`（`refresh_beam_loads`）は `distribute_slab` の
`BeamLoad` を `self.beam_loads` に格納するのみで、消費先は
`crates/squid-n-app/src/viewer/mod.rs:1024-1064` の CMQ 図描画だけである。
線形静解析が読む `LoadCase.member: Vec<MemberLoad>`（`crates/squid-n-core/src/model.rs:257-264`）
へスラブ荷重が自動反映される経路が存在しない（`MemberLoad` の生成箇所は手動入力 UI
`tables/loads.rs:496` とテストフィクスチャのみ）。

床面荷重を応力解析に効かせるには等価部材荷重の手動入力が必要で、
「面荷重→大梁→解析」のパイプラインは事実上デモ／検算専用。
仕様書 P2 §9.1 の最終目的（解析への投入）が未達成。

### 1.2 【高】壁（耐震壁・非耐震壁・雑壁）の自重が地震用重量に算入されない

`crates/squid-n-load/src/story_gen.rs:108` の自重集計は
`ElementKind::Beam` のみ対象で、`Wall` / `Shell` はスキップされる。
マニュアル「壁自重」の「壁の重量を階高の中央で上下階の節点に分配」に相当する処理が
リポジトリ全体に存在しない。耐震壁を持つ建物では層重量 `seismic_weight` が大きく過小評価される。

また、固有値解析用の質量は要素密度由来（TᵀMT 縮約）で壁質量を**含む**ため、
モーダル質量と地震用層重量が不整合になる（固有周期は壁込み、Ai 分布の Wi は壁抜き）。

### 1.3 【高】ダンパー自重が算入されない

同じく `story_gen.rs:108` のフィルタにより、ダンパー・ブレース類の自重計算経路が無い。
マニュアル「ダンパー自重」（装置重量＋支持部重量、鉛直/水平配置による上下階振り分け）
および「自重を考慮しない部材」の除外リストは全て未実装。

### 1.4 【中〜高】地震用節点重量の配分が常に両端 1/2（CMoQo ベースでない）

`story_gen.rs:152-158` は部材荷重の合計を常に両端へ 1/2 ずつ配分する。
マニュアル「地震荷重の計算」は「大梁の CMoQo の計算で求めた梁せん断力」を節点重量とする
（＝台形/三角形分布なら固定端反力比で配分）。層合計は保存されるため Qi への実害は小さいが、
剛床代表節点の**重量重心**（`story_gen.rs:195-219`）に直接影響し、偏心率・ねじれ評価の
精度を損なう。`floor.rs` に正しい FEM 関数（`fem_uniform`/`fem_triangle`/`fem_trapezoid`）が
実装済みなのに `story_gen.rs` が使っていない。

### 1.5 【中】略算一次固有周期 T=h(0.02+0.01α) の α が常に 0 固定

`crates/squid-n-solver/src/analysis.rs:287-288` および
`crates/squid-n-solver/src/pushover.rs:255-256` で `steel_ratio = 0.0` がハードコード。
`Story`（`crates/squid-n-core/src/model.rs:116-123`）に構造種別（S/RC）を持つ
フィールド自体が無く、データモデルレベルで α を表現できない。
S 造・混構造では略算 T が過小（0.02h 固定）となり、Rt の低減が効かず Ci・Qi が過大側に歪む。

### 1.6 【中】多剛床時に同一 Pi が各剛床へ重複載荷される懸念

`analysis.rs:329-341` は `for dia in &story.diaphragms` で階内の全剛床代表節点に
**階全体の Pi をそのまま**載せる。1 階に複数剛床を定義すると水平力合計が
剛床数倍になる。現状の `story_gen.rs` は 1 階 1 剛床しか生成しないため顕在化しないが、
マニュアルの「重量比で分配」（多剛床の設計用せん断力）とも異なる潜在バグ。

### 1.7 【中】地震用重量に使う荷重ケースが「先頭の LoadCase」という暗黙規約

`crates/squid-n-app/src/app.rs:770,906` で `load_cases.first()` を機械的に採用。
`LoadCase` に荷重種別（固定/積載/地震用積載）の区分が無く、マニュアルの
「床の積載荷重は地震用の値とします」（長期用と別値）を表現できない。
ケースの並び順変更で誤った荷重が地震用重量に混入するリスク。

### 1.8 【中】RC/SRC 梁の自重長さが節点間距離（マニュアルは柱面間距離）

`story_gen.rs:120-125` は常に節点間直線距離で ρ·A·L·g を計算。
マニュアルは RC/SRC 大梁は柱面間距離（S 梁は節点間距離で整合）。
`ElementData.rigid_zone`（`face_i`/`face_j`）が剛性側では参照されているのに
自重側では未参照で、RC/SRC 梁の自重を柱幅分だけ過大評価する。

### 1.9 【中】コンクリート単位体積重量が Fc・種別によらず固定

`crates/squid-n-app/src/tables/materials.rs:9-16` は Fc21/24/30 とも
`density = 2.4e-9 t/mm³`（γ≈23.5 kN/m³）固定。マニュアルの表は
γC/γRC/γSRC を 17.0〜26.5 kN/m³ の範囲で Fc・普通/軽量別に規定し、
鉄筋込み（γRC=24.0, Fc≦36）に対して現状値は約 2% 小さい。RC/SRC の割増区別も無い。

### 1.10 【中】荷重組合せの欠落（combo.rs）

`crates/squid-n-load/src/combo.rs:14-37`:
- 地震組合せが正方向のみ（`G+P+Kx` / `G+P+Ky`）。**±両方向**（G+P−K）が生成されない
  （シグネチャが方向あたり 1 ケースのみ受け取る設計）。
- 多雪区域の組合せ（長期 `G+P+0.7S`、短期 `G+P+K+0.35S` / `G+P+W+0.35S`）が無い。
- 暴風時 `G+P+W` が無い（風荷重自体が未実装のため）。
- 長期 `G+P`、積雪短期 `G+P+S` は施行令82条と一致。

### 1.11 【低】重力加速度定数が 2 系統

`story_gen.rs:27` は `9800.0`、`crates/squid-n-core/src/units.rs:1` は `9_806.65`
（`capacity_spectrum.rs:17` も後者）。差 0.07% で実害は小さいが
単一のソースオブトゥルースに統一すべき。

### 1.12 【低】Pi の負値がサイレントに 0 へクランプ

`crates/squid-n-load/src/ai.rs:71` の `p.max(0.0)`。異常な重量分布の検知機会を潰す。
`ΣPi = Q1` の保存則を実行時にも検証するとよい（テストには存在）。

### 1.13 【低】`slab_dimensions` が矩形前提で第 3 頂点を検証しない

`floor.rs:120-137` は `boundary[0],[1],[3]` のみで lx/ly を決め、`boundary[2]` の
整合チェックが無い。非直交四辺形が来ても矩形として黙って処理する。
また `Slab` に荷重伝達方向（XY/X/Y）の指定フィールドが無く、`OneWay` の伝達方向が
境界ノードの並び順に暗黙依存する（`floor.rs:78-92`、マニュアルは床ごとに方向指定）。

## 3. マニュアルにあるが未実装の機能（ギャップ）

P2〜現行フェーズのスコープ外を含む。優先度はプロジェクト方針で判断されたい。

**荷重種別まるごと未実装:**
- **風荷重一式**（速度圧 q=0.6E·V0²、E=Er²·Gf、Kz、外圧/内圧係数、地表面粗度区分表、
  ガスト影響係数、見付面積）— リポジトリ全体に該当コード無し
- **PH 階の震度指定**（k=0.5〜1.0、Qi=k·ΣWj）
- **地下階の水平震度**（Ki=0.1(1−Hi/40)·Z）
- **多剛床の副剛床 Ci 直接入力・重量比分配**

**スラブ荷重まわり:**
- 小梁を介した二段階伝達（`Slab.joists` はデータのみ存在、`distribute_slab` は未参照）
- 片持ちスラブ（片持ち梁・先端リブ小梁の有無による分配、出隅の柱への節点荷重集中）
- 多角形床組（五角形・三角形・台形）— 現状は直交矩形のみ
- 剛域考慮 CMQ（可撓部 CMQ→剛域片持ち梁再合成、荷重条件 3 方式の切替）
- ※入隅の片持ちスラブはマニュアル自体が非対応と明記（問題なし）

**自重まわり:**
- 仕上げ荷重 w_f（四面仕上げ）・壁仕上げ（片面）
- 耐火被覆重量 γc·Ac
- 鉄骨重量割増率 α（部材種別ごと）
- 梁・柱長さの取り方（柱面間/梁面間/柱面〜先端、下階柱なし時の梁せい付加）
- Fc・コンクリート種別による γ の変化（γC/γRC/γSRC 表）
- 壁開口考慮・連スパン壁の 4 節点振り分け・雑壁の 0.5m 分割集計・3 方スリット壁
- K 型ブレースの重量配分選択
- 支える床数に応じた積載荷重低減（柱軸力算定時）
- Z の市町村別対応表（現状はユーザー直接入力のみ。仕様書 §5.2 でも外部データ扱い）

## 5. 推奨対応順

1. **スラブ荷重→`LoadCase.member` への自動接続**（§1.1）— 解析結果に直結する最重要配線
2. **壁自重の地震用重量算入**（§1.2）— 耐震壁付きモデルの Qi が現状過小
3. **地震用節点重量の FEM 反力ベース配分**（§1.4）— `floor.rs` の既存関数を流用可能
4. 略算 T の α 対応（`Story` への構造種別追加）と多剛床 Pi 分配の修正（§1.5, §1.6）
5. 荷重ケース種別（固定/積載/地震用積載）の導入（§1.7）
6. 組合せの ± 方向・多雪区域係数（§1.10）、重力加速度の定数統一（§1.11）
7. ギャップ項目（§3）はフェーズ計画（風荷重・仕上げ荷重等）として別途優先度付け
