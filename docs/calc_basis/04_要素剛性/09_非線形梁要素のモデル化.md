# 4.9 非線形梁要素のモデル化（材端集中・ファイバー・MS）

非線形解析（プッシュオーバー・時刻歴応答）では、線形の弾性ティモシェンコ梁（4.1）に代えて、部材の弾塑性挙動を表す **3 種類の梁要素モデル**を用いる。材端集中ばねモデルは弾性ティモシェンコ梁を土台とし、ファイバー・MS モデルは Timoshenko 適合内挿（φ 依存形状関数、4.9.2）によりせん断変形を曲げと直列に合成する（弾性状態では弾性ティモシェンコ梁と厳密一致）。

| モデル化 | 塑性化の表現 | N–M 相関 | 実装 |
|---|---|---|---|
| 材端集中ばね | 部材端の回転ばね（M–θ 履歴則） | 降伏モーメント低減式 | `frame/concentrated` |
| ファイバー | 積分点断面のファイバー分割（平面保持） | 断面積分で自然に表現 | `frame/fiber` |
| マルチスプリング（MS） | 端部塑性化域を軸ばね群に置換 | 軸ばね合力で自然に表現 | `frame/multi_spring` |

> 各項目は「導入文（何を・どの基準で）、算定式、実装」の順で記す。

## 4.9.1 材端集中ばねモデル

弾性ティモシェンコ梁の両端に**回転ばね（M–θ 履歴則、2.2 節）**を直列接続し、塑性化を材端の集中回転に代表させる（構造動力学で標準的な集中ばね（one-component / two-component）モデル）。

**算定式**

- 履歴則は部材個別指定 → 構造種別ごとの既定表で解決する。RC/SRC/CFT 梁は**武田型トリリニア**、S 梁は**標準型（kinematic バイリニア）**を材端ばねに用いる。
- 軸-曲げ相互作用（N–M 相関）は、現在軸力 \\( N \\) に応じて回転ばねの降伏モーメントを

\\[ M_{\text{lim}} = M_{y0} \cdot \left(1 - \frac{|N|}{N_{\text{allow}}}\right) \\]

で更新する（下限 \\( 0.02 \cdot M_{y0} \\)）。この相関はバイリニア（標準型）にのみ適用し、武田型等の骨格固定の履歴材料は対象外とする。

**実装**：`squid_n_element::frame::concentrated`（`concentrated/mod.rs::ConcentratedSpringBeam`）が算定する。
履歴則の解決は `factory::resolve_member_hysteresis`、材端ばねの構築は `factory::build_flexural_springs`、N–M 相関の有効化は `ConcentratedSpringBeam::with_mn_interaction` による。

## 4.9.2 ファイバーモデル

積分点（ガウス点）の断面を多数の**ファイバー**に分割し、平面保持仮定のもとで各ファイバーが独立した履歴状態を持つ一軸材料として応答する（構造力学のファイバーモデル）。軸力と曲げの連成（N–M 相関）を断面積分から自然に得る。断面応答（M–φ 積分）の詳細は 3.7 節を参照。

**算定式**

- 既定は 2 点ガウス積分（\\( \xi = \mp 1/\sqrt{3} \\)、重み 1）とする。
- 塑性化域考慮モデルでは、端部積分点 \\( \xi = \mp 1 \\)（重み \\( 2L_p/L \\)）に塑性化域断面を配置し、中央部 \\( [L_p,\, L-L_p] \\) は弾性剛性 \\( B(\xi)^{\mathsf T}\,\mathrm{diag}(EA, EI_y, EI_z)\,B(\xi) \\) を 2 点ガウスで厳密積分して加算する。塑性化域長 \\( L_p \\) は部材長の 45% までにクランプする。
- RC 断面はコンクリートファイバー格子に加え、**主筋を点ファイバー（バイリニア鋼材）として分離配置**する。コンクリートは \\( f_c \le 60 \\) で NewRC 構成則、超過で放物線モデルを用いる。
- せん断変形は **Timoshenko 適合内挿**（φ 依存の曲率形状関数＋一定せん断ひずみ場による変位法内挿。interdependent interpolation 系の定式化）で曲げと直列に合成する。せん断変形係数

\\[ \varphi = \frac{12EI}{GA_s L^2} \\]

は**公称断面諸元**（Section の \\( I \\)・\\( A_s \\) と Material の \\( E \\)・\\( G \\)）から算定して凍結する。降伏後も内挿は弾性時の配分を保つ（曲げの Hermite 内挿と同型の近似）。公称値を用いるのは線形解析（4.1 の弾性 Timoshenko 梁）の φ と一致させるためであり、RC 断面ではファイバーの実効初期剛性が公称値と僅かに乖離するため、φ は公称値ベースの近似となる。

- 曲率場は回転項を \\( (1 \pm 3\xi) \to (1 \pm 3\xi + \varphi) \\)、全体を \\( 1/(1+\varphi) \\) 倍に補正する。せん断ひずみは**曲げ面ごとに**要素内一定の

\\[ \gamma_y = \frac{\varphi_y}{2(1+\varphi_y)} \left( \theta_{z,i} + \theta_{z,j} - \frac{2\,\Delta u_y}{L} \right), \qquad \gamma_z = \frac{\varphi_z}{2(1+\varphi_z)} \left( \theta_{y,i} + \theta_{y,j} + \frac{2\,\Delta u_z}{L} \right) \\]

（\\( \Delta u \\) = 端部並進差。**符号は各曲げ面の曲率規約に従い面ごとに異なる**点に注意。いずれも剛体回転で恒等的にゼロとなる客観的な測度）とし、弾性せん断剛性 \\( GA_s \\) を作用させる。曲率剛性とせん断剛性の和は、断面剛性が φ の算定基礎と一致する一様弾性断面で、弾性 Timoshenko 梁（4.1）の剛性と**厳密に一致**する（被積分関数は \\( \xi \\) の 2 次のため 2 点ガウスで厳密）。
- \\( GA_s \le 0 \\)（せん断有効断面積が未設定等）の場合は \\( \varphi = 0 \\)（せん断剛直 = Euler-Bernoulli）へフォールバックする。幾何剛性・整合質量は Euler 系の係数を用いる（4.1 の弾性梁と同じ扱い）。

**実装**：`squid_n_element::frame::fiber`（`fiber/mod.rs::FiberBeam`）が算定する。
断面のファイバー応答は `squid_n_section::fiber::section_response`、塑性化域考慮モデルの構築は `FiberBeam::build_plastic_zone`、φ 依存 B 行列は `FiberBeam::compute_b_matrix`、せん断ひずみ場の剛性は `FiberBeam::compute_shear_stiffness` による。
弾性 Timoshenko 梁との厳密一致・初期剛性の理論値照合・剛体回転の客観性・接線と内力の整合は `fiber/tests.rs` の回帰テストで担保する。

## 4.9.3 マルチスプリング（MS）モデル

部材端の塑性化領域（長さ \\( L_p \\)、既定は断面せいの 0.5 倍）の断面を、少数の**軸方向ばね群**で置換し、中央は弾性材で連結する（RC 柱・梁端部の塑性ヒンジを軸ばね群で表す MS モデル）。軸ばね群の合力が \\( N \\)、図心まわりの偶力が \\( M \\) となるため、N–M 相関を自然に表現できる。ねじりばねは別系統で付加する。

**算定式**

- 端部断面のばね配置は幅方向 × せい方向 = 2 × 5 = **10 本の 2 次元配置**とし、二軸曲げに対応する。
- 実体は塑性化域考慮ファイバー要素（4.9.2 の `build_plastic_zone`）と**同一の定式化**であり、端部断面の分割数だけが粗い（ばね = 粗いファイバー）。したがって trial/commit/rollback・チェックポイントもファイバー要素と同一機構に乗る。
- 妥当性は、一定軸力下の繰り返し曲げにおける N–M 相関・履歴ループがファイバー要素と整合することで担保する（相互検証。実装仕様書 P5.5 §5.1）。
- せん断変形は 4.9.2 の **Timoshenko 適合内挿**と同一（実体がファイバー要素のため定式化も共有する）。

**実装**：`squid_n_element::frame::multi_spring`（`multi_spring.rs::MultiSpringElement`）が算定する。
内部で `FiberBeam::build_plastic_zone(data, model, L_p, 2, 5)` を構築し、`ElementBehavior` の各メソッドを委譲する。

## 4.9.4 モデルの自動選択（ForceRegime）

`ElementKind::Beam` の要素は、部材の `ForceRegime` に基づいて非線形要素種別を選択する。

**選択規則**

- `ForceRegime::UniaxialBendingShear` → 材端集中ばね
- `ForceRegime::AxialBendingInteract` → ファイバー
- `ForceRegime::Auto` → トポロジから判定（ヒューリスティック）。**剛床に所属する水平材（軸力変動が小）→ 材端集中**、それ以外（柱などの鉛直材）→ ファイバー。鉛直材判定は \\( |\Delta z| > (|\Delta x| + |\Delta y|) \cdot 0.5 \\) による。

MS モデルはこの自動選択には乗らず、要素種別 `ElementKind::MultiSpring` を明示指定した場合にのみ用いる。

線形弾性解析は常に弾性 `BeamElement`（`factory::build_behavior`）を用い、非線形解析でのみ上記の非線形要素を構築する（`factory::build_nonlinear_behavior`）。両者を分けるのは、`Auto` 判定が剛床に乗らない梁もファイバーへ振り分けるため、共通化すると線形解析の弾性梁まで非線形要素へ置き換わってしまうためである。

**実装**：非線形要素の生成は `frame::factory::build_nonlinear_behavior`（`factory/mod.rs`）、`ForceRegime::Auto` の判定は `frame::factory::resolve_force_regime`（`factory/regime.rs`）による。
