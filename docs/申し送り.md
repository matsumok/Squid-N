# 申し送り（実装内容と残課題）

作成日: 2026-07-04
対象ブランチ: `claude/structural-analysis-roadmap-jh8om8`
関連文書: `docs/ROADMAP.md`（診断と Phase 別の実装状況）

## 1. 何を実装したか（要約）

GUI 単独で「空モデル → モデル作成 → 荷重 → 解析(静的/固有値/地震/プッシュオーバー/時刻歴) →
検定・層指標 → レポート → 保存」が完結するようになった。あわせてソルバーの
サイレント失敗・panic 経路を除去し、実装中に発見した数値バグ 1 件を修正した。

## 2. 実装方針のポイント（どう実装したか）

### 2.1 編集コマンド(squid-n-edit)
- 追加系(`AddMaterial`/`AddLoadCase` 等)は既存の「末尾に ID=len で追加」規約を踏襲。
- 削除系は **ID=配列インデックス不変条件**（`Model::validate` が強制）を守るため、
  後続 ID を全参照込みで繰り上げる `shift_*_ids` ヘルパを導入した。
  既存の `DeleteMember`/`DeleteSection` はこの再採番をしておらず、
  **末尾以外を削除すると validate が壊れる既存バグ**だったため同時に修正した。
- `DeleteMember` は部材荷重をカスケード削除し、undo で(ケース index, 荷重 index)ごと復元する。
- `DeleteLoadCase` は組合せから参照中なら Noop（UI はボタン無効化+ツールチップ）。

### 2.2 階の自動生成(squid-n-load::story_gen)
- 節点 Z を 1mm 許容でクラスタリングし、最下レベルを基部として除外
  （前回生成分の剛床代表節点＝`model.generated_masters` はクラスタリング対象から除外する）。
- 剛床代表節点は、各階の全構造節点の**慣性力重心**（重量重み付き重心
  `Gx=Σ(iW·ix)/ΣiW`, `Gy=Σ(iW·iy)/ΣiW`）に**専用の仮想節点として自動生成**する
  （既存節点の流用ではない。参考: RESP技術ブログ「剛床に関連する操作や考え方の
  まとめ」(2026-05-29)）。重量が算定できない場合（`ΣiW<=0`）は幾何重心へフォールバックする。
  代表節点は要素が接続しない浮遊節点のため、剛床が拘束しない Uz/Rx/Ry を固定する
  （Ux/Uy/Rz は自由）。質量は常に `None`（要素・節点側の質量が Reducer の TᵀMT 縮約で
  自動的にマスターへ集約されるため、代表節点自体に持たせると二重計上になる）。
  当該階の全構造節点（1 点のみでも）がスレーブになる。
- `Story.diaphragms`（地震力の作用点）と `Constraint::RigidDiaphragm`（Reducer が読む拘束）の
  **両方**を生成する（Reducer は stories.diaphragms を読まないため、片方だけでは力の伝達か
  縮約が欠ける）。
- 再生成時は `model.generated_masters` に記録された既存の代表節点を階の下から順に再利用
  （座標・拘束・所属階を更新）し、足りない分だけ末尾連番で新規生成する。階数が減った場合の
  余った旧代表節点は `restraint=FIXED`・`story=None` で不活性化しつつ `generated_masters` には
  残す（次回再生成時の再利用に備える）。
- 地震重量 = 自重(ρ·A·L·g、両端 1/2 ずつ) + **先頭荷重ケース**の鉛直下向き荷重(節点配分)。
  簡易法であり、通常の「支配面積+仕上げ・積載の地震用低減」より粗い。
- `ApplyStories` コマンドで undo 可能に。既存 RigidDiaphragm は置換、Mpc/RigidLink は保持。
  代表節点（`rep_nodes`）は ID＝配列インデックス不変条件を保って置換または追加し、
  `model.generated_masters` を更新する。逆操作 `RestoreStories` は `model.nodes` を丸ごと
  復元することで、追加された代表節点の除去（truncate）も含めて完全に元へ戻す。

### 2.3 ソルバー安定化(squid-n-solver / squid-n-math)
- `Analysis::prepare` の冒頭で `Model::validate` + `precheck_model` を実行し、
  「支点がない」「断面/材料未割当(部材 ID 列挙)」「孤立節点」を実行前に日本語で報告。
  Cholesky 失敗(特異)時は原因候補を含む診断文に変換。
- `SolveError` に `InvalidInput`/`NonConvergence` を追加。既存の呼び出し側は
  `Display` 経由で表示しているため互換。
- 固有値: 質量トレース≤0 でエラー、部分空間反復が 200 回で収束しなければエラー
  （従来は**収束していなくてもそのまま結果を返していた**）。
- `DirectSparseLu` を faer の sparse LU で実装（`todo!()` panic を除去）。

### 2.4 梁の一貫質量行列バグ修正(squid-n-element/beam.rs) ← 重要
- 曲げブロックを書き込む閉包が DOF [1,5,7,11]/[2,4,8,10] ではなく**連続添字
  [1..4]/[2..5] に書いていた**。回転-回転項が欠落し、誤った位置に連成項が入り、
  一貫質量行列が不定値になっていた。
- 症状: 時刻歴解析の有効剛性 K̂=K+c₂C+c₁M の Cholesky が必ず失敗。
  固有値解析(一貫質量使用)の周期・刺激係数も歪んでいたはず。
- 修正後は M が正定値になり全テスト緑。**既存の固有値検証値がある場合は再確認を推奨**
  （ワークスペース内テストは全て通過している）。

### 2.5 時刻歴(timehistory.rs)
- `ResponseResult` に UI 用の代表応答 `history`（記録節点の X 変位・ベースシア・
  最上階層間変形角、全ステップ）を追加。記録節点は「X 方向自由度を持つ最高標高の節点」。
- ベースシア V(t) = −(rᵀM·a_rel + ẍg·rᵀMr)。X 方向のみ。
- 初期加速度 M·a₀=rhs は Cholesky→LU→(rhs≈0 なら a₀=0)の順でフォールバック。
  地震波の先頭が非ゼロかつ M 特異の場合のみエラー。
- 減衰は UI からは「1 次固有円振動数に対する剛性比例(h 指定)」のみ。Rayleigh は
  ソルバー実装済みだが UI 未配線。

### 2.6 GUI
- 解析は **UI スレッドで同期実行**（進捗バーなし）。小規模モデルでは問題ないが、
  大規模モデルでは UI がブロックする。P8 の非同期ジョブ化は未着手。
- プッシュオーバーは `model.clone()` に対して実行（非線形状態の副作用を GUI モデルに残さない）。
- 波形 CSV は「1 行 1 値(カンマ区切りなら最終列)、単位 gal(×10 で mm/s² に変換)、
  dt は UI の設定値」という簡易仕様。ヘッダ行は数値パース失敗として自動スキップされる。

## 3. 残課題（優先度順の提案）

1. **解析の非同期化+進捗表示**（P8）: 現状 UI 同期実行。時刻歴の長波形でフリーズ感が出る。
2. **CMQ 図のデータ供給**: `app.beam_loads` に書き込む経路がなく、CMQ 表示は常に空
   （床荷重→CMQ の `squid_n_load::floor` は実装済み。スラブ入力 UI がないため）。
3. **荷重組合せの UI**: `linear_combination` はソルバー側共通化済みだが、組合せの
   作成/編集/実行 UI が未配線。
4. **保有耐力比較・Ds の設計タブ統合**: プッシュオーバーの Qu 表示まで。
   `holding_capacity::check_holding_capacity`・`ds` との突合表示は未実装。
5. **階の手動編集**: 自動生成のみ。階高・重量・所属節点の手動調整 UI がない
   （生成後に `seismic_weight` を編集する導線を推奨）。
6. **地震重量の精緻化**: 現状は自重+先頭荷重ケース。ケース選択 UI と
   積載荷重の地震用低減係数の導入余地あり。
7. **時刻歴の Y 方向・2 方向入力**: `GroundMotion.accel_y` はソルバー対応済み、UI 未対応。
   記録応答も X 方向のみ。
8. **固有値検証値の原典照合**: 質量行列修正(2.4)により固有周期が変わった可能性がある。
   `specs/原典照合リスト.md` の該当項目があれば再照合を推奨。
9. **HHT-α / 非線形時刻歴の UI 配線**: ソルバー実装済み・未配線。
10. **レポートの PDF/Excel 化**(P9)・**ST-Bridge**・**GPU/ML/P12** は元計画通り未着手。

## 4. 既知の制約・注意点

- 節点荷重の編集対象ケースは「荷重ケース表でクリック選択したケース」。未選択時は
  最後に実行したケース→先頭ケースの順でフォールバックする。
- `LoadCaseId(0)` は地震静的の結果格納にも使われる（既存仕様を踏襲）。ユーザー作成の
  ケース 0 と結果表示上は同居するため、将来は結果キーを (種別, ID) に分けるのが望ましい。
- 材料の鋼/RC 判定は名称先頭文字（SS/SN/SM/STK/ST）による簡易判定のまま
  （`Material` に種別フィールドを足すのが本筋）。
- プリセット材料の密度は ton/mm³（内部単位系 N-mm-s）: 鋼 7.85e-9、RC 2.4e-9。
- rfd(ファイルダイアログ)は `gui` フィーチャに含めた。ヘッドレス CI では
  ダイアログ関数は呼ばれない（テストはパス直指定の `save_project_to` 等を使用）。
