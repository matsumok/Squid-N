# 申し送り（実装内容と残課題）

作成日: 2026-07-04
対象ブランチ: `claude/structural-analysis-roadmap-jh8om8`
関連文書: `docs/ROADMAP.md`（診断と Phase 別の実装状況）

## 1. 何を実装したか（要約）

GUI 単独で「空モデル → モデル作成 → 荷重 → 解析(静的/固有値/地震/プッシュオーバー/時刻歴) →
検定・層指標 → レポート → 保存」が完結するようになった。あわせてソルバーの
サイレント失敗・panic 経路を除去し、実装中に発見した数値バグ 1 件を修正した。

## 2. 実装方針のポイント（どう実装したか）

### 2.1 編集コマンド(squid-n-edit)
- 追加系(`AddMaterial`/`AddLoadCase` 等)は既存の「末尾に ID=len で追加」規約を踏襲。
- 削除系は **ID=配列インデックス不変条件**（`Model::validate` が強制）を守るため、
  後続 ID を全参照込みで繰り上げる `shift_*_ids` ヘルパを導入した。
  既存の `DeleteMember`/`DeleteSection` はこの再採番をしておらず、
  **末尾以外を削除すると validate が壊れる既存バグ**だったため同時に修正した。
- `DeleteMember` は部材荷重をカスケード削除し、undo で(ケース index, 荷重 index)ごと復元する。
- `DeleteLoadCase` は組合せから参照中なら Noop（UI はボタン無効化+ツールチップ）。

### 2.2 階の自動生成(squid-n-load::story_gen)
- 節点 Z を 1mm 許容でクラスタリングし、最下レベルを基部として除外
  （前回生成分の剛床代表節点＝`model.generated_masters` はクラスタリング対象から除外する）。
- 剛床代表節点は、各階の全構造節点の**慣性力重心**（重量重み付き重心
  `Gx=Σ(iW·ix)/ΣiW`, `Gy=Σ(iW·iy)/ΣiW`）に**専用の仮想節点として自動生成**する
  （既存節点の流用ではない。参考: RESP技術ブログ「剛床に関連する操作や考え方の
  まとめ」(2026-05-29)）。重量が算定できない場合（`ΣiW<=0`）は幾何重心へフォールバックする。
  代表節点は要素が接続しない浮遊節点のため、剛床が拘束しない Uz/Rx/Ry を固定する
  （Ux/Uy/Rz は自由）。質量は常に `None`（要素・節点側の質量が Reducer の TᵀMT 縮約で
  自動的にマスターへ集約されるため、代表節点自体に持たせると二重計上になる）。
  当該階の全構造節点（1 点のみでも）がスレーブになる。
- `Story.diaphragms`（地震力の作用点）と `Constraint::RigidDiaphragm`（Reducer が読む拘束）の
  **両方**を生成する（Reducer は stories.diaphragms を読まないため、片方だけでは力の伝達か
  縮約が欠ける）。
- 再生成時は `model.generated_masters` に記録された既存の代表節点を階の下から順に再利用
  （座標・拘束・所属階を更新）し、足りない分だけ末尾連番で新規生成する。階数が減った場合の
  余った旧代表節点は `restraint=FIXED`・`story=None` で不活性化しつつ `generated_masters` には
  残す（次回再生成時の再利用に備える）。
- 地震重量 = 自重(ρ·A·L·g、両端 1/2 ずつ) + **先頭荷重ケース**の鉛直下向き荷重(節点配分)。
  簡易法であり、通常の「支配面積+仕上げ・積載の地震用低減」より粗い。
- `ApplyStories` コマンドで undo 可能に。既存 RigidDiaphragm は置換、Mpc/RigidLink は保持。
  代表節点（`rep_nodes`）は ID＝配列インデックス不変条件を保って置換または追加し、
  `model.generated_masters` を更新する。逆操作 `RestoreStories` は `model.nodes` を丸ごと
  復元することで、追加された代表節点の除去（truncate）も含めて完全に元へ戻す。

### 2.3 ソルバー安定化(squid-n-solver / squid-n-math)
- `Analysis::prepare` の冒頭で `Model::validate` + `precheck_model` を実行し、
  「支点がない」「断面/材料未割当(部材 ID 列挙)」「孤立節点」を実行前に日本語で報告。
  Cholesky 失敗(特異)時は原因候補を含む診断文に変換。
- `SolveError` に `InvalidInput`/`NonConvergence` を追加。既存の呼び出し側は
  `Display` 経由で表示しているため互換。
- 固有値: 質量トレース≤0 でエラー、部分空間反復が 200 回で収束しなければエラー
  （従来は**収束していなくてもそのまま結果を返していた**）。
- `DirectSparseLu` を faer の sparse LU で実装（`todo!()` panic を除去）。

### 2.4 梁の一貫質量行列バグ修正(squid-n-element/beam.rs) ← 重要
- 曲げブロックを書き込む閉包が DOF [1,5,7,11]/[2,4,8,10] ではなく**連続添字
  [1..4]/[2..5] に書いていた**。回転-回転項が欠落し、誤った位置に連成項が入り、
  一貫質量行列が不定値になっていた。
- 症状: 時刻歴解析の有効剛性 K̂=K+c₂C+c₁M の Cholesky が必ず失敗。
  固有値解析(一貫質量使用)の周期・刺激係数も歪んでいたはず。
- 修正後は M が正定値になり全テスト緑。**既存の固有値検証値がある場合は再確認を推奨**
  （ワークスペース内テストは全て通過している）。

### 2.5 時刻歴(timehistory.rs)
- `ResponseResult` に UI 用の代表応答 `history`（記録節点の X 変位・ベースシア・
  最上階層間変形角、全ステップ）を追加。記録節点は「X 方向自由度を持つ最高標高の節点」。
- ベースシア V(t) = −(rᵀM·a_rel + ẍg·rᵀMr)。X 方向のみ。
- 初期加速度 M·a₀=rhs は Cholesky→LU→(rhs≈0 なら a₀=0)の順でフォールバック。
  地震波の先頭が非ゼロかつ M 特異の場合のみエラー。
- 減衰は UI からは「1 次固有円振動数に対する剛性比例(h 指定)」のみ。Rayleigh は
  ソルバー実装済みだが UI 未配線。

### 2.6 GUI
- 解析は **UI スレッドで同期実行**（進捗バーなし）。小規模モデルでは問題ないが、
  大規模モデルでは UI がブロックする。P8 の非同期ジョブ化は未着手。
- プッシュオーバーは `model.clone()` に対して実行（非線形状態の副作用を GUI モデルに残さない）。
- 波形 CSV は「1 行 1 値(カンマ区切りなら最終列)、単位 gal(×10 で mm/s² に変換)、
  dt は UI の設定値」という簡易仕様。ヘッダ行は数値パース失敗として自動スキップされる。

## 3. 残課題（優先度順の提案）

1. **解析の非同期化+進捗表示**（P8）: 現状 UI 同期実行。時刻歴の長波形でフリーズ感が出る。
2. **CMQ 図のデータ供給**: `app.beam_loads` に書き込む経路がなく、CMQ 表示は常に空
   （床荷重→CMQ の `squid_n_load::floor` は実装済み。スラブ入力 UI がないため）。
3. **荷重組合せの UI**: `linear_combination` はソルバー側共通化済みだが、組合せの
   作成/編集/実行 UI が未配線。
4. **保有耐力比較・Ds の設計タブ統合**: プッシュオーバーの Qu 表示まで。
   `holding_capacity::check_holding_capacity`・`ds` との突合表示は未実装。
5. **階の手動編集**: 自動生成のみ。階高・重量・所属節点の手動調整 UI がない
   （生成後に `seismic_weight` を編集する導線を推奨）。
6. **地震重量の精緻化**: 現状は自重+先頭荷重ケース。ケース選択 UI と
   積載荷重の地震用低減係数の導入余地あり。
7. **時刻歴の Y 方向・2 方向入力**: `GroundMotion.accel_y` はソルバー対応済み、UI 未対応。
   記録応答も X 方向のみ。
8. **固有値検証値の原典照合**: 質量行列修正(2.4)により固有周期が変わった可能性がある。
   `specs/原典照合リスト.md` の該当項目があれば再照合を推奨。
9. **HHT-α / 非線形時刻歴の UI 配線**: ソルバー実装済み・未配線。
10. **レポートの PDF/Excel 化**(P9)・**ST-Bridge**・**GPU/ML/P12** は元計画通り未着手。

## 4. 既知の制約・注意点

- 節点荷重の編集対象ケースは「荷重ケース表でクリック選択したケース」。未選択時は
  最後に実行したケース→先頭ケースの順でフォールバックする。
- `LoadCaseId(0)` は地震静的の結果格納にも使われる（既存仕様を踏襲）。ユーザー作成の
  ケース 0 と結果表示上は同居するため、将来は結果キーを (種別, ID) に分けるのが望ましい。
- 材料の鋼/RC 判定は名称先頭文字（SS/SN/SM/STK/ST）による簡易判定のまま
  （`Material` に種別フィールドを足すのが本筋）。
- プリセット材料の密度は ton/mm³（内部単位系 N-mm-s）: 鋼 7.85e-9、RC 2.4e-9。
- rfd(ファイルダイアログ)は `gui` フィーチャに含めた。ヘッドレス CI では
  ダイアログ関数は呼ばれない（テストはパス直指定の `save_project_to` 等を使用）。

# 申し送り追記（断面検定の RESP-D マニュアル準拠化）

追記日: 2026-07-10
対象ブランチ: `claude/resp-cross-section-calc-24ds3o`
参照マニュアル: RESP-D 操作・計算マニュアル 計算編「04. 断面検定（許容応力度検定）」
（https://so-suzuki-kke.github.io/resp-docs/calculation2/04_cross_section_test/index.html）

## 1. 実装した内容

一次設計（許容応力度検定）を RESP-D マニュアル 04 章準拠に全面書き換えた。

### squid-n-design-jp
- 共有型を拡張: `MemberForcesAt` を二軸内力（n/qy/qz/my/mz、n は**引張正**）に、
  `DesignCtx` に部材種別（柱/梁/ブレース）・部材長・座屈長さ lk/横座屈長さ lb・
  せん断スパン比代表値・RC 短期せん断の損傷制御/安全確保フラグを追加。
- `steel.rs`（旧 allowable_stress.rs を置換）:
  - F 値表を板厚区分対応（40mm 超は低減、SM520 は 3 区分）に修正。
    **SS490=275**（旧実装 285 は誤り）、STKR/BCR/BCP/SA440 追加、
    マニュアルに無い SN550/SM570 は削除。
  - 座屈を考慮した許容圧縮 fc（λ/Λ 分岐・ν=3/2+(2/3)(λ/Λ)²）、
    横座屈を考慮した H 形強軸の許容曲げ fb=max(fb1,fb2)≤ft を実装。
    **モーメント修正係数 C は端部モーメント比未伝達のため C=1.0（安全側）固定**。
  - S 梁: σb/fb と、H 形はウェブの von Mises 照査 √(σb′²+3τ²)/ft・τ/fs。
  - S 柱: σc/fc + σbX/fb + σbY/fb ≤ 1（角形・円形は fb=ft、円形は二軸合成 σb）。
  - 鉄骨ブレース: 軸力のみ（圧縮は fc、引張は ft）。
- `rc.rs`:
  - コンクリート許容せん断を RC 規準 2010 の min(Fc/30, 0.49+Fc/100)・短期 1.5 倍に修正
    （旧実装は令 91 条の Fc/20・短期 2 倍）。断面算定用ヤング係数比 n 表を追加。
  - 鉄筋許容応力度表（SR235〜SD490、D29 以上の長期低減、短期=F 値）を実装。
  - RC 梁曲げ: MA=min(at·ft·j, 圧縮側支配 MA_c)（RC 規準 13 条の pt≤ptb 分岐と等価、
    中立軸はひび割れ換算断面から解く）。
  - RC 梁せん断: QA=b·j(α·fs+0.5·w_ft(pw−0.002))、α=4/(M/(Qd)+1)∈[1,2]、
    pw 上限 0.6%(長期)/1.2%(短期)、短期は損傷制御(2/3α)/安全確保を選択可。
  - RC 柱: 許容軸力 NA=min(fc·Ae, ft·Ae/n)、RC 規準 14 条の 3 支配条件による
    M-N 包絡線（中立軸スキャン+線形補間）、二軸は矩形=線形和/円形=2乗和、
    せん断は α∈[1,1.5]・柱の安全確保式は α 無し。円形柱は b=(D/2)√π の等価矩形。
- 配筋・板厚は `Section.shape`（SectionShape）から取得。shape 無しの鋼断面は
  断面名（"H-"/"BOX-"/"PIPE-"）+ thickness 単一板厚で近似、RC は検定スキップ。

### パイプライン（squid-n-load / element / app / mcp）
- 荷重組合せに地震の負側加力 G+P−Kx/−Ky を追加（正負両加力）。
  組合せ名から長期/短期を自動判定（`is_short_term_combo`）し検定に連動。
- **梁要素の軸力回復バグ修正**: 旧実装は両端節点力の線形補間で部材中央 N=0 と
  なっていた。引張正の部材内力（端別採用）に統一。N 図・RC σ0 算定も新規約に更新。
- app/mcp の検定実行を部材種別判定（軸の鉛直成分: ≥0.8 柱 / ≤0.2 梁 / 中間 ブレース）・
  二軸内力・せん断スパン比（|M|最大位置の M,Q）対応に書き換え。

## 2. マニュアルからの簡略化（既知の相違）

- 横座屈の修正係数 C=1.0 固定（DesignCtx に端部モーメント比が無い）。
- 柱の座屈長さ係数 K は未実装（lk=部材長=K:1.0。マニュアルは鋼構造塑性設計指針の
  G_A/G_B 式）。lb も横補剛データが無いため部材長。
- 採用応力のモーメント 2 次曲線分布・一本部材指定・T 形梁の協力幅・2 段筋の
  個別重心入力は未実装（評価位置は eval_sections の解析値をそのまま使用）。
- RC 柱の直交筋は断面中央集約（マニュアル準拠）だが、隅角部主筋の二方向重複は
  未控除。円形柱の MA は等価矩形近似。
- 損傷制御/安全確保は `rc_damage_control` フラグ（既定: 損傷制御）で、UI 未配線。

## 3. スコープ外（未実装のまま）

SRC 造（梁・柱・耐震壁・パネル）、CFT 柱、冷間成形角形鋼管の耐力比チェック、
RC 耐震壁・柱梁接合部・付着・水平接合面の検定、高強度せん断補強筋の製品別式、
大梁必要横補剛数・たわみ検定、継手/スカラップ断面欠損、座屈補剛ブレース製品対応。

# 申し送り追記2（断面検定の残項目実装: SRC/CFT/接合部/耐震壁/高強度筋/付着ほか）

追記日: 2026-07-10（同日追記）

前回「スコープ外」とした項目のうち、資料（マニュアル04章）が揃っているものを
すべて実装した。**資料不足で残るのは「JFEシビル/日鉄エンジニアリングの
座屈補剛ブレース」のみ**（製品固有の限界座屈長の数表がマニュアルに無い）。

## 実装済み（追加分）

- **SRC梁・柱**（src_cft.rs、SRC規準1987・単純累加/累加強度式）:
  MA=sZ·s_ft+at·ft·j、せん断の弾性分担（sQ=sZ/(sZ+at·rj)·Q）と
  sQA=tw·dw·s_fs / rQA=min(rQA1,rQA2)、柱は fc′=fc(1−15·s_pc) と
  中立軸スキャンによる rM(N)+鉄骨累加の3分岐、二軸線形和。
  断面は SectionShape::SrcRect（RC矩形+内蔵H形鉄骨、steel_grade保持）
- **CFT柱**（src_cft.rs）: 矩形充填部は閉形式（cN/cM の Xn パラメータ式）、
  円形は数値積分。鋼管累加 sM=sZ(s_fc−sN/sA)、せん断 sQa=sAw·s_fs
- **RC柱梁接合部**（joint.rs）: QAj=κA(fs−0.5)bj·D（κA=10/7/5/3）、
  Qdj=min(ΣM/j·(1−ξ), QD(1−ξ)/ξ)。有効幅はRC規準15条の min(bi/2,D/4)
  （マニュアル抽出テキストの「大きい方」表記は原文誤記の可能性、安全側を採用）
- **S造パネルゾーン**（joint.rs）: pM=bML+bMR−(cQU+cQL)db/2、
  pMy=Ve·κ√(1−n²)Fy/√3。κ式はMathML崩れのため分数2項和として再構成
- **冷間成形角形鋼管の柱梁耐力比**（joint.rs）: ΣMpc≧Σmin(1.5Mpb,1.3Mpp)、
  ν・Mppの軸力比分岐。NGでも耐力低減はしない（マニュアル通り、表示のみ）
- **RC耐震壁**（joint.rs + SectionShape::RcWall）: Qa=max(Q1,Q2)、
  Qw=ps·t·le·w_ft、側柱 Qc、開口低減 r=min(γ1,γ2,γ3)（配線上は無開口 r=1）
- **高強度せん断補強筋**（rc.rs、ShearBar.grade）: 製品別 w_ft（585/590）、
  pw 上限表、(pw−0.001) 暫定対応式。製品固有の精算式（ウルボン1275の√式、
  KH785のβc式等）は未実装（マニュアル自身も「暫定対応」と記載）
- **RC梁付着**（rc.rs、RC規準1999）: ldb=σt·As/(K·fb·φ)、
  K=0.3(C+W)/db+0.4≦2.5、通し筋仮定（カットオフ位置データがモデルに無い）
- **鋼の横座屈修正係数C**（steel.rs）: 端部モーメント比から算定（≦2.3、
  複曲率正/単曲率負、中央大でC=1）。前回の C=1.0 固定を解消
- **大梁必要横補剛数・たわみ**（steel.rs）: n=(170−λy)/20 等、
  S=5M0l²/48EI−(ML+MR)l²/16EI（長期のみ・情報出力）
- **節点単位検定の配線**（joint_wiring.rs + app/mcp/design_view）:
  接合部形状の自動判定（上下柱の有無×梁本数）、耐震壁の側柱自動収集、
  設計タブ「接合部・耐震壁の検定」表と MCP サマリ（n_joint_checks/n_joint_ng）

## 残る簡略化（実装済み項目の中の相違点）

- SRC の短期設計用せん断力のヒンジ考慮式（rQD1=rQL+(rM1+rM2)/l′ 等）は
  弾性分担で代替。SRC/CFT の鋼材許容圧縮は座屈無視（s_fc=s_ft）
- 冷間耐力比の存在軸力は当該ケースの軸力（NL+1.5·NE の割増未対応。
