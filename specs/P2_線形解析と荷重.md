## 0. このフェーズについて

### 0.1 目的

**単一モデル**の上で、線形静的解析と振動（固有値）解析が**シームレスに走り**、日本の地震荷重
（Ai分布）と床荷重を自動生成して解析へ流し込むところまで作る。設計書 §9 / §10.1–§10.4 / §12.3。

- **Analysis API**：複数荷重ケース・組合せの線形静的を、分解1回・後退代入多数で高速処理（§10.2）。
- **固有値解析** `Kφ=λMφ`：**部分空間反復法**（下位モード）。分解は **P0 の SPD Cholesky を再利用**。
- **squid-n-load**：床面荷重→部材荷重の自動分割（小梁考慮・CMQ図）、Ai分布（略算＋準精算）。
- **一気通貫**：固有値→実固有周期 T→Ai分布→静的解析への水平力投入を自動接続（設計書 §9.3 の核心）。
- **結果I/O**：Parquet で結果を列指向格納（ストリーミング書込・部分読出し）。

### 0.2 完了像（ゴール）

設計書 §18 P2 完了基準「**単一モデルで静的＋振動／告示Ai一致**」を次で判定:

1. **線形静的**：単純梁・ラーメンで変位・応力が手計算と一致（厳密、相対 1e-9）。複数荷重ケース・
   組合せが1回の分解で解ける（§7.1）。
2. **固有値**：等価せん断モデルの固有周期が理論値に**収束**（反復法なので許容差判定）。有効質量比
   合計が（十分なモード数で）≈100%（§7.2）。
3. **告示 Ai**：Ai 式の自己整合（計算例）と、**出典告示の計算例**との Ci・Qi 一致（§7.4）。
4. **CMQ・荷重保存**：三角／台形分配の総和が床総荷重に一致（厳密）、CMQ（固定端モーメント・せん断）が
   理論値と一致（§7.3）。
5. **一気通貫**：固有値由来 T で Ai が変わり、その水平力で静的解析が走る（§7.5）。
6. **結果I/O**：静的・モーダル結果を Parquet にストリーミング書込し、特定ケース・部材を部分読出し（§7.6）。
7. 決定性：同一入力・単一スレッドでビット一致（P0/P1 踏襲）。**ただし反復解法（固有値）の
   「理論への一致」は許容差判定**で、「実行間ビット一致」とは別概念（§7.2 の注意）。

### 0.4 重要な技術決定（書き始める前に確認した事項）

- **固有値は部分空間反復法（subspace iteration）を採用。** 理由：用途は「下位〜30モード」（§1.3）で、
  この解法は **K 単独を分解**（`K X_{k+1} = M X_k` を反復）するため **K は SPD のまま＝P0 の Cholesky を
  そのまま使える**。一方 **shift-invert Lanczos は `(K − σM)` を分解するが、これは対称*不定*** になり、
  P0 が見送った LDLT/Bunch-Kaufman が必要になる。内部モードが要るまで shift-invert は**見送る**。
- **Ai 精算は「準精算」まで。** SRSS/CQC のモード重ね合わせは、刺激係数・組合せ規則を早期に持ち込み、
  検証が時刻歴包絡（P6）に依存するため**見送る**（ユーザ確認済み）。
- **Parquet の DoD は P2 が実際に生む結果（静的＋モーダル＝小規模）に合わせる。** §12.6 の
  「1GB級→100ms部分読出し」は時刻歴（P6）のスケール。**機構（ストリーミング書込・部分読出し）は作るが、
  テストは P2 規模**で行う。

## 2. T0: 質量行列（集中／整合）

設計書 §6.1（`mass_matrix`）/ §10.4。P1 で雛形だった `ElementBehavior::mass_matrix` を実装する。

**集中質量・整合質量の明示式（閉形式。文献転記不要）。** `m = ρ·A·L`（要素全質量）。

```
■ 集中質量（対角）: 並進 6 自由度（ux,uy,uz × 両端）に m/2 ずつ。回転 3×2 は 0。
    M_lumped = diag(m/2, m/2, m/2, 0,0,0,  m/2, m/2, m/2, 0,0,0)

■ 整合質量（曲げ。Bernoulli 整合質量。DOF 並びは §4.1 の梁と同じ）:
    曲げ面（例 uy,rz）の 4×4 ブロックは (ρAL/420)·
        [ 156    22L     54    −13L  ]
        [ 22L    4L²     13L   −3L²  ]
        [ 54     13L     156   −22L  ]
        [ −13L   −3L²    −22L   4L²  ]
    軸（ux_i,ux_j）: (ρAL/6)·[[2,1],[1,2]]
    ねじり（rx）: (ρ·J/A·L/6)·[[2,1],[1,2]]（極慣性。簡易には集中可）
    xz 面（uz,ry）も同じ 4×4（符号は §4.1 の曲げ規約に合わせる）
```

```rust
// squid-n-element：BeamElement / ShellElement に実装
impl ElementBehavior for BeamElement {
    fn mass_matrix(&self, opt: MassOption) -> LocalMat {
        let m = /* ρ */ self.density() * self.a * self.length; // 要素全質量
        let mut mm = LocalMat::zeros(12);
        match opt {
            MassOption::Lumped => {
                for d in [0,1,2,6,7,8] { mm.set(d, d, m/2.0); } // 並進のみ、回転 0
            }
            MassOption::Consistent => {
                // 上の (ρAL/420) 係数の 4×4 を uy-rz, uz-ry に、(ρAL/6) を軸・ねじりに配置
                // （DOF 添字は §4.1 と同一。各項を set で埋める）
            }
        }
        mm
    }
}
```

### 2.1 集中質量が M を特異にする点（★固有値に直結）

- 集中質量は**回転自由度の質量が 0**。よって全体 M は**特異**（質量ゼロの自由度を持つ）。
- **部分空間反復法では扱える**：`M·X_k` は質量ゼロ自由度の右辺成分が 0 になるだけで、分解する K は
  SPD のまま。ただし:
  - **意味のあるモード数は「非ゼロ質量自由度の数」で上限**が決まる。
  - Rayleigh–Ritz で **M ノルムがゼロの Ritz ベクトルをガード**する（質量ゼロ方向に偽モードを出さない）。
- 整合質量は SPD（特異でない）。集中質量で問題が出る場合の代替として用意（`MassOption`）。

**DoD（T0）:** 1要素の集中質量の総和（並進）が `ρ·A·L`（梁）/ `ρ·t·面積`（シェル）に一致。
整合質量が対称・正定値。集中質量行列が対角。

## 4. T2: 固有値解析（部分空間反復法）

設計書 §10.4。一般化固有値問題 `K φ = λ M φ` の**下位 n モード**を解く。

> **アルゴリズムは下記 §4.1 に手順を明示済み（写経可）。** 収束証明・派生（シフト等）の詳細のみ
> **K.-J. Bathe, *Finite Element Procedures*** を参照。実装は §4.1 の手順＋§8.2 の固有周期で照合する。

### 4.1 アルゴリズム（部分空間反復法）

> **⚠ M も K と同じ縮約空間に置く（必須・正確性に直結）:** 固有値問題は **縮約後の自由度空間**で
> `K_red φ = λ M_red φ` として解く。`K_red = TᵀKT` と同様に **`M_red = TᵀMT`** を作ること。
> とくに**剛床ダイアフラム下では、`TᵀMT` がスレーブ節点の並進質量をマスター節点へ移す**働きをする。
> これを省くとモード質量・**有効質量比（DoD #2）が物理的に誤る**（単なるラベル違いではない）。
> 実装：M を組み立てる `assemble_global_m`（K と同型・集中/整合）を追加し、**K と同じ `Reducer` に通す**。
> `Reducer` には `reduce_m`（= `reduce_k` と同一の `TᵀXT` トリプレット変換。対称疎行列なら何でも可）を足す。

```
入力: K_red (SPD, 縮約済み), M_red (= TᵀMT, 縮約済み), モード数 n, 部分空間サイズ q (= min(2n, n+8) 程度)
0. K_red を分解（P0 の DirectSparseCholesky を再利用 ← K_red は SPD なのでそのまま）
   ※ M_red = TᵀMT を K と同じ Reducer で事前に構築しておく
1. 初期部分空間 X0 (active×q) を用意（ランダム or 質量集中自由度に単位ベクトル）
2. 反復 k=0,1,...:
   a. K Y = M X_k を解く（前進後退代入のみ。q 本の右辺）
   b. 縮約: K* = Yᵀ K Y,  M* = Yᵀ M Y     （q×q の小行列）
   c. 小さい一般化固有値問題 K* z = θ M* z を密ソルバ（faer）で解く（Rayleigh–Ritz）
   d. X_{k+1} = Y z       （Ritz ベクトル）
   e. 収束判定: 下位 n 個の θ の相対変化 < tol で停止
3. λ_i = θ_i, φ_i = X 列。固有周期 T_i = 2π/√λ_i
```

### 4.2 刺激係数・有効質量（設計書 §9.3）

- 各モードの刺激係数 `βi = (φiᵀ M r) / (φiᵀ M φi)`（`r` は方向影響ベクトル）。
- 有効質量 `Meff,i = (φiᵀ M r)² / (φiᵀ M φi)`。全モード和は**全質量**（十分なモード数で ≈100%）。

### 4.3 DoD（T2）— ★「理論一致(許容差)」と「実行間ビット一致」を区別

- **理論一致（許容差）:** 等価せん断モデルの固有周期が理論値に**収束**（反復法なので相対 1e-8 程度）。
  - 1自由度: `T = 2π√(m/k)`。例 `k=1000 N/mm, m=1.0 N·s²/mm → T = 2π√(1/1000) = 0.198692 s`。
  - 2層等質量等剛性せん断モデル: `K=[[2k,-k],[-k,k]], M=mI`。
    `λ = (k/m)·(3∓√5)/2`。例 `k=1000, m=1`：`λ1=382.0, T1=0.32150 s`／`λ2=2618.0, T2=0.12280 s`。
    2モードで**有効質量比合計＝100%**（モード1≈91.1%, モード2≈8.9%）。
- **実行間ビット一致（決定性）:** 同一入力・単一スレッドで `eigen` を 100 回 → **同一結果**
  （初期部分空間を固定シードにする）。※ これは「理論への一致」とは別物（反復の収束結果が
  毎回同一バイト、という意味）。
- **縮約空間の確認:** 固有値は `K_red φ = λ M_red φ` を解いていること。**M が K と同じ `Reducer`
  を通っている**こと（剛床ありモデルで、剛床なし等価モデルと固有周期・有効質量比が一致するかで検証）。

### 5.1 床荷重分割（floor.rs）— 設計書 §9.1

解析モデルは柱・大梁まで（小梁は剛性に入れない）。スラブ（P0 §4.7 の `Slab`：範囲＋小梁配置）を
使って面荷重を大梁へ流す。**2段階**:

1. 面荷重 → 小梁へ（小梁が無い領域は直接大梁へ）。
2. 小梁反力 → 支持する大梁へ（集中／分布荷重）。

分配方式（スラブごと `DistributionMethod`）:
- **三角形・台形（45°）**：二方向スラブ。短辺方向は三角形、長辺方向は台形。
- **一方向**：一方向スラブ。
- **負担面積法**：面積按分。

**固定端モーメント FEM の閉形式（両端固定梁・スパン L。文献転記不要）:**

```
■ 等分布 w（一方向スラブ・全長）:          FEM = w·L²/12,        端せん断 Q = w·L/2
■ 対称三角形（ピーク w0 中央、二方向の短辺側）: FEM = 5·w0·L²/96,    Q = w0·L/4
■ 対称台形（中央が等高 w0、両端 a 区間で0→w0 に立上り、b=L−2a）:
     FEM = (w0/12)·(L² − ... )  ※台形は「三角形2つ＋中央等分布」の重ね合わせで合成する
     （実装は基本形＝等分布・三角形の FEM を区間ごとに重ね合わせて算定）
```

**CMQ の数値例（§8.4・テスト用）:** 正方形スラブ 1辺 `a=4000`, 等分布面荷重 `w=0.005 N/mm²`、
二方向（三角形分配）。ピーク強度 `w0 = w·a/2 = 10 N/mm`、`FEM = 5·w0·a²/96 = 5·10·4000²/96 =
8.3333e6 N·mm`。総和保存：`Σ部材荷重 = w·a² = 80000 N`。

### 5.2 Ai分布（ai.rs）— 設計書 §9.2/§9.3

**告示の式・値（昭和55建告1793号。法令＝公有。下記をそのまま実装）。** 調べ直す不要:

```
■ Ci = Z · Rt · Ai · C0      （C0：一次設計=0.2、二次設計=1.0）

■ Ai（高さ方向分布、告示式）:
    Ai = 1 + (1/√αi − αi) · 2T/(1+3T)
    αi = (i 階以上の地震用重量の和) / (全地震用重量)

■ Rt（振動特性係数、告示1793 第2。Tc=地盤種別の固有周期）:
    T < Tc            : Rt = 1.0
    Tc ≤ T < 2Tc      : Rt = 1 − 0.2·(T/Tc − 1)²
    2Tc ≤ T           : Rt = 1.6·Tc / T

■ Tc（地盤種別ごとの値[秒]）:  第1種 = 0.4,  第2種 = 0.6,  第3種 = 0.8

■ T（設計用一次固有周期。略算）:  T = h·(0.02 + 0.01·αs)
    h = 建物高さ[m]、αs = 鉄骨造部分の高さ比（RC造は αs=0）
    （準精算では固有値解析の実 T を代入。略算式の T をこれで置換）

■ Z（地域係数。取りうる値）:  1.0 / 0.9 / 0.8 / 0.7（沖縄県）
```

use parquet::arrow::arrow_writer::ArrowWriter;
use parquet::file::properties::WriterProperties;
use arrow::record_batch::RecordBatch;
use arrow::datatypes::SchemaRef;
use std::{fs::File, sync::Arc};

// --- ストリーミング書込（ResultWriter 実装）---
pub struct ParquetWriter { inner: ArrowWriter<File> }
impl ParquetWriter {
    pub fn create(path: &str, schema: SchemaRef) -> parquet::errors::Result<Self> {
        let file = File::create(path).unwrap();
        let props = WriterProperties::builder()
            .set_max_row_group_size(64 * 1024)   // RowGroup の行数上限（部分読出しの粒度）
            .build();
        // 確定 API: ArrowWriter::try_new(writer, schema, Some(props))
        Ok(Self { inner: ArrowWriter::try_new(file, schema, Some(props))? })
    }
}
impl super::results::ResultWriter for ParquetWriter {
    fn write_rows(&mut self, batch: &RecordBatch) { self.inner.write(batch).unwrap(); } // バッチ追記
    fn finish(self: Box<Self>) { self.inner.close().unwrap(); }                          // フッタ確定
}
// 必要に応じ inner.flush() で早期に RowGroup を切り出す。

// --- 部分読出し（特定 RowGroup・特定列のみ）---
use parquet::arrow::arrow_reader::ParquetRecordBatchReaderBuilder;
use parquet::arrow::ProjectionMask;
pub fn read_partial(path: &str, row_groups: Vec<usize>, columns: Vec<usize>)
    -> parquet::errors::Result<Vec<RecordBatch>>
{
    let file = File::open(path).unwrap();
    let builder = ParquetRecordBatchReaderBuilder::try_new(file)?;
    let mask = ProjectionMask::roots(builder.parquet_schema(), columns); // 列射影
    let reader = builder
        .with_row_groups(row_groups)   // 必要な RowGroup だけ読む（ステップ範囲→RowGroup に対応づけ）
        .with_projection(mask)         // 必要な列だけ読む（部材/節点フィルタ）
        .build()?;
    reader.collect::<Result<Vec<_>, _>>().map_err(Into::into)
}

## 8. 数式・理論補足（ジュニア向け）

### 8.1 質量行列（→ §2）
- **集中質量**：要素質量を端部並進へ配分した対角行列。簡便・高速だが回転質量 0 で M が特異。
- **整合質量**：剛性と同じ形状関数から作る密行列。SPD。精度が高いが計算量増。

### 8.2 一般化固有値と部分空間反復（→ §4）
振動は `K φ = ω² M φ`（`λ=ω²`）。固有周期 `T=2π/ω`。下位モードだけ要るので、q 次元の部分空間で
`K Y = M X` を反復し、小さな q×q 問題に縮約（Rayleigh–Ritz）して下位 n 個へ収束させる。**K は SPD のまま
分解**できるのが利点（P0 Cholesky を再利用）。反復解法なので「理論への一致」は許容差で見る。

### 8.3 Ai 分布（→ §5.2）
地震層せん断力係数の高さ分布。`Ci = Z·Rt·Ai·C0`。`αi` は i 階以上の重量比で、上階ほど割増（Ai>1）。
`Ai = 1 + (1/√αi − αi)·2T/(1+3T)`。T が大きいほど上階割増が強い。略算は T を略算式で、準精算は
固有値解析の実 T を使う（同じ式に、より正確な T を入れる）。

### 8.4 CMQ（荷重項）と固定端モーメント（→ §5.1）
部材荷重による両端の固定端モーメント C/M とせん断 Q。床の二方向分配では、各大梁は三角形／台形の
分布荷重を受ける。対称三角形荷重（ピーク `w0` が中央）の固定端モーメントは `FEM = 5 w0 L² / 96`
（§5.1 に閉形式を明示済み：等分布 wL²/12・対称三角形 5w0L²/96）。総和は必ず床総荷重に一致する
（保存則＝最良の検算）。

### 8.5 三角形・台形分配（45°法）（→ §5.1）
二方向スラブは、各辺から 45°線を引いて床を4分割し、短辺側は三角形、長辺側は台形の荷重として
隣接大梁へ流す近似。正方形なら4辺すべて三角形。面積（＝荷重）の総和は床全体に一致する。
