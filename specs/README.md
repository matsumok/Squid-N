# 実装仕様書（フェーズ別）

このディレクトリは、ルートの『建築構造計算一貫プログラム 実装設計書（開発指示書）』
（`../構造計算一貫プログラム_実装設計書.md`）をもとに、**フェーズ単位**で
「ジュニアエンジニアがそのまま着手できる」粒度に落とした実装仕様書を置く場所です。

- 親文書（設計書）= **何を作るか・なぜそう作るか**（方針・確定事項）
- 本ディレクトリ（仕様書）= **どう作るか**（ファイル構成・型シグネチャ・実装手順・テスト・DoD）

仕様書中で `設計書 §4.2` のように参照している番号は、すべて親設計書の章番号です。

## 仕様書の読み方（ジュニア向け）

1. 各仕様書の「0. このフェーズについて」でスコープ（含む／含まない）を必ず確認する。
2. 「1. タスク一覧と依存順序」の順に着手する。前のタスクが終わらないと次に進めない。
3. 各タスクの **実装ステップ** を上から実施し、**DoD（受け入れ条件）** を満たしたら完了。
4. コード断片は「雛形」。シグネチャ（型・関数名・引数）はそのまま使い、中身を実装する。
5. `// 要検証` 注記がある外部ライブラリ呼び出しは、必ず該当バージョンの docs.rs で
   最新APIを確認してから書く（メモリや本書の断片を鵜呑みにしない）。
