## 0. このフェーズについて

### 0.1 目的

P5 で見送った**特殊な非線形要素**を追加する。設計書 §6.4 / §6.6 / §20-risk1。

- **MS（マルチスプリング）要素**：RC 柱・梁端部の塑性ヒンジを**軸ばね群**で表現。N–M 相関を自然に表現。
  ファイバ要素（P5）との**相互検証**で妥当性を担保。
- **壁谷澤 TVLEM（Three Vertical Line Element Model）**：連層耐震壁を**マクロ要素**で置換。

> **なぜ P5 から分離したか:** どちらも**実験照合が妥当性の前提**の特殊要素。P5 の目玉（架構の弾塑性・
> 終局機構）はファイバ＋集中ばねで達成済み。MS は「ファイバとの相互検証」が DoD なのでファイバが先に
> 必要。壁は「採用文献の確定」「係数外部化」「実験照合」が必須（§20-risk1）で重い。

### 0.2 完了像（ゴール）

設計書 §6.4 / §6.6 の DoD:

1. **MS**：一定軸力下の繰り返し曲げで、N–M 相関・履歴ループが**ファイバ要素と整合**（相互検証。§5.1）。
2. **壁**：既往の壁実験（または商用ソフト結果）に対し、初期剛性・降伏点・最大耐力・履歴形状が
   許容差内（§5.2）。

## 3. T1: MS（マルチスプリング）要素

設計書 §6.4。主に **RC 柱・梁端部の塑性ヒンジ**表現。

- 部材端の断面を **軸方向ばねの集合**に置換し、中央は弾性材で連結（two-component／端部ばね型）。
- 各ばねは**軸力負担位置（断面内座標）**を持ち、軸方向の非線形履歴則（コンクリばね・鉄筋ばね、P4）に従う。
- **曲げと軸力の相互作用（N–M 相関）を自然に表現**できるのが利点（軸ばね群の合力＝N、偶力＝M）。
- せん断ばね・ねじりばねは**別系統**で付加。
- 履歴状態は `ElemState`／材料（P4）側。commit/rollback（P5 §6）に乗る。

```rust
// sc-element/src/ms.rs
pub struct MsElement {
    pub springs: Vec<AxialSpring>,   // (断面内座標 y,z, 材料)
    pub elastic_mid: BeamElement,    // 中央弾性部（P1）
    pub shear: ShearSpring,          // 別系統（P5）
}
pub struct AxialSpring { pub y: f64, pub z: f64, pub material: sc_core::ids::MaterialId }
/// 回転ばね（TVLEM 中央の曲げ用）。スケルトン／履歴は P4 から供給。
pub struct RotSpring { pub skeleton: sc_skeleton::MemberSkeleton }
impl ElementBehavior for MsElement { /* 軸ばね群の積分で N–M・接線、commit/rollback */ }
```

> **自動選択（P5 §5）との関係:** `AxialBendingInteract` 部材の既定はファイバ。MS は**選択肢**
> （`ForceRegime` 上書き or 明示指定）として割当可能にする。

**DoD（T1）:** 一定軸力の繰り返し曲げで、N–M 相関・履歴ループが**ファイバ要素（P5）と整合**（相互検証）。

## 4. T2: 壁谷澤 TVLEM（連層耐震壁マクロ要素）

設計書 §6.6。連層耐震壁を**マクロ要素**で置換する。**採用文献（§2）に厳密準拠**。

TVLEM の構成（採用文献で確定した配置・式を使う。下は一般形）:

- 両側柱位置に **2本の軸ばね**（境界部の軸変形・ロッキング）。
- 中央に **軸ばね＋回転ばね**（曲げ）。
- 全体に **水平せん断ばね**。
- 上下端を**剛梁**で連結し、**回転中心高さ比 c**（多くは 0.5 前後）を持つ。

各ばねの履歴則は P4（`sc-material`／`sc-skeleton`）から供給:
- せん断ばねは**原点指向型**、軸ばねは**軸力依存スケルトン**。

フレームへの組込み:
- 上下の剛梁端を周辺梁・床の節点へ**剛接続（リジッドリンク、設計書 §4.6＝P1 §9 で実装）**して行う。

```rust
// sc-element/src/wall_tvlem.rs
pub struct TvlemWall {
    pub axial_side: [AxialSpring; 2],   // 両側柱
    pub axial_center: AxialSpring,
    pub rot_center: RotSpring,
    pub shear: ShearSpring,
    pub c: f64,                          // 回転中心高さ比（外部設定）
    pub rigid_beams: [NodeId; 4],        // 上下剛梁の取付節点
}
impl ElementBehavior for TvlemWall { /* 各ばね合成＋剛梁拘束、commit/rollback */ }
```

**DoD（T2）:** 既往の壁実験／商用ソフト結果に対し、初期剛性・降伏点・最大耐力・履歴形状が許容差内
（**参照は技術リード提供**。R4/R23。設計書 §6.6）。

## 6. 数式・理論補足（ジュニア向け）

### 6.1 マクロ要素という考え方（→ §4）
壁全体を多数の有限要素で細かく分割すると重い。TVLEM は壁1枚を「数本の鉛直ばね＋せん断ばね＋剛梁」の
**少数自由度のマクロ要素**に置き換え、曲げ（両側軸ばねの伸縮差）・軸（軸ばね合力）・せん断（せん断ばね）・
ロッキング（回転中心 c）をまとめて表す。実挙動に合わせるには係数（スケルトン・c）の校正＝実験照合が要る。

### 6.2 MS の N–M 相関（→ §3）
断面を軸ばね群に置換すると、各ばねの軸力の**合力が N、図心まわりの偶力が M**。軸力が増えると各ばねの
状態が変わり M 耐力も変わる＝N–M 相関が自然に出る。ファイバ要素と原理が近く、相互検証できる
（ファイバは連続断面、MS は離散ばね）。

### 6.3 なぜ実験照合が必須か（→ §2,§5.2）
壁やマクロ要素は「現象を簡略化したモデル」なので、式が正しく解けている（Verification）だけでは不十分で、
実物の挙動を表しているか（Validation＝実験照合）を確認しないと使えない。だから採用文献の確定と
実験照合をフェーズの必須条件にする。
