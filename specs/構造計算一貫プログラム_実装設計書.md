## 改訂履歴・確定した主要決定（Draft 1.1）

レビュー（R1〜R30）の回答を反映。実装上の確定事項は以下。詳細は各章本文。

- **要素モデル（R1）:** core はデータ（`ElementData`）、squid-n-element は挙動（`ElementBehavior` トレイト）に分離。ファクトリで生成。→ 4.2 / 6.1
- **単位（R2）:** 内部は N・mm・s（質量 = N·s²/mm）。応力表記は **N/mm²**。入力は日本慣用（N/mm²・g/cm³ 等）で受け、境界で内部単位へ変換して保存。→ 4.1
- **階の定義（R3）:** **新規に「階（層）の定義」を追加**。剛床・Ai適用・層間変形・剛性率/偏心率の基盤。→ 4.5
- **拘束（R3）:** 要素内剛体／剛床ダイアフラム／**汎用多点拘束（1対多、柱–床接続用）** を変換行列法で統合。→ 4.6
- **非線形モデル自動選択（R5/R25）:** 剛床内で一軸曲げ＋せん断のみの梁は集中ばね、軸力–曲げ相関を受ける部材は **既定でファイバ**。要素の応力自由度属性（`ForceRegime`）で自動判定。→ 4.2 / 6.8
- **幾何剛性（R6）:** 線形化幾何剛性（P-Δ）を**任意機能**として実装。共回転定式化は不要。→ 10.6
- **減衰（R9）:** 質量比例／剛性比例／Rayleigh を UI 選択可。既定は **1次モード剛性比例（規定減衰 RC/S）**。→ 10.5
- **床荷重（R10）:** 解析モデルは柱・大梁まで。**スラブ範囲＋小梁配置**を入力実体として持ち、小梁を考慮して大梁へ分配。検証用に **CMQ図**を出力。→ 4.7 / 9.1 / 14
- **荷重方向（R12）:** 既定 XY±、オプション 45°。付加偏心は**不要**（日本法令で非要求）。
- **Ds（R14）:** 鉄骨は幅厚比による部材ランクと耐力比を併用して自動分類（ユーザ上書き可）。→ 11.1
- **ハンチ／テーパー（R16）:** 剛性は**一様断面として無視**（慣用）。剛域近似・可変断面剛性は将来オプション。→ 6.2
- **パネルゾーン（R18）:** v1 は**弾性のみ**。添付『パネルゾーンの力学』(小野瀬) の釣り合いを正準定式とし、フェースモーメント変換・パネルせん断 pQ・検算を実装。非線形パネルは将来。→ 6.7 / 11.1
- **限界耐力計算（R20）:** v1 対象外（v2）。GPU（R21）は f32 の高速スクリーニング／最適化用途。
- **計画（R22）:** v0.5 プロト → v1.0 製品 → v2.0 の3層ロードマップに再編。→ 18
- **性能基準・V&V（R4/R23）:** ベンチマークモデルと照合参照は技術リードが別途用意。本書の性能値はそれに対して定義。
- **軽微（R24・R26〜R30）:** 推奨デフォルトで確定（部材軸規約・柱=梁要素・ファイバせん断・.scz閾値・決定性範囲・MCP単一ライタ・準拠規準版）。本文へ反映済み。要変更があれば指摘のこと。

## 0. 本書の使い方

本書は「自前で全部書かない」ことを前提にした実装方針書です。各章の末尾に **【採用ライブラリ】** と **【自前実装】** を明記しています。原則として、

- **数値計算・線形代数・GUI・ML・GPU・MCP・直交化/固有値などの汎用基盤は既存クレートに委譲する。**
- **構造工学のドメインロジック（要素定式化・履歴則・断面算定・荷重変形特性の自動生成）は自前で実装する。** ここが製品の価値であり、他社ライブラリには存在しないため。

各担当者は、担当章の「受け入れ条件（DoD）」を満たすことをもって完了とします。

## 1. プロジェクト概要

### 1.1 目的
日本の建築基準に準拠した、線形・非線形・振動を**単一モデル**で扱える高速な構造計算一貫プログラムを Rust で構築する。

### 1.2 解析対象と機能スコープ

| 区分 | 内容 |
|---|---|
| 主要素 | ティモシェンコ梁要素、板/シェル要素（床・スラブ） |
| 梁要素機能 | 部材端の剛性／接合条件指定、柱梁接合部パネルゾーン、断面算定位置・危険断面位置の設定と降伏追跡 |
| 非線形梁 | マルチファイバー要素、マルチスプリング（MS）要素 |
| 耐震壁 | 壁エレメント置換（壁谷澤モデル / TVLEM） |
| 解析種別 | 線形静的、非線形静的増分（プッシュオーバー）、固有値（振動）、時刻歴応答 |
| 自動化 | 部材ごとの荷重変形特性（スケルトン/履歴）の自動算定 |
| 荷重生成 | 床荷重の自動分割（面荷重→部材荷重）、Ai分布の自動計算（略算／固有値による精算） |
| 設計 | 日本基準による断面算定（許容応力度・保有水平耐力・限界耐力） |
| 操作 | MCPサーバーによるモデル操作・解析実行・結果取得 |
| UI | GUI＋テーブル入力によるモデル作成、3Dビューア |
| 高速化 | マルチコア（既定）＋GPUオプション |
| 提案 | 機械学習による断面提案 |

### 1.3 非機能要件
- **速度:** 中規模（〜数万自由度）モデルの線形静的解析を 1 秒未満、固有値解析（下位30モード）を数秒オーダーで完了。プッシュオーバーは1ステップあたりサブ秒。
  - **基準モデル・基準機・対象工程（cold＝分解込み／warm＝後退代入のみ）は技術リードが別途ベンチマークとして用意する。** 上記の数値はそのベンチマーク上で定義・合否判定する（本書の値は暫定目安）。
- **再現性:** 同一入力で完全に同一出力（並列でも決定的に。後述の集約順序固定）。決定性の範囲は付録A参照。
- **検証性:** すべての要素・断面算定に対し、手計算または商用ソフトとの照合テストを用意。
- **移植性:** Windows / macOS / Linux。GPUオプションは無くても全機能が動作（CPUフォールバック必須）。

### 2.1 Cargoワークスペース構成

```
squid-n/                      # ワークスペースルート
├── Cargo.toml                   # [workspace]

│   ├── squid-n-material/             # 材料・履歴モデル（コンクリ/鋼/せん断）
│   ├── squid-n-section/              # 断面の幾何・剛性・ファイバ/スプリング分割
│   ├── squid-n-element/              # 要素ライブラリ（梁/板/ファイバ/MS/壁）

│   ├── squid-n-design-jp/            # 日本基準の断面算定（許容/保有/限界）

│   ├── squid-n-ml/                   # ML断面提案（burn/candle）
```

依存方向は **下流 → 上流の単方向**（`squid-n-app`/`squid-n-mcp` が最上位、`squid-n-core`/`squid-n-math` が最下層）。循環依存は CI で禁止（`cargo-deny` + 手製チェック）。

### 2.2 レイヤー図

```
 ┌─────────────────────────────────────────────┐
 │  squid-n-app (GUI)        squid-n-mcp (MCP server)      │  ← 操作層
 ├─────────────────────────────────────────────┤
 │  squid-n-design-jp   squid-n-ml    squid-n-io                │  ← 応用層
 ├─────────────────────────────────────────────┤
 │  squid-n-solver  ←  squid-n-skeleton                    │  ← 解析層
 ├─────────────────────────────────────────────┤
 │  squid-n-element  ←  squid-n-section  ←  squid-n-material     │  ← 力学モデル層
 ├─────────────────────────────────────────────┤
 │  squid-n-core (model)   squid-n-math (LA)   squid-n-gpu       │  ← 基盤層
 └─────────────────────────────────────────────┘
```

### 2.3 設計原則
- **データ指向設計（DOD）。** ノード・要素は構造体配列（SoA寄り）で持ち、`Vec<T>` + インデックス（`u32`）で参照。ポインタ参照やグラフ構造は避け、キャッシュ効率と並列化を優先。
- **トレイトによる要素抽象。** すべての要素は `Element` トレイト（後述）を実装し、ソルバは要素種別を知らずに剛性・内力を集約する。
- **featureフラグで重量依存を分離。** `gpu`, `ml`, `gui`, `mcp` はすべて opt-in feature。コア解析はこれらなしでビルド可能。

## 4. 共通データモデル（squid-n-core）

### 4.1 自由度・座標系・単位系
- 各節点は 3次元 6自由度（並進3、回転3）を基本とする。
- グローバル直交座標系、要素ローカル座標系、断面座標系を区別。
- **内部単位は一貫した SI（長さ=mm、力=N、時間=s）。** これより質量=N·s²/mm（=10³kg）、応力=N/mm²（数値的に MPa と同一だが**表記は N/mm² に統一**＝日本慣用）。時刻歴・固有値で破綻しないよう、この単位系を全クレートの不変条件とする。
- **入力は日本慣用単位で受ける:** 応力・強度は N/mm²、コンクリート/鋼材の密度・単位体積重量は g/cm³（または kN/m³）、部材長は m、荷重は kN・kN/m 等。UI／IO はこれらを受け取り、**境界で内部単位へ変換して保存**する（変換は `squid-n-io`／入力層が担当。内部は常に N・mm・s）。

### 4.2 主要データ構造（骨子）

**データと挙動の分離（R1）:** core は serde 対象の純データ（`ElementData`）のみを持ち、力学挙動は squid-n-element の `trait ElementBehavior`（6.1）が担う。両者はファクトリ（レジストリ）で対応付け、`ElemState`（履歴状態）は要素 index で別配列に保持する。これにより同名衝突を避け、状態の巻き戻し・直列化を分離できる。

```rust
// squid-n-core/src/model.rs
pub type NodeId = u32;
pub type ElemId = u32;
pub type StoryId = u32;
pub type SlabId  = u32;

#[derive(Clone, Debug)]
pub struct Node {
    pub id: NodeId,
    pub coord: [f64; 3],
    pub restraint: Dof6Mask,     // 拘束（支点条件）
    pub mass: Option<[f64; 6]>,  // 集中質量（振動解析用、内部単位 N·s²/mm）
    pub story: Option<StoryId>,  // 所属階（4.5）
}

#[derive(Clone, Debug)]
pub struct ElementData {                 // ← 純データ。挙動は持たない
    pub id: ElemId,
    pub kind: ElementKind,               // Beam / Shell / Fiber / MS / Wall / PanelZone ...
    pub nodes: smallvec::SmallVec<[NodeId; 8]>,
    pub section: Option<SectionId>,
    pub material: Option<MaterialId>,
    pub local_axis: LocalAxis,           // 部材軸の向き（コード角。規約は付録A）
    pub end_cond: [EndCondition; 2],     // 端部接合・剛域・端ばね（6.2.1/6.2.2）
    pub force_regime: ForceRegime,       // 非線形モデル自動選択用（R5/R25）
}

/// 部材が受ける応力自由度。非線形モデル選択を駆動する（6.8）。
#[derive(Clone, Copy, Debug)]
pub enum ForceRegime {
    UniaxialBendingShear,  // 剛床内の大梁等 → 集中ばねで十分
    AxialBendingInteract,  // 柱・軸力変動部材 → ファイバ（既定）/ MS
    Auto,                  // モデル接続から自動判定
}

pub struct Model {
    pub nodes: Vec<Node>,
    pub elements: Vec<ElementData>,
    pub sections: Vec<Section>,
    pub materials: Vec<Material>,
    pub stories: Vec<Story>,             // 階の定義（4.5）
    pub slabs: Vec<Slab>,                // スラブ範囲＋小梁配置（4.7）
    pub constraints: Vec<Constraint>,    // 剛床・MPC・リジッドリンク（4.6）
    pub load_cases: Vec<LoadCase>,
    pub combinations: Vec<LoadCombination>,
    pub dof_map: DofMap,
}
```

> `ElementKind` は判別用の列挙、`ElementBehavior` は挙動トレイト。`fn build_behavior(data: &ElementData, model: &Model) -> (Box<dyn ElementBehavior>, ElemState)` をファクトリとして squid-n-element に置く。`Dof6Mask(u8)`（bit0..5 が各自由度の拘束）等の補助型も core に置く。

### 4.3 DOFマップ
- 拘束自由度・従属自由度を除いた**独立自由度**に通し番号（`active_dof: u32`）を振り、全ソルバが共通利用する。
- 剛床・多点拘束による従属自由度の縮約は 4.6 の統合フレームワークで一元処理する。

### 4.4 受け入れ条件（DoD）
- 1万節点規模のモデルを生成・走査して 50ms 未満。
- `serde` でラウンドトリップ（保存→読込で完全一致）。

### 4.5 階（層）の定義 【R3・新規】
剛床・Ai分布の適用・層間変形・剛性率/偏心率・層ごとの設計判定の基盤となる、構造一貫プログラムの中核概念。

- **階（Story）** は、ほぼ同一高さで一つの床面を構成する節点群をまとめた論理単位。属性は: 代表標高、所属節点・部材、地震用重量、所属ダイアフラム（0..n）、各ダイアフラムのマスター節点（既定＝重心）。
- **定義方法:** 標高（Z レベル）による自動グルーピング、または明示割当。1つの階に**複数の独立ダイアフラム**を許す（エキスパンションジョイント・分割床）。
- **用途:**
  - 剛床: 階（ダイアフラム）ごとに従属節点の水平面内自由度をマスター節点へ拘束（4.6）。
  - Ai分布: 層せん断→層水平力をダイアフラムのマスター節点（重心）に作用（9.2）。
  - 層間変形角: 連続する階のマスター節点水平変位差／階高。
  - 剛性率・偏心率: 階ごとに算定（重心・剛心、11章）。
```rust
pub struct Story {
    pub id: StoryId, pub name: String,
    pub elevation: f64,
    pub node_ids: Vec<NodeId>,
    pub diaphragms: Vec<DiaphragmDef>,   // マスター節点・剛/柔の別
    pub seismic_weight: Option<f64>,      // 地震用重量（自動集計可）
}
```

### 4.6 拘束・剛体機構の統合フレームワーク 【R3】
3種の拘束を、単一の変換行列法（`u_full = T · u_indep`、`K_red = Tᵀ K T`）で一元化する。

1. **要素内剛体（局所）:** 剛域・端部ばね。**要素が 12×12 へ縮約**して返す（組立前に解決。6.2.1/6.2.2）。
2. **剛床ダイアフラム:** 階ごとに従属節点の水平2＋面内回転をマスター節点へ拘束。変換行列法で縮約。
3. **汎用多点拘束（MPC、1対多を含む）:** 1マスター↔多スレーブ／多↔1 を許す一般拘束。**柱（梁要素）と床（板要素）の接続**（節点非共有時に柱節点を周辺床節点へ拘束）、メッシュ遷移、リジッドリンクに使用。

```rust
pub enum Constraint {
    RigidDiaphragm { story: StoryId, master: NodeId, slaves: Vec<NodeId> },
    Mpc { master: NodeId, terms: Vec<(NodeId, Dof, f64)> }, // 1対多／多対1の線形結合
    RigidLink { master: NodeId, slaves: Vec<NodeId>, dofs: Dof6Mask },
}
```
- すべて `T` 行列に合成してから縮約。要素内剛体は組立前縮約なので二重計上が起きない（R3）。

### 4.7 スラブ（床）と小梁の定義 【R10・新規】
解析モデルは**柱・大梁まで**しか含めない。床荷重を大梁へ正しく流すため、スラブ範囲と内部小梁配置を入力実体として持つ（解析剛性には寄与させず、荷重分配にのみ使用）。

```rust
    pub loads: Vec<AreaLoad>,         // 面荷重（荷重種別・長期/地震/積雪別）
```

## 5. 数値計算基盤（squid-n-math）

### 5.1 ソルバ抽象

```rust
pub trait LinearSolver {
    /// K x = f を解く（factorize済みを再利用できるよう設計）
    fn solve(&mut self, rhs: faer::MatRef<f64>) -> faer::Mat<f64>;
}

pub enum SolverBackend {
    DirectSparseCholesky,   // 対称正定値（線形静的・主力）。faer疎LLT/LDLT
    DirectSparseLu,         // 非対称・不定（接合部等）
    IterativePcg { tol: f64, max_iter: usize }, // 大規模・GPU向け
}
```

### 5.2 方針
- **線形静的の主力は疎直接法（Cholesky/LDLᵀ）。** 同一の係数行列に対して複数荷重ケースを解くため、**分解（factorize）を1回・前進後退代入を多数回**の構成にする。
- 反復法（前処理付きCG, PCG）は GPU 経路と超大規模用。前処理は対角/ヤコビ→不完全Cholesky（ic0）の順で用意。
- 行列組立は **要素ごとのローカル剛性を rayon で並列計算 → 決定的順序でグローバルへ集約**（集約は単スレッド、またはトリプレット列を安定ソートして加算）。浮動小数の非結合性による非決定性を排除する。

### 5.3 受け入れ条件
- 既知の単純梁・ラーメンで手計算と 1e-9 相対誤差以内一致。
- 同一モデルを 100 回解いてビット一致（決定性）。

## 6. 要素ライブラリ（squid-n-element）

### 6.1 共通トレイト

挙動トレイトは `ElementBehavior`（core の純データ `ElementData` と区別。R1）。ファクトリ `build_behavior` が `ElementData` から生成し、`ElemState` を初期化する。

```rust
pub trait ElementBehavior {
    fn n_dof(&self) -> usize;
    fn dof_indices(&self, dof: &DofMap) -> SmallVec<[usize; 24]>;

    /// 接線剛性（線形要素では一定、非線形では状態依存）
    fn tangent_stiffness(&self, state: &ElemState, ctx: &Ctx) -> LocalMat;

    /// 現在変位における内力ベクトル（非線形の釣合計算で使用）
    fn internal_force(&self, state: &ElemState, ctx: &Ctx) -> LocalVec;

    /// 状態更新（ひずみ→応力、履歴変数の更新）。収束後に確定コミット
    fn update_state(&mut self, du: &LocalVec, commit: bool, ctx: &Ctx);

    /// 質量行列（整合質量／集中質量を選択可）
    fn mass_matrix(&self, opt: MassOption) -> LocalMat;

```

> `state` を要素から分離して `ElemState`（履歴変数列）に置くことで、解析の試行ステップ（リジェクト時の巻き戻し）を容易にする。**commit/rollback パターン**を全非線形要素で統一する。

### 6.2 ティモシェンコ梁要素 【自前実装】
- 2節点・各端6自由度（計12自由度）。**せん断変形を考慮**する。
- せん断ロッキング回避のため、**厳密2節点ティモシェンコ剛性（相互依存補間 / interdependent interpolation element, IIE）**を採用。等分布外力に対しても厳密。
  - パラメータ φ = 12EI / (G·As·L²)（As=せん断有効断面積、k=せん断補正係数）。
  - 剛性行列の各項は φ を含む解析式で与える（実装は閉形式）。
- 線形・分布バネ基礎・温度荷重に対応。
- **ハンチ・テーパー（R16）:** 慣用に従い、**剛性は一様断面として扱い、ハンチ／テーパーは無視**する（ハンチを除いた基準断面で剛性算定）。可変断面剛性・ハンチの剛域近似は将来オプションとして拡張点に残す。
  - ハンチの**形状（ハンチ長・せい増分・幅増分）と継手（ジョイント）位置**は、剛性に影響しない**部材付帯情報**（`Model::member_detail_attrs`、`MemberDetailAttr`）として保持する。断面算定の検定位置追加（ハンチ端・継手位置。6.2.3）と数量拾いの基礎データに用いる。

#### 6.2.1 剛域の考慮と自動算定
柱梁接合部の有限寸法により部材端が剛体的に挙動する領域（剛域）を扱う。本機能は要件の中核として標準実装する。

- **剛性への反映:** 要素の可とう長を `L' = L − λ_i − λ_j`（λ＝各端の剛域長）とし、剛域部は剛体リンクとして接合部中心（節点）の自由度と可とう部端部の自由度を変換行列 `T` で関係づける。曲げ・せん断・軸・ねじりすべてに作用し、最終的に 12×12 のグローバル剛性へ縮約する。
- **剛域長の自動算定:** 各部材端で接合する直交部材（梁に対しては柱、柱に対しては梁／基礎、壁取付き部は壁）をモデルの接続情報から自動検出し、断面寸法から剛域長を算定する。
  - 標準式（慣用・AIJ系）: `λ = D_orth/2 − D_self/4`（`D_orth`＝直交部材せい、`D_self`＝自部材せい）。**負値は 0 に丸める。**
  - 例: 梁端剛域 `= 柱せい/2 − 梁せい/4`。柱頭・柱脚剛域 `= 梁せい/2 − 柱せい/4`。
  - **剛域長低減率** `r`（既定 1.0、慣用 0.0〜1.0）を乗じて `λ = r·(D_orth/2 − D_self/4)` と調整可能。
  - 偏心接合・ハンチ・段差梁・壁端・複数部材が集まる節点の扱いは設定で選択（採用ルールを技術リードが確定）。
- **手動上書き:** 端部ごとに λ を直接指定可能。自動値か手動値かのフラグを保持し、**再計算（断面変更時の再自動算定）で手動値を保護**する。
- **設計位置との区別:** 剛性に使う剛域（D/2−D/4）と、断面算定の評価位置（6.2.3、既定は柱フェース＝D_orth/2）は別概念として両方を保持する。

```rust
pub struct RigidZone {
    pub length_i: f64, pub length_j: f64,   // 確定剛域長
    pub source_i: ZoneSource, pub source_j: ZoneSource, // Auto / Manual
    pub reduction: f64,                       // 低減率 r
}
pub fn auto_rigid_zones(model: &Model, elem: ElemId, rule: &RigidZoneRule) -> RigidZone;
```

**DoD:** 既知寸法で λ が標準式と一致し負値が 0 に丸められる。λ→0 で剛域なしと一致。剛域ありラーメンの材端モーメント・層間変形が手計算／商用ソフトと一致。断面変更→自動再算定で手動値が保持される。

#### 6.2.2 部材端の接合条件・端部ばね
両端に以下を独立指定できる「端部処理レイヤ」を梁要素にラップする（剛域の内側に直列）。
- **端部接合条件:** 剛接（fix）／ピン（moment release）／**半剛接合（回転ばね）**。半剛は端部に回転ばね定数 `kθ`（または剛度 `R`）を直列接続し、要素の縮約（static condensation）で 12×12 剛性へ織り込む。
- **端部ばね:** せん断・軸・ねじりの部分剛性も同様に直列ばねで指定可。
- 端部ばねを非線形（履歴ばね）にすると、後述の集中ばねモデル（塑性ヒンジ）として機能する。

#### 6.2.3 断面算定位置・危険断面位置の設定
- 部材に沿って任意数の **評価断面位置**（材軸の正規化座標 0..1、または絶対距離）を定義できる。各位置で内力（N, Q, M）を復元（force recovery）する。
- 既定の危険断面は **柱フェース位置（剛域端）と中央**。断面算定（第11章）はこのフェース位置の応力で行う（節点芯の応力をそのまま使わない）。
- 部材付帯情報（`MemberDetailAttr`。6.2 ハンチ・テーパー参照）を持つ部材は、**ハンチ端（フェース＋ハンチ長）と継手位置**を評価断面・検定位置へ自動追加する（`MemberDetailAttr::extra_check_positions`。応力評価断面と検定位置で同一実装を共有し位置を一致させる）。
- 各危険断面に降伏判定属性を持たせ、非線形解析（第10章プッシュオーバー・時刻歴）で**降伏・ヒンジ発生・塑性率を位置ごとに追跡**する。ヒンジ位置は断面算定位置と整合させる。

**DoD:** φ→0 でベルヌーイ・オイラー梁に一致、片持ち梁先端たわみが理論値（曲げ＋せん断）と一致。半剛接合がピン⇔剛接の極限に収束。剛域長を変えると材端モーメントとフェース位置モーメントの関係が手計算と一致。

### 6.3 マルチファイバー要素 【自前実装】
- 部材長さ方向に複数の積分断面（ガウス点）、各断面を多数のファイバに分割。
- 各ファイバは `squid-n-material` の **一軸応力–ひずみ履歴則**を持つ。
- 断面の力－変形（N, My, Mz – 軸ひずみ・曲率）は平面保持仮定でファイバ積分。
- せん断は別途（断面のせん断ばね or ティモシェンコ的せん断剛性を付加）。
- 定式化は **変位ベース**（剛性法）を基本とし、将来 **力ベース（フォースベース）ファイバ要素**を拡張点として確保（収束性・要素数低減に有利）。

```rust
pub struct FiberSection {
    pub fibers: Vec<Fiber>,        // (y, z, area, material_id)
}
// 断面戻り値: 軸力・二軸曲げと、その接線断面剛性 D(3x3 or 含むせん断)
fn section_response(sec: &FiberSection, strain: SectionStrain, st: &mut [MatState])
    -> (SectionForce, SectionStiffness);
```

**DoD:** 単軸引張・圧縮、純曲げで断面ファイバ積分がモーメント–曲率関係（M-φ）の理論値に一致。

### 6.4 マルチスプリング（MS）要素 【自前実装】
- 主に RC 柱・梁端部の塑性ヒンジ表現。部材端の断面を **軸方向ばねの集合**に置換し、中央は弾性材で連結する（two-component / 端部ばね型）。
- 各ばねは軸力負担位置（断面内座標）を持ち、軸方向の非線形履歴則（コンクリばね・鉄筋ばね）に従う。曲げと軸力の相互作用（N-M相関）を自然に表現できるのが利点。
- せん断ばね・ねじりばねを別系統で付加。

**DoD:** 一定軸力下の繰り返し曲げで、N-M 相関・履歴ループがファイバ要素と整合（相互検証）。

### 6.5 板 / シェル要素（床・スラブ） 【自前実装＋定式化は文献準拠】
- 4節点シェル要素。**膜＋曲げ＋せん断**を扱う。
- **せん断ロッキング回避に MITC4（仮定ひずみ法）**を採用。面内は適合要素＋ドリリング自由度の安定化（Allman/Cook流）。
- 床は主に**面内剛性（剛床効果）と鉛直荷重伝達**で使うが、面外曲げも評価できる汎用シェルとする。
- **剛床との二重計上回避（R13）:** 剛床ダイアフラムを適用した階の床については、当該シェルの**面内剛性を無効化**する（面内は剛床拘束が担う）。柔床解析時のみシェル面内剛性を有効にする。
- 数値積分は面内 2×2、必要に応じ低減積分＋砂時計制御。

**DoD:** パッチテスト合格、片持ち板・単純支持板のたわみが理論/参照解に一致。

### 6.6 耐震壁：壁エレメント置換（壁谷澤モデル / TVLEM） 【自前実装・文献厳密準拠】
- 連層耐震壁を**マクロ要素**で置換する。壁谷澤の **Three Vertical Line Element Model (TVLEM)** をベースとする:
  - 両側柱位置に **2本の軸ばね**（境界部の軸変形・ロッキング）、
  - 中央に **軸ばね＋回転ばね**（曲げ）、
  - 全体に **水平せん断ばね**、
  - 上下端を剛梁で連結し、**回転中心高さ比 c**（多くは 0.5 前後）を持つ。
- 各ばねの履歴則は `squid-n-material`/`squid-n-skeleton` から供給。せん断ばねは原点指向型、軸ばねは軸力依存スケルトンを使用。
- フレームへの組込みは、上下の剛梁端を周辺梁・床の節点へ剛接続（リジッドリンク）して行う。

> **重要:** TVLEM はパラメータ定義（ばね配置・回転中心・スケルトン算定式）が原著および各種改良版で異なる。**実装は採用する原著論文（壁谷澤ら）と社内設計規準を1つ確定**してから着手し、係数表をコード外（設定ファイル）に分離すること。検証は実験データ／既往解析との照合を必須とする。

**DoD:** 既往の壁実験（または商用ソフト結果）に対し、初期剛性・降伏点・最大耐力・履歴形状が許容差内。

### 6.7 柱梁接合部パネルゾーンのモデル化 【自前実装・添付資料準拠】

**正準定式は添付資料『パネルゾーンの力学』(小野瀬, 2009) とする。** 同資料の釣り合い（フェースモーメント変換・パネルせん断・検算）をそのまま実装する。**v1 は弾性のみ**（非線形パネルは将来）。

**(1) パネル寸法と符号**
- パネル寸法: `dc = Dc − tcs`（柱せい方向＝梁のパネル幅）、`db = Db − tbs`（梁せい方向＝柱のパネル高さ）、`tp`＝パネル板厚。
- 符号は「左から水平力を受けたときの標準 M 図」を正とする（資料の規約）。

**(2) フェースモーメント変換（断面算定位置と直結。6.2.3）**
解析が与える**節点中心モーメント**（アスタリスク付き）から、パネル枠位置の**フェースモーメント**を求める:
```
梁フェース:  bML = ML*b − bQL·(dc/2),   bMR = MR*b − bQR·(dc/2)
柱フェース:  cML = ML*c − cQL·(db/2),   cMU = MU*c − cQU·(db/2)
```
断面算定（11章）と危険断面追跡（6.2.3）はこのフェースモーメントを用いる。

**(3) パネルせん断（パネルモーメント）**
```
pQc·db = (bML + bMR) − (cQL + cQU)·(db/2)      … 梁側 (5)
pQb·dc = (cMU + cML) − (bQL + bQR)·(dc/2)      … 柱側 (6)
整合条件: pQc·db = pQb·dc                       … (4)
パネルせん断応力: τ = pQc/(dc·tp) = pQb/(db·tp)
```
- ＋型・ト型・L型・T型・連続梁の各節点は、欠落部材の項を 0 とすることで同一式でカバーする（資料 図23）。

**(4) 剛性モデル（弾性）**
- **方式A：剛域近似（既定・軽量）** — 接合部を剛域（6.2.1）として扱う。
- **方式B：弾性せん断パネル** — パネルを純せん断変形する弾性パネル（せん断剛性 `Kp = G·tp·dc`相当）としてフレーム水平変形へ寄与させる（剛棒＋ピン＋薄板＝ひし形変形の理想化、資料 図13–15）。軸・曲げ変形は生じないものとする。
- いずれも梁要素端・柱要素端と剛リンクで接続し、6.2.1 の剛域・6.2.2 の端部ばねと整合させる。

**(5) 設計連携**
- (3) の `τ` は鋼構造の**柱梁接合部パネルのせん断検定**（11章）に渡す。
- 梁軸力（`bNL − bNR = cQL − cQU`）は水平釣り合い上必要であり、パネル計算では無視しない（資料 5章の指摘）。

**DoD:** 添付資料の数値計算例（図18–20 の3ケース）について、フェースモーメント・`pQc`・`pQb`・検算 `τc=τb` が資料の値と一致（例: ケース1で pQc=851.135kN, pQb=1702.273kN, τc=τb=42.557/tp）。方式Aが剛域モデルと一致。

### 6.8 非線形モデルの自動選択 【R5/R25】
プッシュオーバー・時刻歴で、部材が受ける応力自由度（`ForceRegime`、4.2）に応じて非線形モデルを自動で割り当てる。線形静的解析では非線形を考慮しないため対象外。

- **剛床内で一軸曲げ＋せん断のみを受ける大梁** → **集中ばね（one/two-component＋武田型）** で十分（軸力変動が無視できる）。
- **軸力と曲げを同時に受け N–M 相関の評価が要る部材**（柱、軸力変動する梁、ブレース置換部材等）→ **既定でファイバ要素**（MS も選択可）。
- 判定は `ForceRegime::Auto` のとき、剛床所属・軸力変動の有無・部材種別（柱/梁）からモデルが自動で行い、`UniaxialBendingShear` / `AxialBendingInteract` を確定。ユーザ上書き可。
- 1モデル内で集中ばね部材とファイバ部材が**混在**してよい（要素単位で独立）。

**DoD:** 同一架構で、剛床梁＝集中ばね・柱＝ファイバが自動割当され、N一定の柱でファイバとMSのN–M履歴が整合（相互検証）。

## 7. 材料・履歴特性モデル（squid-n-material） 【自前実装】

一軸履歴則を共通トレイトで提供し、ファイバ/MS/壁ばねが共有する。

```rust
pub trait UniaxialMaterial {
    fn trial(&mut self, strain: f64) -> (f64 /*stress*/, f64 /*tangent*/);
    fn commit(&mut self);
    fn revert(&mut self);   // 試行の巻き戻し
}
```

最低限実装するモデル:
- **コンクリート:** 圧縮側の上昇・軟化＋引張のひび割れ・テンションスティフニング。包絡線は AIJ系の応力–ひずみ式を選択可能に。
- **鉄筋/鋼材:** バイリニア、武田型、Menegotto–Pinto（バウシンガー考慮）。
- **RC部材の曲げ履歴:** **武田モデル（Takeda）**およびその degrading 版（剛性低下型トリリニア）。
- **せん断履歴:** 原点指向型、スリップ型。

設計時に係数（包絡線・除荷剛性低下指数等）を**データ駆動**（テーブル/設定）にし、コードへハードコードしない。

**DoD:** 各履歴則について、規定の繰り返し変位履歴に対し参照ループと一致（スナップショット試験 `insta`）。

## 8. 部材荷重変形特性の自動算定（squid-n-skeleton） 【自前実装：本製品の核心の一つ】

各部材について、断面・配筋・材料・スパン・想定軸力から **スケルトン曲線（荷重–変形関係）を自動生成**し、集中ばねモデル（one/two-component）や壁ばねに供給する。

処理フロー:
1. 断面ファイバモデルから **M–φ 関係**を数値積分で算定（軸力依存）。
2. ひび割れ・降伏・終局点を抽出し **トリリニア化**（折れ点の決め方は設計規準準拠）。
3. 部材の境界条件・スパン・反曲点位置から **M–φ → M–θ（部材端モーメント–回転角）** へ積分変換（柔性法）。
4. せん断・付着・抜出しによる回転を加算。
5. 履歴則（武田型等）と組み合わせ、要素に渡すスケルトン＋履歴パラメータを確定。

```rust
pub struct MemberSkeleton {
    pub points: Vec<(f64 /*変形*/, f64 /*耐力*/)>, // トリリニア折れ点
    pub hysteresis: HysteresisRule,
    pub axial_dependency: AxialInteraction, // N によるスケルトン補正
}
pub fn build_member_skeleton(member: &MemberData, n_axial: f64) -> MemberSkeleton;
```

**DoD:** 代表的 RC 梁・柱で、手計算（規準式によるひび割れ/降伏/終局）と折れ点が一致。プッシュオーバーに投入して破壊機構が妥当。

## 9. 荷重の自動生成（squid-n-load） 【自前実装：日本基準準拠】

設計者が床に面荷重を、建物に地震規模を与えるだけで、解析に必要な部材荷重・節点荷重・節点質量・地震時水平力を自動生成する。**規準式・係数（Z, Rt, C0 等）はデータ駆動で外部化**し、出典告示をコメントに残す。

### 9.1 床荷重の自動分割（面荷重 → 部材・節点荷重）
解析モデルは**柱・大梁まで**しか含めない（小梁は解析剛性に入れない）。床荷重を大梁へ正しく流すため、4.7 のスラブ実体（範囲＋**小梁配置**）を用いて分配する。
- スラブ／床の面荷重（固定荷重・積載荷重を、**積載は床用／架構用／地震用の3区分**で、長期/地震/積雪別に）を分配対象とする。
- **小梁を考慮した分配（R10）:** 単純なスラブ範囲按分ではなく、内部の小梁配置（方向・間隔・支持）を考慮する。すなわち、まず面荷重を小梁へ（小梁が無い領域は直接大梁へ）流し、小梁反力を支持する大梁へ集中／分布荷重として与える、という2段階の荷重伝達を行う。
- 分配方式（スラブごとに選択可）: 三角形・台形分布（45°）、一方向、負担面積法、不整形は多角形分割。
- 出力: 大梁要素への分布荷重（等分布／台形）、節点集中荷重、および**床自重・積載 → 各階の地震用重量・節点質量**（4.5 の階・振動解析へ直結）。荷重組合せは法令の標準組合せを**自動生成**（係数は外部化）＋手動追加可。
- **検証出力（R10）:** 自動分配の妥当性確認のため、各大梁の **CMQ図（荷重項：両端固定端モーメント C/M とせん断 Q）** を可視化（14章）。

**DoD:** 矩形スラブで三角形／台形分配の総和が床総荷重に一致（質量・荷重保存）、小梁経由の反力が手計算と一致、CMQ図が荷重項の理論値と一致。

### 9.2 Ai分布（地震層せん断力分布）— 略算
- 各階の地震用重量から、告示式に基づく層せん断力係数 **Ci = Z · Rt · Ai · C0** を算定する。
- **Ai 略算式:** `Ai = 1 + (1/√αi − αi) · 2T / (1 + 3T)`
  - `αi =（i 階より上の地震用重量の和）/（全地震用重量）`
  - `T`: 設計用一次固有周期。略算では略算式 `T = h(0.02 + 0.01·α)`（h=高さ、α=鉄骨部分の割合）等を使用可。
- `Z`（地域係数）、`Rt`（振動特性係数：T と地盤種別 Tc から）、`C0`（標準せん断力係数：一次設計 0.2／二次設計 1.0）を保持。
- 層せん断力 `Qi = Ci · Wi'`（Wi'＝i 階以上の重量）→ 各層の水平力 `Pi = Qi − Q(i+1)` に分解し、層・ダイアフラムのマスター節点（重心）へ作用させる（4.5/4.6）。
- **解析方向（R12）:** 既定は **X／Y の正負（XY±）**、オプションで45°方向。**付加偏心は日本法令で非要求のため実装しない**（実偏心＝重心・剛心のずれは剛床マスター節点位置として自然に考慮される）。

**DoD:** 告示の計算例と Ci・Qi が一致。

### 9.3 Ai分布 — 精算（固有値解析の利用）
- **準精算:** 固有値解析（第10章の固有値ソルバ）で得た**実固有周期 T** を Ai 略算式へ代入し、略算式の T 依存性を実態に合わせる。
- **精算:** モード応答（モード形・刺激係数・有効質量）から各モードの層せん断力を求め、**モード重ね合わせ（SRSS / CQC）**で層せん断力分布を直接算定する。略算 Ai に代えてこの分布を用い、必要なら等価 Ai を逆算して出力する。
- **一気通貫:** 単一モデル上で「固有値解析 → 分布算定 → 静的解析への水平力投入」を自動接続する（本製品の核心要件）。略算／精算は同一 API で切替え。

**DoD:** 略算が告示計算例と一致。精算では固有値由来 T により分布が変化することを確認し、モード重ね合わせの層せん断が時刻歴応答の包絡と整合。


## 10. 解析エンジン（squid-n-solver）

### 10.1 共通設計：単一モデルでの解析統合
- **モデルは1つ**。解析種別は「組立てた K（と必要なら M）をどう使うか」の違いに過ぎない設計にする。
- 非線形解析が確定した接線剛性・履歴状態を保持し、**その状態のまま固有値解析（割線/接線剛性による瞬間固有値）**を実行できる API を用意する（「シームレス」要件の核）。

```rust
pub struct Analysis<'m> { model: &'m mut Model, asm: Assembler, /* ... */ }
impl Analysis<'_> {
    fn linear_static(&mut self, lc: LoadCaseId) -> StaticResult;
    fn pushover(&mut self, cfg: PushoverCfg) -> PushoverResult;
    fn eigen(&mut self, n_modes: usize, basis: StiffnessKind) -> ModalResult; // 初期/接線/割線
    fn time_history(&mut self, wave: &GroundMotion, cfg: NewmarkCfg) -> ResponseResult;
}
```

### 10.2 線形静的
- K を疎直接分解 → 各荷重ケース・組合せを後退代入で高速処理。
- 反力・部材応力・層間変形・剛床変位を出力。

### 10.3 非線形静的増分（プッシュオーバー） 【自前実装】
- **水平力分布・制御（R7）:** 既定は **Ai分布比例の荷重増分 ＋ 代表層（最上階重心マスター節点）の変位制御**。耐力低下を追跡する局面では弧長法へ切替。1次モード比例分布も選択肢として用意。
- 荷重制御／**変位制御**／**弧長法（arc-length）**を選択可能に（軟化・耐力低下を追跡するため弧長法は必須）。
- 釣合反復は **Newton–Raphson（修正NRも）**。各反復で要素 `tangent_stiffness`/`internal_force` を集約。
- 収束判定はエネルギー・力・変位ノルムの複合。発散時はステップ縮小・要素状態 rollback。
- ステップごとに層せん断力–層間変形、部材ヒンジ状態、保有耐力到達を記録。
- **危険断面位置（6.2.3）での降伏追跡:** 各部材の柱フェース・中央等で M がスケルトン折れ点（ひび割れ／降伏／終局）に到達した時刻（ステップ）と塑性率を記録し、**ヒンジ発生順序・崩壊機構（全体崩壊／層崩壊）を自動判定**する。保有水平耐力時の機構を第11章の Ds 判定へ引き渡す。

**DoD:** ばね単体・1層ラーメンで理論的な弾塑性挙動・終局機構が再現。弧長法で耐力低下枝を追跡可能。

### 10.4 固有値（振動）解析 【自前定式化、分解は faer】
- 一般化固有値問題 **K φ = λ M φ** を解く。
- 大規模疎では **shift-invert Lanczos／部分空間反復法**を自前実装し、内部の (K − σM) 分解は faer の疎分解を利用。
- 質量は集中／整合を選択。固有周期・刺激係数・有効質量比・モード形を出力。
- 接線/割線剛性での瞬間モード解析（10.1）に対応。

**DoD:** 等価せん断棒・既知ラーメンで固有周期が理論値と一致、有効質量比合計≈100%。

### 10.5 時刻歴応答解析 【自前実装】
- **入力規約（R8）:** v1 は**基盤一様加振・水平1〜2方向**（上下動は将来）。地震波は**入力受領のみ**（告示スペクトル適合波の生成は別フェーズ）。ベースライン補正・スケーリングの有無は入力設定で明示。
- **Newmark-β**（平均加速度・線形加速度）と HHT-α を用意。
- **減衰（R9）:** **質量比例／剛性比例／Rayleigh を UI で選択可**。既定は日本の建築慣行に合わせ **1次モードに対する剛性比例減衰**（規定減衰：RC h=0.03／S h=0.02 等、設定値で外部化）。剛性比例は初期/接線を選択。
- 非線形要素の状態更新を各時刻で実施（内部反復付き）。
- 地震波（観測波・告示波）を入力。最大応答・層間変形角・累積塑性率を出力。

**DoD:** SDOF の解析解と一致、線形MDOFがモード重ね合わせと一致。各減衰タイプが理論減衰比と整合。

### 10.6 幾何剛性（P-Δ） 【任意機能・R6】
- 日本の建築構造設計では P-Δ は必須ではないため、**線形化幾何剛性（geometric stiffness `Kg`）を任意機能**として実装する。共回転定式化・大変形は実装しない。
- 軸力から `Kg` を構成し `K_t = K + Kg` として静的・固有値・時刻歴に加算可能とする（ON/OFF 切替）。最適化・検討用途で使用。

**DoD:** 単純な軸力付き柱の座屈荷重近傍で固有振動数低下（P-Δ効果）が理論傾向と一致。OFF時は従来結果と一致。

## 11. 断面算定（日本基準）（squid-n-design-jp） 【自前実装・規準準拠】

> 設計式・係数は**建築基準法施行令・関連告示・各学会規準（AIJ）**に準拠。**規準の数値・係数は版管理されたデータファイルとして外部化**し、改定に追従できるようにする。法令の条文番号・規準ページは実装コメントに明記する。

### 11.1 対象
- **許容応力度計算（一次設計）:** 長期・短期の許容応力度に対する断面検定（RC：曲げ・せん断・付着・定着、S：曲げ・せん断・座屈・接合部、SRC）。
- **柱梁接合部パネルのせん断検定（S）:** 6.7 のパネルせん断 `τ` に対する検定（鋼構造接合部設計指針に基づく）。
- **保有水平耐力計算（二次設計）:** プッシュオーバーから保有水平耐力 Qu を取得し、必要保有水平耐力 Qun = Ds·Fes·Qud と比較。
  - **Ds 自動分類（R14）:** 破壊モード（曲げ／せん断耐力比）による部材ランク（FA〜FD）に加え、**鋼部材は板要素の幅厚比（FA〜FD区分）も併用**して部材ランクを決定し、層 Ds を算定する。自動値は提案とし、**ユーザ上書きを前提**とする。
  - Fes（形状係数：剛性率・偏心率より）、層崩壊機構の判定（6.2.3 の降伏追跡から）。
- **限界耐力計算:** **v1 対象外（v2 で実装）。** 当面は許容応力度＋保有水平耐力ルートを必須機能とする。
- **層間変形角・剛性率・偏心率**等の各種規定チェック（階の定義 4.5 に基づく）。

### 11.2 構成
```rust
pub trait DesignCheck {
    fn check(&self, forces: &MemberForces, sec: &Section, mat: &Material, ctx: &DesignCtx)
        -> CheckResult;  // 検定比、判定、根拠（適用条文）
}
```
- 材種別（RC/S/SRC）×検定項目で実装を分割。
- **断面算定位置:** 検定は 6.2.3 で定義した危険断面位置（既定は柱フェース・中央）の内力で行う。節点芯ではなくフェース位置のモーメントを用い、剛域・パネルゾーンと整合させる。位置はユーザが追加・変更可能で、その全位置で検定比を算出する。
- 出力は**検定比（demand/capacity）と適用根拠**をセットで返し、後段（GUI/MCP/レポート）で表に展開。検定比は危険断面位置ごと・荷重組合せごとに表示。

**DoD:** 代表断面で手計算（規準例題）と検定比が一致。境界値（ちょうど1.0）の扱いを試験。

## 12. 入出力・データ永続化設計（squid-n-io） 【一部採用ライブラリ：serde / arrow・parquet】

### 12.1 設計方針
- 正準データモデル `squid-n-core::Model` を唯一の真実とし、**(a) モデル保存／(b) 結果I/O／(c) 途中計算（チェックポイント・スクラッチ）／(d) 外部連携** の4系統を分離する。
- すべての形式に `schema_version` を持たせ、読み込み時にマイグレーションで最新化（後方互換維持）。
  **ただし初回リリースまでは後方互換を持たない**: `CURRENT_SCHEMA_VERSION = 1` のまま
  スキーマを直接変更し、現行版以外は拒否する。マイグレーション機構は骨子のみ維持し、
  リリース後の版上げから有効化する（2026-07-11 改訂）。
- 内部 SI 固定、入出力境界でのみ単位変換。
- 大容量結果はメモリに全保持せず、逐次書き出し・部分読み出しを基本とする。

### 12.2 モデル保存形式（プロジェクトファイル）
- serde で2系統の表現を提供:
  - **可読・差分管理向け:** JSON または RON（バージョン管理・レビュー・デバッグ用）。
  - **コンパクト・高速:** MessagePack（`rmp-serde`）— 大規模モデルの既定。
- **プロジェクトコンテナ `.scz`（zip）** に関連物を束ねる:
  ```
  project.scz
  ├── manifest.json     # スキーマ版・単位系・作成情報・各エントリのハッシュ
  ├── model.msgpack     # 正準モデル（節点/要素/断面/材料/荷重/組合せ/剛域）
  ├── settings.json     # 解析設定・準拠規準の版・剛域ルール・荷重分配方式
  ├── results/          # 結果（12.3。外部管理も可）
  └── attachments/      # 図面・メモ等
  ```
- 安定 ID（`u32` newtype）により、結果・MCP からの外部参照が版をまたいで有効。
- 保存は**原子的**（temp へ書き出し → rename）。ハッシュ検証で破損を検出。

### 12.3 結果I/O（列指向・部分読み出し）
- 結果は大容量（特に時刻歴）になるため **列指向フォーマット（Apache Arrow / Parquet：`arrow`・`parquet` クレート）** で格納。圧縮・選択読み出し・Python/pandas 連携が容易。
- レイアウト（解析ケース → 結果種別 → テーブル）:
  ```
  results/
  ├── index.json               # マニフェスト（ケース・種別・行数・範囲）
  └── case_<id>/
      ├── nodal_disp.parquet    # 節点変位・反力
      ├── member_force.parquet  # 部材内力（評価位置別）
      ├── story.parquet         # 層せん断・層間変形・剛性率・偏心率
      ├── modal.parquet         # 固有周期・刺激係数・有効質量・モード形
      ├── pushover.parquet      # 荷重係数・層せん断–層間変形・ヒンジ履歴
      └── design_check.parquet  # 検定比（位置別・組合せ別・適用条文）
  ```
- **ストリーミング書き込み:** 解析中に RowGroup 単位で追記し、全保持しない。
- **部分読み出し:** 「ケース X・部材 Y・ステップ範囲 Z」のみ取得（GUI/MCP の即応性の要）。
- 結果はモデルと分離可能（`.scz` 内／外部ディレクトリ）。大規模は外部格納を推奨。

```rust
pub trait ResultStore {
    fn writer(&mut self, case: CaseId, kind: ResultKind) -> Box<dyn ResultWriter>;
    fn query(&self, q: &ResultQuery) -> ResultBatch;  // 部分読み出し
    fn manifest(&self) -> &ResultManifest;
}
```

### 12.4 途中計算・チェックポイント・スクラッチ
長時間の非線形／時刻歴解析の**再開・巻き戻し**のため、ジョブ作業領域を定義する。
- **ジョブワークスペース:**
  ```
  <workdir>/job_<id>/
  ├── checkpoint/   # 解析状態スナップショット（再開用）
  ├── scratch/      # 行列分解・組立の一時ファイル（再生成可・削除可）
  └── log/          # tracing ログ・収束履歴
  ```
- **チェックポイント内容:** 現在の変位・速度・加速度、全要素の確定履歴状態（commit 状態）、荷重係数／時刻、ステップ番号、（あれば）乱数状態、スキーマ版。形式は高速バイナリ（`bincode`／MessagePack）、必要に応じ memmap。
- **再開（restart）:** 直近チェックポイントから継続。プッシュオーバーの任意ステップからの分岐再解析にも対応。
- **スクラッチ:** 疎分解の中間結果や巨大一時行列は、メモリ上限を超える場合のみディスク退避。再生成可能なキャッシュ扱いとし、ジョブ終了時に自動掃除。
- チェックポイントとモデルを**整合性ハッシュ**で対応付け、モデル変更後の不正な再開を防止。

### 12.5 外部連携（インポート／エクスポート）
- **構造データ交換:** 日本の構造分野標準 **ST-Bridge（XML, 2.0 系を対象）** のインポート／エクスポートを用意（他社一貫プログラム・BIM との相互運用）。**完全往復は保証せず、対象は幾何・断面・部材・荷重まで**（解析結果や独自属性は対象外）と期待値を明記。BIM 連携が必要なら **IFC** も検討。
- **表データ:** 入力テーブル・結果表の **CSV** 入出力（GUI のコピー＆ペーストと共通経路）。
- **計算書:** **PDF / Excel(xlsx)** で出力（社内テンプレート）。
- すべての外部形式は内部モデルへの**変換層**を介し、正準モデルを唯一の真実とする（外部形式を直接の解析対象にしない）。

### 12.6 受け入れ条件（DoD）
- モデルの保存→読込で完全一致（ラウンドトリップ）。旧スキーマの自動移行が通る。
- 1GB 級の時刻歴結果から、特定部材・特定ステップのみを 100ms 級で部分読み出し。
- 解析を途中強制終了 → チェックポイントから再開し、無停止実行と同一結果に収束。
- ST-Bridge の代表モデルが往復（import→export）で意味的に保存される。


## 13. MCPサーバー（squid-n-mcp） 【採用ライブラリ: rmcp 1.x】

MCP 経由でモデル操作・解析実行・結果取得・断面算定を行えるツール群を公開する。**rmcp の `#[tool]` マクロ**でツール定義、stdio/HTTP両トランスポート対応。

公開ツール（例）:

| ツール名 | 機能 |
|---|---|
| `model.load` / `model.save` | モデルの読込・保存 |
| `model.query` | 節点・部材・断面の検索（フィルタ） |
| `model.edit` | 部材追加・断面変更・荷重設定 |
| `analysis.run` | 解析実行（種別・パラメータ指定） |
| `result.get` | 応力・変形・固有周期・検定比の取得 |
| `design.check` | 指定範囲の断面算定実行 |

設計上の注意:
- すべての書き込み系ツール（編集・保存・上書き）は **破壊的操作を識別子付きで返し**、クライアント側で確認させる前提のレスポンス設計にする。
- 解析は時間がかかるため、**ジョブ投入＋進捗（resource/notification）**の非同期パターンを採用。
- 入力スキーマは `schemars` で型安全に生成。

**DoD:** MCP Inspector からモデル読込→解析→結果取得が一連で動作。スキーマがクライアントに正しく提示される。

## 15. GPU高速化オプション（squid-n-gpu） 【採用: wgpu + cubecl, feature = "gpu"】

CPUで全機能完結を保証した上で、以下を GPU にオフロードする任意機能とする。

対象処理（効果の高い順）:
1. **疎行列ベクトル積（SpMV）と PCG 反復** — 大規模線形解析・固有値の内部反復を高速化。
2. **要素剛性・内力の一括計算（ファイバ積分）** — プッシュオーバー/時刻歴で要素数×ファイバ数が大きい場合に有効。データ並列性が高い。
3. **ML 推論/学習**（`squid-n-ml` と GPU バックエンド共有）。

設計:
- `wgpu`（Vulkan/Metal/DX12）で移植性確保、計算カーネルは `cubecl` で記述（CUDA/wgpu バックエンドを切替可能）。
- GPUは**反復解法経路に限定**し、疎直接分解はCPU（faer）に残す（GPU疎直接は実装コスト大のため初期スコープ外）。
- **精度（R21）:** wgpu の f64 は実質非対応のため **GPU 経路は f32（必要に応じ混合精度）**。GPUは「高速スクリーニング／最適化ループ用の概算モード」と位置づけ、**最終解は CPU の f64 で確定**する。
- 数値の再現性：GPU経路は決定的でないことを明記し、検証はCPU基準解との許容差で行う。

**DoD:** SpMV/PCG が CPU 比で十分高速化（規模依存、目標は大規模で数倍）。GPU無効ビルドでも全テスト通過。

## 16. ML による断面提案（squid-n-ml） 【採用: burn（第一候補）】

### 16.1 位置づけ
設計者の初期断面選定を支援する**提案エンジン**。最終判断と適合性は必ず `squid-n-design-jp` の断面算定で検証する（MLは提案、判定は規準計算）。

### 16.2 アプローチ（段階導入）
1. **回帰/分類モデル（初期版）:** 入力＝スパン・荷重・階数・部材種別・想定軸力・地域係数等、出力＝推奨断面寸法・配筋クラス。教師データは過去設計＋規準計算で生成可能。
2. **代理モデル（サロゲート）:** プッシュオーバー/検定の結果（保有耐力・検定比）を近似し、候補断面の高速スクリーニングに使用。
3. **最適化ループ（将来）:** 提案→規準計算で検証→フィードバック（ベイズ最適化／簡易RL）。

### 16.3 実装
- **burn** でモデル定義・学習・推論。バックエンドは ndarray（CPU）／wgpu／CUDA を feature 切替。
- 学習データは `squid-n-design-jp` と `squid-n-solver` を回して**自動生成（合成データ）**＋過去案件で増強。
- モデルは onnx 互換または burn ネイティブ形式で配布、推論は MCP/GUI から呼出し。

**DoD:** 提案断面が規準算定で「検定比 ≤ 1.0」を高確率で満たす。提案→検証ループが MCP から実行可能。

## 20. リスクと留意事項

1. **壁谷澤モデルの定義揺れ:** 原著・改良版で係数・ばね配置が異なる。**採用文献を確定**し、係数を外部化。実験照合を必須に。
2. **規準の改定追従:** 設計式・係数をコードに埋め込まない（データ駆動＋版管理）。条文出典をコメントに残す。
3. **数値の決定性 vs 並列/GPU:** 並列集約順序を固定。GPU経路は非決定を許容しCPU基準で検証。
4. **非線形収束:** 弧長法・ステップ縮小・状態 rollback を最初から設計に織り込む（後付けは困難）。
5. **疎ソルバ選定:** faer の疎分解で性能不足なら、外部高性能ソルバ（CHOLMOD/MUMPS/Pardiso 等）への FFI 差し替えを `LinearSolver` トレイト越しに可能にしておく（抽象化で逃げ道確保）。
6. **MLは補助:** 提案の妥当性判断は必ず規準算定が行う。MLの出力を無検証で設計に使わせない。
7. **法的・責任範囲:** 本ソフトの算定結果は技術者の確認・判断を前提とする旨を出力・ドキュメントに明記。
8. **幾何剛性のスコープ:** P-Δ は任意機能・線形化のみ（共回転定式化は実装しない）。大変形が要る用途は対象外と明示する。
9. **パネルゾーン非線形:** v1 は弾性のみ。非線形パネルを後で足す際、弾性段階の釣り合い式（添付資料）と矛盾しない拡張点を確保しておく。

## 付録A. 命名・コーディング規約／確定した既定（抜粋）
- **単位（R2）:** 内部は N・mm・s（質量 N·s²/mm、応力 N/mm²）。入力は日本慣用（N/mm²・g/cm³・m・kN）で受け、境界変換。
- 物理量は内部単位固定、入出力境界でのみ単位変換。
- ID は `u32` newtype（`NodeId(u32)` 等）で型安全に。
- 公開APIは `thiserror` でエラー型を返し、`unwrap`/`panic` をライブラリ内で禁止（テスト除く）。
- `tracing` で各解析ステップに span を張り、計時を取得可能にする。
- **部材軸の向き（R24）:** 参照ベクトル方式でローカル座標を定義。鉛直材の既定はローカル強軸をグローバル +X 向きとする。
- **梁と柱（R25）:** 同一の3D骨組要素を梁・柱の双方に使用（要素を分けない）。非線形モデルの差は `ForceRegime` 属性で表現（6.8）。
- **ファイバのせん断（R26）:** 軸–曲げと別系統の独立せん断ばね（スケルトンは squid-n-skeleton 供給）。
- **`.scz` 結果内包の既定（R27）:** 結果が 50MB 未満なら内包、超過は外部ディレクトリ。
- **決定性の範囲（R28）:** 「CPU・単一スレッド設定で solve がビット一致」を保証。並列／Parquet書き出し／GPU はビット一致を保証外（値一致で検証）。
- **モデル所有権（R29）:** 単一ライタ方針。GUI／MCP の編集はコマンド（編集トランザクション）経由で直列化。
- **準拠規準の版（R30）:** 対象とする AIJ 規準・告示の版は技術リードが確定し、外部データとして版管理（11章）。

## 付録B. 用語
- **TVLEM:** Three Vertical Line Element Model（壁谷澤の連層壁マクロモデル）。
- **MS要素:** マルチスプリング要素。断面を軸ばね群に置換した塑性ヒンジモデル。
- **保有水平耐力:** 二次設計で確認する建物の終局水平耐力。Qun = Ds·Fes·Qud と比較。
- **限界耐力計算:** 等価1自由度化と応答スペクトルによる性能評価法。
- **Ai分布:** 地震層せん断力係数の高さ方向分布。略算は告示式、精算は固有値・モード解析を利用。Ci = Z·Rt·Ai·C0。
- **パネルゾーン:** 柱梁接合部のせん断パネル。フェースモーメント・パネルせん断 pQ で評価（添付資料準拠）。v1 は弾性（剛域近似または弾性せん断パネル）。
- **階（Story）:** ほぼ同一高さで床を構成する節点群の論理単位。剛床・Ai適用・層間変形・剛性率/偏心率の基盤。
- **CMQ（荷重項）:** 部材荷重による両端の固定端モーメント・せん断。床荷重分配の検証に図示。
- **ForceRegime:** 部材が受ける応力自由度（一軸曲げ＋せん断／軸力–曲げ相関）。非線形モデルの自動選択に使用。
- **フェースモーメント:** 柱・梁フェース（パネル枠）位置のモーメント。節点中心モーメントからせん断×半パネル寸法を差し引いて算定。断面算定に使用。
- **危険断面位置:** 断面算定・降伏判定を行う部材上の評価位置。既定は柱フェースと中央。
- **剛域:** 柱梁接合部の有限寸法により部材端が剛体的に挙動する領域。自動算定式 λ = D_orth/2 − D_self/4（負値は0）。
- **チェックポイント:** 長時間解析を再開可能にする状態スナップショット（変位・履歴状態・荷重係数等）。
- **ST-Bridge:** 日本の建築構造分野のデータ交換標準（XML）。他社一貫プログラム・BIM との相互運用に使用。
- **.scz:** 本ソフトのプロジェクトコンテナ（モデル・設定・結果を束ねた zip）。
