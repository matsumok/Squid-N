## 0. このフェーズについて

### 0.1 目的

骨組の**弾塑性挙動・終局機構**を、プッシュオーバー（非線形静的増分）で追跡できるようにする。
設計書 §6.3 / §6.8 / §10.3 / §10.6。

- **非線形要素**：集中ばね（one/two-component＋武田、P4 スケルトン使用）と**変位ベースのファイバ要素**
  （P4 FiberSection 使用）。`ElementBehavior` の `update_state`/`internal_force` を実装（P1 ではスタブ）。
- **非線形モデル自動選択**：`ForceRegime`（§6.8）で集中ばね／ファイバを部材ごとに自動割当。
- **プッシュオーバー**：荷重制御→**変位制御**→（最後に）弧長法。Newton–Raphson、収束判定、
  **ステップ縮小＋全体状態 rollback**、危険断面での降伏追跡、**ヒンジ発生順序・崩壊機構判定**、保有水平耐力。
- **線形化幾何剛性 Kg（P-Δ）**：任意機能（設計書 §10.6）。

### 0.2 完了像（ゴール）

設計書 §18 P5 完了基準「**弾塑性・終局機構・ヒンジ追跡**」を次で判定:

1. ばね単体・1層ラーメンで**理論的な弾塑性挙動・終局機構**が再現（§10.1、機構荷重は厳密）。
2. 危険断面（P1 §6.2.3）で**降伏・ヒンジ発生順序**が追跡され、**崩壊機構（全体／層崩壊）が自動判定**（§10.2）。
3. **変位制御で耐力低下枝（軟化）を追跡**できる（§10.3）。弧長法はスナップバックを追跡（§10.4、非ゲート）。
4. 幾何剛性 ON で軸力による固有振動数低下が理論傾向と一致、OFF で線形結果に一致（§10.5）。

## 2. T0: ElementBehavior の非線形化

設計書 §6.1。P1 でスタブだった `update_state`/`internal_force` を実装する。

```rust
// sc-element：ElementBehavior（P1 で定義済み）の非線形メソッドを実装
//   fn tangent_stiffness(&self, state, ctx) -> LocalMat   // 状態依存（接線）
//   fn internal_force(&self, state, ctx) -> LocalVec      // 現在状態の内力（釣合用）
//   fn update_state(&mut self, du, commit, ctx)           // ひずみ→応力・履歴更新。commit で確定
```

- `update_state(du, commit=false, ..)` は **試行**（材料 `trial`）、`commit=true` で**確定**（材料 `commit`）。
- 巻き戻しは要素状態 `ElemState` と材料の `revert` で行う（§6 の全体トランザクションが駆動）。

**DoD（T0）:** 線形材料の要素で、`internal_force ≈ K·u`、`tangent_stiffness` が一定。非線形材料で
状態が `update_state` により更新され、`commit`/巻き戻しが効く。

## 3. T1: 集中ばね要素（one/two-component＋武田）

設計書 §6.8 / §6.4（端部ばね型）。**剛床内で一軸曲げ＋せん断のみの大梁**に使う（軸力変動が無視できる）。

## 4. T2: 変位ベース ファイバ要素

## 5. T3: 非線形モデル自動選択（ForceRegime）

設計書 §6.8。**`ForceRegime::Auto` はトポロジで判定**（軸力変動は解く前に分からないので、計算値でなく
構造から判定するヒューリスティック）。

判定ルール（ユーザ上書き可）:

```
Auto のとき:
  剛床所属の階 かつ 梁（柱でない）かつ 種別的に軸力変動が小 → UniaxialBendingShear → 集中ばね
  それ以外（柱・ブレース・軸力変動部材）                    → AxialBendingInteract → ファイバ（既定）
```

- 1モデル内で**集中ばね部材とファイバ部材が混在してよい**（要素単位で独立）。
- ユーザが `ForceRegime` を明示すれば自動判定より優先。

```rust
// sc-element/src/factory.rs（非線形ビルド時）
/// 自動選択の確定結果（どの非線形要素を使うか）。
pub enum ResolvedRegime { ConcentratedSpring, Fiber }
pub fn resolve_force_regime(data: &ElementData, model: &Model) -> ResolvedRegime { /* 上記ルール */ }
```

**DoD（T3）:** 剛床梁→集中ばね、柱→ファイバが自動割当。混在モデルが解ける。ユーザ上書きが効く。

## 6. T4: ★全体 commit/rollback トランザクション（第一級）

設計書 §20-risk4「弧長法・ステップ縮小・状態 rollback を最初から設計に織り込む（後付けは困難）」。
**これはプッシュオーバーの最重要アーキテクチャ**。最初に作る。

### 6.1 なぜ必要か

非線形応答は**経路依存**。プッシュオーバーは「試行ステップ→Newton 反復（各反復で状態を試行更新）→
収束なら全確定／不収束ならステップ縮小して全巻き戻し」を繰り返す。**ある要素だけが確定して
ステップが後で棄却されると、解析全体が静かに壊れる**（履歴が不整合になる）。

### 6.2 規約（必須）

```rust
// sc-solver/src/transaction.rs
/// 全要素・全材料の確定状態のスナップショット（経路の1点）。
pub struct StateSnapshot { /* 全 ElemState・全材料 committed のコピー or 版番号 */ }

pub trait StatefulModel {
    fn snapshot(&self) -> StateSnapshot;          // 確定状態を取る
    fn restore(&mut self, snap: &StateSnapshot);  // 確定状態へ戻す（全要素一括）
    fn commit_all(&mut self);                     // 試行→確定（全要素一括）
}
```

- **スナップショットの所有者はプッシュオーバー駆動側**（個々の要素ではない）。
- **rollback はステップ単位で all-or-nothing。** 部分確定を禁止。
- **ステップ縮小は「直前の確定状態」から再入**する（経路を壊さない）。
- 決定性：経路依存・反復解法だが、**単一スレッドでは実行間ビット一致**（同じステップ縮小・同じ反復回数・
  同じ経路をたどる）。非決定なステップ縮小は履歴を分岐させるので禁止。

**DoD（T4）:** ステップを途中で棄却→`restore`→再実行で、無停止実行と同一の確定履歴に収束。
全要素が一括で確定／巻き戻しされる（部分確定が起きない）。

## 7. T5: プッシュオーバー（荷重／変位制御）

設計書 §10.3。

### 7.1 制御方式（順序）

1. **荷重制御**：弾性〜初期降伏まで。荷重増分で解く。
2. **変位制御（主力）**：代表層（最上階重心マスター節点）の変位を増分。**耐力低下枝（軟化）も
   屋根変位が単調増加なら追跡できる**＝本フェーズの目玉 DoD をこれで達成。
3. **弧長法**：§8（最後・非ゲート）。スナップバック（変位反転）専用。

> **水平力分布（設計書 §10.3/R7）:** 既定は **Ai分布比例（P2 §5.2）＋代表層 変位制御**。1次モード比例も選択可。

### 7.2 釣合反復（Newton–Raphson）

```
各ステップ:
  snapshot ← 確定状態（§6）
  目標増分（荷重 or 変位）を設定
  反復 i:
    K_t ← Σ tangent_stiffness（拘束縮約 P1§9・幾何剛性 §9 を含む）
    残差 r ← 外力 − Σ internal_force
    Δu ← solve(K_t, r)  （P0 ソルバ。変位制御は拘束行を入替）
    u += Δu;  全要素 update_state(Δu, commit=false)
    収束判定（力／変位／エネルギーノルムの複合）
  収束 → commit_all（§6）／ステップ記録
  不収束 → restore（§6）→ ステップ縮小して再入
```

- 修正 NR（接線を更新しない）も選択可。発散時はステップ縮小＋rollback（§6）。

### 7.3 降伏追跡・崩壊機構判定（設計書 §10.3）

各部材の危険断面（P1 §6.2.3）で M がスケルトン折点（ひび割れ／降伏／終局）に到達した
**ステップ・位置・塑性率**を記録し、**ヒンジ発生順序**と**崩壊機構（全体崩壊／層崩壊）**を自動判定する。

### 7.4 出力型（事前定義）

```rust
// sc-solver/src/pushover.rs
pub struct PushoverResult {
    /// 性能曲線：各ステップの (代表変位, ベースシア) ＋ 層せん断–層間変形。
    pub capacity_curve: Vec<CapacityPoint>,
    /// ヒンジ事象（発生順）。P7 の Ds 判定が読む。
    pub hinges: Vec<HingeEvent>,
    pub mechanism: MechanismType,
    pub qu: f64,                 // 保有水平耐力（機構形成時のベースシア）
}
pub struct CapacityPoint { pub step: u32, pub roof_disp: f64, pub base_shear: f64,
                           pub story_shear: Vec<f64>, pub story_drift: Vec<f64> }
pub struct HingeEvent { pub step: u32, pub elem: ElemId, pub pos: f64,
                        pub level: HingeLevel, pub ductility: f64 }
pub enum HingeLevel { Crack, Yield, Ultimate }
/// 崩壊機構：全体崩壊（高さ方向に分散）／層崩壊（1層に集中）。判定規則は §11.5。
pub enum MechanismType { Overall, StoryCollapse { story: StoryId }, Partial }
```

**DoD（T5）:** 1層ラーメンで終局機構・崩壊荷重が手計算一致。ヒンジ発生順序が記録され、機構が
正しく分類される。変位制御で軟化枝が追跡できる。

## 8. T6: 弧長法（arc-length）— 最後・非ゲート

設計書 §10.3。**スナップバック（変位反転）**を追跡するための制御。変位制御で取れない経路用。

**弧長拘束式（円筒型＝最も単純で実用十分。閉形式。Crisfield）。** 各ステップで増分変位 `Δu` と
荷重係数増分 `Δλ` を未知にし、弧長 `Δl` を一定に保つ:

```
拘束（円筒型）:   Δuᵀ·Δu = Δl²            （Δλ を含めない最単純形）
拘束（球面型）:   Δuᵀ·Δu + ψ²·Δλ²·(qᵀq) = Δl²   （q=参照荷重ベクトル, ψ=スケール）

反復: 接線解 δu = δū + Δλ·δu_t （δū=残差分, δu_t=K_t⁻¹q）を拘束式へ代入 →
      Δλ の2次方程式 a·Δλ² + b·Δλ + c = 0 を解き、前ステップ接線と同符号の根を選ぶ。
```

> **このタスクが遅れても P5 は成立する**（目玉 DoD は変位制御が担う。§7.1）。最後に実装する。
> 円筒型（上段）なら `Δl² = ΔuᵀΔu` の1本で済み、まず円筒型を実装すればよい。

**DoD（T6）:** 既知の軟化単自由度系（下降枝あり）でスナップバックを追跡（許容差）。

## 9. T7: 線形化幾何剛性 Kg（P-Δ）— 任意

設計書 §10.6。軸力から幾何剛性 `Kg` を構成し `K_t = K + Kg`。**共回転・大変形はやらない**（線形化のみ）。

**線形化幾何剛性 Kg の明示式（軸力 N、引張正。閉形式。文献転記不要）。** 曲げ面（uy,rz）の 4×4:

```
Kg(uy_i, rz_i, uy_j, rz_j) = (N/L)·
   [ 6/5     L/10     −6/5    L/10   ]
   [ L/10    2L²/15   −L/10  −L²/30  ]
   [ −6/5   −L/10      6/5   −L/10   ]
   [ L/10   −L²/30    −L/10   2L²/15 ]
```

xz 面（uz,ry）は、§4.1(P1) の曲げ規約（ry の向き）により**並進-回転結合項の符号が xy 面と逆**
になる（P1 §4.1 `K24 = −6·ay·L ← xy面と符号が逆` 参照。線形剛性・剛域変換 Tr・整合質量と同規約）。
対角項・回転-回転結合項は xy 面と同符号。軸・ねじりは 0。

```
Kg(uz_i, ry_i, uz_j, ry_j) = (N/L)·
   [ 6/5     −L/10    −6/5    −L/10  ]
   [ −L/10    2L²/15   L/10   −L²/30 ]
   [ −6/5     L/10      6/5    L/10  ]
   [ −L/10   −L²/30    L/10    2L²/15]
```

```rust
// sc-element：軸力 N（引張正）から線形化幾何剛性（梁の標準 Kg）を返す
fn geometric_stiffness(&self, n: f64) -> LocalMat {
    let l = self.length; let c = n / l;
    let mut kg = crate::behavior::LocalMat::zeros(12);
    let mut s = |i,j,v| { kg.set(i,j,v); if i!=j { kg.set(j,i,v); } };
    // xy面（uy=1,rz=5 / uy_j=7,rz_j=11）
    s(1,1, c*6.0/5.0); s(7,7, c*6.0/5.0); s(1,7, -c*6.0/5.0);
    s(1,5, c*l/10.0); s(1,11, c*l/10.0); s(5,7, -c*l/10.0); s(7,11, -c*l/10.0);
    s(5,5, c*2.0*l*l/15.0); s(11,11, c*2.0*l*l/15.0); s(5,11, -c*l*l/30.0);
    // xz面（uz=2,ry=4 / uz_j=8,ry_j=10）§4.1 規約で並進-回転結合項の符号が逆（ry の向き）
    s(2,2, c*6.0/5.0); s(8,8, c*6.0/5.0); s(2,8, -c*6.0/5.0);

}
```

## 11. 数式・理論補足（ジュニア向け）

### 11.1 Newton–Raphson（→ §7.2）
非線形の釣合 `内力(u) = 外力` を、接線剛性 `K_t = ∂内力/∂u` で線形化し `K_t·Δu = 残差` を反復で解く。
残差が小さくなれば収束。接線を毎回更新するのが完全 NR、固定が修正 NR（収束は遅いが分解再利用で速い）。

### 11.2 経路依存とトランザクション（→ §6）
塑性は経路依存（同じ変位でも履歴で応力が違う）。だから「試したが棄却したステップ」を確実に巻き戻す
必要がある。全要素・全材料の確定状態をスナップショットし、ステップ単位で all-or-nothing に
確定／巻き戻す。これを怠ると履歴が静かに壊れる（設計書 §20-risk4）。

### 11.3 変位制御 vs 弧長法（→ §7.1, §8）
荷重制御はピーク（最大耐力）で破綻する（増やせない）。**変位制御**は屋根変位を増やすので、耐力が
下がる軟化枝も追える。だが変位が反転する**スナップバック**は変位制御でも破綻する。そこだけ**弧長法**
（荷重係数と変位を同時に未知にし、弧長の拘束式で進む）が要る。建物のプッシュオーバーでスナップバックは
稀なので、弧長法は最後でよい。

### 11.4 幾何剛性 Kg（→ §9）
圧縮軸力は曲げ剛性を実効的に下げる（P-Δ 効果）。これを軸力に比例する `Kg` で線形化し `K+Kg` とする。
軸力が座屈荷重 `Pcr=π²EI/L²` に近づくと実効剛性→0、固有振動数→0。共回転（大回転）は扱わない。

### 11.5 崩壊機構の分類（→ §7.3）
全部材端にヒンジが分散して建物全体が機構化＝**全体崩壊**（望ましい）。特定の1層に塑性が集中して
その層だけ機構化＝**層崩壊**（危険、Ds が大きくなる）。ヒンジ分布が高さ方向に分散か1層集中かで分類する。
