## 0. このフェーズについて

### 0.1 目的

構造計算一貫プログラムの **土台となる4クレート** を作り、上位の解析・要素を載せられる
状態にする。具体的には:

- `squid-n-core` … 共通データモデル（節点・要素データ・断面・材料・階・拘束・スラブ・荷重）と
  自由度管理（DofMap）、単位系。
- `squid-n-math` … 線形ソルバ抽象（`LinearSolver` トレイト）と faer による疎直接法、
  **決定的な行列組立ヘルパ**。
- `squid-n-io` … モデルの保存・読込（`.scz` コンテナ、原子的保存、ハッシュ検証、スキーマ版）。
- workspace … 上記をまとめる Cargo workspace と、依存方向チェック等の `xtask`。

### 0.2 このフェーズの完了像（ゴール）

設計書 §18 の P0 完了基準「**線形静的が手計算一致／保存読込一致**」を、P0 のスコープに
合わせて次のように具体化する:

1. **手組みの疎行列 K と荷重 f** を `squid-n-math` のソルバに渡して `K x = f` を解き、
   手計算の解と相対誤差 1e-9 以内で一致する（§4 の 2自由度ばね系で検証）。
2. 同一系を **CPU・単一スレッドで 100 回解いてビット一致**（決定性。設計書 §5.3 / R28）。
3. モデルを `.scz` に保存→読込して **完全一致**（ラウンドトリップ。設計書 §4.4 / §12.6）。
4. 1万節点規模のモデルを生成・走査して **50ms 未満**（設計書 §4.4）。

> **重要:** P0 には「要素から全体剛性を自動で組み立てる処理」は**含まれない**（それは P1/P2）。
> したがって「線形静的一致」は *手で用意した K・f を解く* ことで判定する。要素の定式化
> （ティモシェンコ梁など）は P1 で実装する。

### 0.3 スコープ境界（含む／含まない）

| 項目 | P0 で | 備考 |
|---|---|---|
| データ構造 `Node`/`ElementData`/`Section`/`Material`/`Story`/`Slab`/`Constraint`/`Model` | **含む** | serde 対応の純データ（設計書 §4.2） |
| 単位系（内部 N・mm・s）と境界変換の基盤 | **含む** | §4.1。変換関数を `squid-n-core` に置く |

### 2.4 feature フラグ方針（設計書 §2.3）

重量級依存（GPU/ML/GUI/MCP）は **opt-in feature**。P0 では**スイッチだけ用意**して中身は空。
ルートに「全部入りビルド」用の集約クレートは作らず、各上位クレートの feature で制御する。
例（squid-n-gpu）:

```toml
[features]

```

**P0 の不変条件:** `cargo build`（feature 無し）でコア3クレートが完全にビルド・テストできること。

持たない（それは P1 の squid-n-element）。すべての主要型は `#[derive(Serialize, Deserialize)]` 可能にする。

ファイル構成（目安）:


### 3.1 単位系（units.rs）— 設計書 §4.1 / 付録A

**内部単位は常に N・mm・s** に固定する（不変条件）。質量 = N·s²/mm（= 10³ kg）、応力 = N/mm²。
入力は日本慣用単位で受け、**境界（squid-n-io / 入力層）で内部単位に変換して保存**する。

P0 では「変換関数を一箇所にまとめて提供する」ところまで実装する（呼び出しは squid-n-io / 後続UI）。

```rust
// squid-n-core/src/units.rs
/// 重力加速度 [mm/s^2]（9.80665 m/s^2）。重量↔質量変換に使う。
pub const GRAVITY_MM_S2: f64 = 9_806.65;

/// 入力 → 内部単位への変換関数群（純関数。すべて f64）。
pub mod to_internal {
    /// 長さ: m → mm
    pub fn length_m(m: f64) -> f64 { m * 1_000.0 }
    /// 力: kN → N
    pub fn force_kn(kn: f64) -> f64 { kn * 1_000.0 }
    /// 分布荷重: kN/m → N/mm
    pub fn line_load_kn_per_m(v: f64) -> f64 { v }          // kN/m = N/mm（数値同一）
    /// 面荷重: kN/m^2 → N/mm^2
    pub fn area_load_kn_per_m2(v: f64) -> f64 { v / 1_000.0 }
    /// 応力・強度: N/mm^2 はそのまま（内部表記と同一）
    pub fn stress_n_per_mm2(v: f64) -> f64 { v }
    /// 質量密度: g/cm^3 → 内部 [N·s^2/mm / mm^3] = [N·s^2/mm^4]
    /// 2.4 g/cm^3 → 2.4e-9
    pub fn mass_density_g_per_cm3(v: f64) -> f64 { v * 1.0e-9 }
    /// 単位体積重量: kN/m^3 → N/mm^3 （24 kN/m^3 → 2.4e-5）
    pub fn unit_weight_kn_per_m3(v: f64) -> f64 { v * 1.0e-6 }
    /// 重量 → 質量: W[N] / g → [N·s^2/mm]
    pub fn weight_n_to_mass(w_n: f64) -> f64 { w_n / super::GRAVITY_MM_S2 }
}
```

> **検算用の確定値（テストにそのまま使う）:**
> | 入力 | 変換関数 | 内部値 |
> |---|---|---|
> | `L = 6 m` | `length_m(6.0)` | `6000.0` mm |
> | `w = 10 kN/m` | `line_load_kn_per_m(10.0)` | `10.0` N/mm |
> | `P = 50 kN` | `force_kn(50.0)` | `50000.0` N |
> | `Fc = 24 N/mm²` | `stress_n_per_mm2(24.0)` | `24.0` N/mm² |
> | `ρ = 2.4 g/cm³` | `mass_density_g_per_cm3(2.4)` | `2.4e-9` N·s²/mm⁴ |
> | `γ = 24 kN/m³` | `unit_weight_kn_per_m3(24.0)` | `2.4e-5` N/mm³ |
> | `W = 1000 kN`（=1e6 N） | `weight_n_to_mass(1.0e6)` | `101.9716…` N·s²/mm（≈102 t） |
>
> 導出は §7.1 を参照。

### 3.2 ID（ids.rs）— 設計書 付録A

ID は `u32` の newtype にして型安全にする（`NodeId` に `ElemId` を渡す事故を防ぐ）。

```rust
// squid-n-core/src/ids.rs
macro_rules! id_newtype {
    ($name:ident) => {
        #[derive(Clone, Copy, PartialEq, Eq, Hash, PartialOrd, Ord, Debug,
                 serde::Serialize, serde::Deserialize)]
        pub struct $name(pub u32);
        impl $name { pub fn index(self) -> usize { self.0 as usize } }
    };
}
id_newtype!(NodeId);
id_newtype!(ElemId);
id_newtype!(StoryId);
id_newtype!(SlabId);
id_newtype!(SectionId);
id_newtype!(MaterialId);
id_newtype!(LoadCaseId);
```

> `index()` は「ID → `Vec` 添字」変換。設計書 §2.3 の DOD（`Vec<T>`＋`u32` 添字）に合わせる。
> P0 の前提として **ID = 配列添字が一致するように生成する**（ID `n` は `vec[n]`）。
> 不連続IDの対応（HashMap）は P1 以降の拡張点。

### 3.3 自由度（dof.rs の前半）— 設計書 §4.1

各節点は 3次元 6自由度（並進 X/Y/Z、回転 RX/RY/RZ）。

```rust
// squid-n-core/src/dof.rs
/// 1節点内の自由度番号（0..6）。
#[derive(Clone, Copy, PartialEq, Eq, Debug, serde::Serialize, serde::Deserialize)]
pub enum Dof { Ux = 0, Uy = 1, Uz = 2, Rx = 3, Ry = 4, Rz = 5 }
pub const DOF_PER_NODE: usize = 6;

/// 6自由度の拘束マスク。bit i が立っていれば自由度 i は固定（支点）。
#[derive(Clone, Copy, Default, PartialEq, Eq, Debug,
         serde::Serialize, serde::Deserialize)]
pub struct Dof6Mask(pub u8);

impl Dof6Mask {
    pub const FREE: Self = Dof6Mask(0b000000);
    pub const FIXED: Self = Dof6Mask(0b111111);          // 完全固定
    pub const PINNED: Self = Dof6Mask(0b000111);         // 並進拘束・回転自由
    pub fn is_fixed(self, d: Dof) -> bool { self.0 & (1 << d as u8) != 0 }
    pub fn set_fixed(&mut self, d: Dof) { self.0 |= 1 << d as u8; }
}
```

### 3.5 要素データ（model.rs）— 設計書 §4.2

`ElementData` は**純データ**。挙動（`ElementBehavior`）は P1 の squid-n-element で実装する。

```rust
#[derive(Clone, Copy, Debug, PartialEq, Eq, serde::Serialize, serde::Deserialize)]
pub enum ElementKind { Beam, Shell, Fiber, Ms, Wall, PanelZone }
```

### 3.12 DofMap（dof.rs の後半）— 設計書 §4.3【P0 は支点拘束のみ】

**目的:** 全節点の 6自由度のうち「支点で固定された自由度」を除いた **独立自由度** に通し番号
（`active` インデックス）を振る。全ソルバはこの番号で行列の行・列を扱う。

> **P0 の範囲:** 拘束（剛床/MPC）による「従属自由度の縮約」は**やらない**（P1）。
> P0 でやるのは「`Dof6Mask` で固定された自由度を飛ばして採番する」ことだけ。

```rust
// squid-n-core/src/dof.rs（後半）
use crate::model::Model;

/// 全体自由度番号 = node.index() * 6 + dof。
pub type GlobalDof = usize;

#[derive(Clone, Debug, Default)]
pub struct DofMap {
    /// global_dof -> Some(active番号) / None(拘束されている)
    active_of: Vec<Option<u32>>,
    /// active番号 -> global_dof（逆引き）
    global_of: Vec<GlobalDof>,
    n_active: usize,
}

impl DofMap {
    /// 支点拘束だけを考慮して独立自由度を採番する。
    /// 採番順は「node 昇順 × dof 昇順」で固定（決定性のため）。
    pub fn build(model: &Model) -> Self {
        let n_global = model.nodes.len() * DOF_PER_NODE;
        let mut active_of = vec![None; n_global];
        let mut global_of = Vec::new();
        let mut counter = 0u32;
        for (ni, node) in model.nodes.iter().enumerate() {
            for d in 0..DOF_PER_NODE {
                let g = ni * DOF_PER_NODE + d;
                let dof = match d { 0=>Dof::Ux,1=>Dof::Uy,2=>Dof::Uz,3=>Dof::Rx,4=>Dof::Ry,_=>Dof::Rz };
                if !node.restraint.is_fixed(dof) {
                    active_of[g] = Some(counter);
                    global_of.push(g);
                    counter += 1;
                }
            }
        }
        DofMap { active_of, global_of, n_active: counter as usize }
    }

    pub fn n_active(&self) -> usize { self.n_active }
    /// 全体自由度 → 独立自由度番号（拘束なら None）
    pub fn active(&self, g: GlobalDof) -> Option<u32> { self.active_of[g] }
    /// 独立自由度番号 → 全体自由度
    pub fn global(&self, a: u32) -> GlobalDof { self.global_of[a as usize] }
}
```

### 4.1 ソルバ抽象（solver.rs）— 設計書 §5.1

```rust
// squid-n-math/src/solver.rs
use faer::sparse::SparseColMat;

/// K x = f を解く。分解（factorize）を1回・後退代入を多数回できるよう、
/// 実装は内部に分解結果を保持する設計にする（同一 K で複数 rhs を解くため）。
pub trait LinearSolver {
    /// 係数行列 K を与えて分解する（対称正定値を仮定）。
    fn factorize(&mut self, k: &SparseColMat<usize, f64>) -> Result<(), SolveError>;
    /// 分解済みの K に対して右辺 f を解く。複数回呼べる。
    fn solve(&self, rhs: &[f64]) -> Result<Vec<f64>, SolveError>;
}

#[derive(Debug, thiserror::Error)]
pub enum SolveError {
    #[error("not factorized yet")] NotFactorized,
    #[error("matrix not positive definite")] NotPositiveDefinite,
    #[error("dimension mismatch: K={k}, rhs={rhs}")] DimMismatch { k: usize, rhs: usize },
    #[error("faer error: {0}")] Backend(String),
}
```

> **設計意図（ジュニア向け補足）:** 構造解析では「同じ K に対して荷重ケースを何通りも解く」のが普通。
> 行列分解は重い処理なので **1回だけ分解 → 各荷重で後退代入** が鉄則（設計書 §5.2）。
> そのため `factorize` と `solve` を分けている。

### 4.3 faer 疎直接法（cholesky.rs）— 設計書 §5.2

**faer 0.24 の実 API（docs.rs で確認済み・そのまま使える）。** 調べ直す必要はない。以下を写経して埋める。

確定している API（faer 0.24.0）:
- 構築: `SparseColMat::<usize,f64>::try_new_from_triplets(nrows, ncols, &[Triplet]) -> Result<Self, CreationError>`。
  `Triplet` は `faer::sparse::Triplet::new(row, col, val)`、フィールドは `.row/.col/.val`。
- シンボリック分解: `faer::sparse::linalg::solvers::SymbolicLlt::try_new(mat.symbolic(), Side::Lower) -> Result<Self, FaerError>`。
  **`Side::Lower` を渡すと下三角のみ参照**（＝対称全体を入れてよい。上三角は無視される ⇒ §4.4 の「全体 K を組む」で OK）。
- 数値分解: `faer::sparse::linalg::solvers::Llt::try_new_with_symbolic(symbolic, mat.as_ref(), Side::Lower) -> Result<Self, LltError>`。
- 求解: `Solve` トレイト（`use faer::linalg::solvers::Solve;`）の `llt.solve(rhs_mat_ref) -> Mat<f64>`（新規行列で返す。複数回呼べる）。

```rust
// squid-n-math/src/cholesky.rs
use faer::sparse::SparseColMat;
use faer::sparse::linalg::solvers::{Llt, SymbolicLlt};
use faer::linalg::solvers::Solve;   // .solve() を生やすブランケットトレイト
use faer::{Mat, Side};
use crate::solver::{LinearSolver, SolveError};

#[derive(Default)]
pub struct CholeskySolver {
    factor: Option<Llt<usize, f64>>,   // 分解結果（シンボリック＋数値）を保持
    n: usize,
}

impl LinearSolver for CholeskySolver {
    fn factorize(&mut self, k: &SparseColMat<usize, f64>) -> Result<(), SolveError> {
        self.n = k.nrows();
        // 1. シンボリック分解（フィルイン削減の並べ替え）。Side::Lower で下三角のみ参照。
        let symbolic = SymbolicLlt::try_new(k.symbolic(), Side::Lower)
            .map_err(|e| SolveError::Backend(format!("symbolic: {e:?}")))?;
        // 2. 数値分解（LLᵀ）。非正定値なら Err → NotPositiveDefinite。
        let llt = Llt::try_new_with_symbolic(symbolic, k.as_ref(), Side::Lower)
            .map_err(|_| SolveError::NotPositiveDefinite)?;
        self.factor = Some(llt);
        Ok(())
    }

    fn solve(&self, rhs: &[f64]) -> Result<Vec<f64>, SolveError> {
        let llt = self.factor.as_ref().ok_or(SolveError::NotFactorized)?;
        if rhs.len() != self.n {
            return Err(SolveError::DimMismatch { k: self.n, rhs: rhs.len() });
        }
        let b = Mat::from_fn(self.n, 1, |i, _| rhs[i]);   // n×1 列ベクトル
        let x = llt.solve(b.as_ref());                     // 前進後退代入（分解は再利用）
        Ok((0..self.n).map(|i| x[(i, 0)]).collect())
    }
}
```

> **補足:** P0 で扱う K は「対称正定値」（拘束済みの剛性行列の性質）。非正定値なら
> `Llt::try_new_with_symbolic` が `Err` を返すので `NotPositiveDefinite` に写す。LU（非対称・
> `solvers::Lu`）や LDLT（不定）は P1 以降。**トレイトの import 行（`Solve` 等）はコンパイラが
> 不足を教えるので、調査ではなく機械的に追加すればよい**（関数名・シグネチャは上記で確定）。

### 4.4 決定的トリプレット→疎行列 組立（sparse.rs）— 設計書 §5.2 / §5.3

**なぜ必要か（ジュニア向け）:** 浮動小数の加算は結合則が成り立たない（`(a+b)+c ≠ a+(b+c)` が起き得る）。
要素剛性を並列に計算して同じ位置へ足し込むと、**足す順序がブレて結果が毎回変わる**ことがある。
これを防ぐため、**トリプレット（行,列,値）を貯めてから決定的な順序で集約**する。

```rust
// squid-n-math/src/sparse.rs
use faer::sparse::SparseColMat;

/// 1要素ぶんの寄与。(row, col, value)。行列は対称だが、ここでは与えられた通り格納する。
#[derive(Clone, Copy, Debug)]
pub struct Triplet { pub row: usize, pub col: usize, pub val: f64 }

/// トリプレット列から、決定的に CSC 疎行列を組み立てる。
///
/// 手順:
///  1. (col, row) の辞書順で **安定ソート**（同一位置の寄与を隣接させる）。
///  2. 同一 (row, col) の値を **ソート後の順序で順番に**加算（並列で足さない）。
///  3. faer の SparseColMat に変換して返す。
///
/// これにより、入力トリプレットの生成順が並列で前後しても、
/// (col,row) ソート後の集約順序は一意 → ビット一致が保証される。
pub fn assemble_csc(n: usize, mut triplets: Vec<Triplet>) -> SparseColMat<usize, f64> {
    // 1. (col,row) 辞書順で安定ソート（同一位置を隣接させる）
    triplets.sort_by(|a, b| (a.col, a.row).cmp(&(b.col, b.row)));
    // 2. 同一 (row,col) を順番に加算（決定的）。faer の Triplet 列へ畳み込む。
    let mut merged: Vec<faer::sparse::Triplet<usize, usize, f64>> = Vec::with_capacity(triplets.len());
    for t in triplets {
        match merged.last_mut() {
            Some(m) if m.row == t.row && m.col == t.col => m.val += t.val,
            _ => merged.push(faer::sparse::Triplet::new(t.row, t.col, t.val)),
        }
    }
    // 3. faer 0.24 の確定 API：try_new_from_triplets(nrows, ncols, &[Triplet])
    SparseColMat::try_new_from_triplets(n, n, &merged).expect("valid triplets")
}
```

> **並列との関係:** 「要素ローカル剛性の計算」は将来 rayon で並列化してよい（重い処理）。
> しかし「集約（足し込み）」は上記のとおり**決定的順序**にする。P0 では要素が無いので、
> `assemble_csc` 単体の決定性をテストすればよい（§6.5）。

> **⚠ faer の並列実行と決定性（重要・API確定済み）:** faer はグローバル並列度設定を持つ。
> 分解が並列で走ると浮動小数の集約順序が変わり**ビット一致が崩れる**ことがある。
> 決定性（§0.2 ゴール2 / R28）は **CPU・単一スレッド**を前提とする。決定性テスト（およびビット一致を
> 要する経路）では、解く前に **`faer::set_global_parallelism(faer::Par::Seq)`** を呼んでシリアル固定する
> （`Par` は `faer` 直下、`Par::Seq` が逐次。faer 0.24 確定 API）。P0 の 2×2 系は小さく並列化されないため
> 実害は出にくいが、P1/P2 で規模が増えると効くので**今のうちに入口を作る**。

## 7. 数式・理論補足（ジュニア向け）

### 7.1 単位系と変換の導出（→ §3.1）

内部単位は **長さ mm・力 N・時間 s**。ここから他の単位が決まる。

- **応力** = 力/面積 = N/mm²（数値は MPa と同じだが、日本慣用に合わせ表記は N/mm²）。
- **質量**: 運動方程式 `F = m·a` より `m = F/a`。内部では `F[N]`, `a[mm/s²]` なので
  `m` の単位は `N·s²/mm`。これは SI の kg と `1 [N·s²/mm] = 1000 kg` の関係:
  `1 N = 1 kg·(m/s²) = 1 kg·(1000 mm/s²)` なので `1 N·s²/mm = (kg·1000 mm/s²)·s²/mm = 1000 kg`。
- **質量密度** g/cm³ → 内部 `N·s²/mm⁴`:
  `2.4 g/cm³ = 2400 kg/m³ = 2400 kg / 1e9 mm³`。
  `kg → N·s²/mm` は `÷1000` なので `2400/1000 / 1e9 = 2.4e-9 [N·s²/mm⁴]`。
- **単位体積重量** kN/m³ → N/mm³: `24 kN/m³ = 24000 N / 1e9 mm³ = 2.4e-5 N/mm³`。
- **重量→質量**: `m = W/g`、`g = 9806.65 mm/s²`。`W=1e6 N → m = 1e6/9806.65 = 101.97 N·s²/mm`。

### 7.2 2自由度ばね系（ソルバ検証モデルの導出）（→ §4.6, §6.5）

直列ばね2本。地面—（k1）—節点1—（k2）—節点2。節点2 に荷重 P。

剛性行列（各ばねの寄与を足し込む）:

```
K = | k1+k2   -k2 |     f = | 0 |
    |  -k2     k2 |         | P |
```

`k1=100, k2=200, P=1000` を入れると `K=[[300,-200],[-200,200]]`, `f=[0,1000]`。

手計算（直列ばねは変位が累積する）:
- `x1 = P/k1 = 1000/100 = 10`
- `x2 = P/k1 + P/k2 = 10 + 5 = 15`

`K x = f` を解くとこの `x=[10,15]` になる（代入確認:
`300·10 − 200·15 = 3000−3000 = 0` ✔、`−200·10 + 200·15 = −2000+3000 = 1000` ✔）。

### 7.3 自由度の採番と支点拘束（→ §3.12）

全節点は 6自由度。全体自由度番号は `g = node_index*6 + dof`。
支点で固定された自由度は方程式から除く（変位が既知=0）。残った**独立自由度**だけに
連番 `active` を振り、剛性行列はこの `active` 次元（`n_active × n_active`）で構成する。
P0 ではここまで（剛床/MPC による従属関係の縮約は P1 で `K_red = TᵀKT` として実装）。

### 7.4 疎行列 CSC と決定的組立（→ §4.4）

- **疎行列**: 大半が 0 の行列。0 を保存せず「非ゼロの位置と値」だけ持つ。
- **CSC（Compressed Sparse Column）**: 列ごとに非ゼロをまとめた格納形式。faer の `SparseColMat` がこれ。
- **決定性**: §4.4 のとおり、トリプレットを `(col,row)` で安定ソートしてから順番に加算することで、
  並列生成の順序揺れに依存しない一意な結果を得る（設計書 §5.3 / R28: 決定性は **CPU 単一スレッド**で保証）。

### 7.5 疎 Cholesky（疎直接法）の考え方（→ §4.3）

対称正定値行列 `K` は `K = L Lᵀ`（下三角 `L`）に分解できる（Cholesky 分解）。
これがあれば `Kx=f` は `Ly=f`（前進代入）→ `Lᵀx=y`（後退代入）の2段で解ける。
分解が重く・代入が軽いので、**分解1回／代入多数**にする（§4.1）。疎行列では分解中に
新たな非ゼロ（フィルイン）が生じるため、事前の並べ替え（順序付け）でこれを減らす ──
この一切を faer の疎 Cholesky が担う。**自前では書かない**（設計書 §0 の方針）。

## 8. 用語（P0 で出てくるもの）

| 用語 | 意味 |
|---|---|
| 自由度（DOF） | 節点が動ける独立方向。本システムは並進3＋回転3＝6/節点 |
| 独立自由度（active dof） | 支点拘束・従属を除いた、実際に解く未知数。P0 は支点拘束のみ考慮 |
| 剛性行列 K | 変位と力を結ぶ行列。`K x = f` |
| 疎行列 | ほとんどが 0 の行列。非ゼロだけを格納（CSC 形式） |
| 直接法（Cholesky） | 行列分解で `Kx=f` を厳密に解く方法。P0 の主力 |
| トリプレット | (行, 列, 値) の3つ組。疎行列の組み立て素材 |
| 決定性 | 同一入力で同一出力（ビット一致）。CPU 単一スレッドで保証（R28） |
| 剛床ダイアフラム | 床面内を剛体扱いする拘束。**データ定義は P0／縮約演算は P1** |
| MPC | 多点拘束。1対多の線形拘束。**データ定義は P0／縮約は P1** |
| `.scz` | 本ソフトのプロジェクトファイル（zip コンテナ） |
| schema_version | 保存形式の版。読込時にマイグレーションで最新化 |
