### 1.1 横断ルール（構造ソフト特有）

- **工程タブはフロー順だが進行ロックしない**。構造設計は反復（断面変更→再計算）なので、いつでも前工程へ
  戻れ、モデルは常に編集可能。
- **3D とモデル状態は常時アクセス可能**。ナビゲータ選択 ⇄ 3Dピック ⇄ テーブル行が**双方向連動**。
- **stale（要再計算）の可視化**（§5）。
- 全編集は**編集トランザクション**（P3 §3.3）経由＝Undo可・MCP（P8）と共通経路（単一ライタ R29）。

## 3. 3D選択 → 断面編集（共有編集＋複製）

- 3D/ナビゲータ/テーブルで部材選択 → **右インスペクタに断面（`SectionShape`＋算定プロパティ）を表示**。
- **編集はその断面（SectionId）を参照する全部材に波及**。インスペクタに
  **「この断面を使う N 部材に影響します」**と影響数を常時表示（誤爆防止）。
- 並びに**「複製してこの部材だけ別断面に」ボタン** → 新 SectionId を生成し当該部材のみ再割当。
- いずれも `EditCommand`（P3 §3.3）として適用 → **Undo可・MCP と共通**。

```rust
// sc-app/src/command.rs（編集コマンドの例。sc-edit に置く）
pub struct EditSectionShape { pub section: SectionId, pub new_shape: SectionShape } // 共有断面を編集（波及）
pub struct DuplicateSectionForMember { pub member: ElemId }                          // 複製→当該部材のみ再割当
```

**DoD:** 断面編集が共有部材に波及し影響数が出る。複製ボタンで当該部材のみ別断面化。両操作が Undo 可能。

## 4. 断面作成UI（パラメトリック `SectionShape`）

### 4.1 断面タイプとパラメータ

```
鋼材:
  H形    : H(せい), B(幅), tw(ウェブ厚), tf(フランジ厚), r(フィレット任意)
  箱形/角形鋼管 : H, B, t（板厚）
  L形(山形)     : A, B(脚長), t
  C形(溝形)     : H, B, tw, tf
  T形           : H, B, tw, tf
  円形(鋼管)    : D(径), t（板厚）

RC:
  矩形 : B × D
  円形 : D
  配筋:
    主筋   : X方向・Y方向 別に〔本数・径(D10〜D41)・段数・かぶり〕
    せん断補強筋(帯筋/あばら筋) : 径・ピッチ・組数(脚数)
```

- **断面プレビュー**（寸法図＋RCは配筋図）をインスペクタ内に描画。
- 規格断面ライブラリ（JIS 形鋼）からの選択は**任意拡張**（パラメトリック入力が基本）。

### 4.2 データモデル（新設 `SectionShape`）

現行 `sc-core::Section` は**計算済みプロパティ**（A/I/J/As/depth/width…）。その上流に
**パラメトリック定義 `SectionShape`** を新設し、プロパティを**自動算定**して `Section` を生成する。

```rust
// sc-core/src/section_shape.rs（新設）
#[derive(Clone, Debug, serde::Serialize, serde::Deserialize)]
pub enum SectionShape {
    SteelH    { h: f64, b: f64, tw: f64, tf: f64, r: f64 },
    SteelBox  { h: f64, b: f64, t: f64 },          // 角形鋼管
    SteelAngle{ a: f64, b: f64, t: f64 },          // L形
    SteelChannel { h: f64, b: f64, tw: f64, tf: f64 }, // C形
    SteelTee  { h: f64, b: f64, tw: f64, tf: f64 },
    SteelPipe { d: f64, t: f64 },                  // 円形鋼管
    RcRect    { b: f64, d: f64, rebar: RcRebar },
    RcCircle  { d: f64, rebar: RcRebar },
}

#[derive(Clone, Debug, serde::Serialize, serde::Deserialize)]
pub struct RcRebar {
    pub main_x: BarSet,   // X方向 主筋
    pub main_y: BarSet,   // Y方向 主筋
    pub cover: f64,       // かぶり
    pub shear: ShearBar,  // せん断補強筋
}
#[derive(Clone, Debug, serde::Serialize, serde::Deserialize)]
pub struct BarSet { pub count: u32, pub dia: f64, pub layers: u32 } // 本数・径・段数
#[derive(Clone, Debug, serde::Serialize, serde::Deserialize)]
pub struct ShearBar { pub dia: f64, pub pitch: f64, pub legs: u32 } // 径・ピッチ・組数

impl SectionShape {
    /// 形状・寸法から断面性能（A, Iy, Iz, J, As_y, As_z, depth, width）を算定し Section を作る。
    /// 鋼の閉形式・矩形/円の積分は明示（実装で埋める）。
    pub fn to_section(&self, id: SectionId, name: String) -> Section { todo!() }
}
```

- **`Section` は `SectionShape` の派生**（`SectionShape` を保持し、プロパティはキャッシュ）。寸法変更で再算定。
- **配筋 `RcRebar`** は P4 `Reinforcement`（[P4 §7](P4_材料と断面.md)）へ供給：ファイバ断面・スケルトン・
  P7 の Ds 耐力比に使う。弾性剛性は原則 gross（ひび割れ考慮は P4/非線形側）。

**DoD（§4）:** 各タイプで寸法→A/I/J/As が手計算一致（矩形 `A=BD`, `Iz=BD³/12` 等。鋼Hは部材表と照合）。
RC 配筋がプレビューに反映され、P4 のファイバ断面へ渡る。

## 5. stale（要再計算）と実行トリガ

**方針：手動再計算＋stale表示**（重い解析を勝手に走らせない）。

- モデル/荷重編集 → 依存する下流（結果・設計）に **「要再計算」バッジ（オレンジ）**。
- 工程タブに状態アイコン **✓最新 ／ ⚠stale ／ ▷未実行**。
- 主トリガは**解析タブの実行ボタン**。結果/設計タブ上部に「最新解析時刻 ／ 現モデルと不一致」を表示。
- **依存追跡**は編集トランザクションのスコープから判定（例：ある断面の変更→その断面を使う部材の結果のみ stale）。
- **部分再計算**：荷重のみ変更は後退代入のみ（同一 K 再利用、P2 §3）、断面/形状変更は再組立。
  決定性（R28）・チェックポイント（P6 §5）と整合。

```rust
// sc-app/src/app.rs
pub struct Staleness { pub results_stale: bool, pub design_stale: bool, pub last_run: Option<std::time::SystemTime> }
```

**DoD（§5）:** 編集で下流が stale 表示。実行で最新化。荷重のみ変更は部分再計算で速い。

## 6. 結果可視化の種類

- 変形図（静的・モード、倍率スライダ／アニメ）
- 部材応力図 **N／Q／M**（部材ローカル・評価位置、P1 §6.2.3）
- **CMQ図**（荷重項、フェースモーメント位置併記、P2 §5.1）
- シェル応力コンター（膜力・モーメント、P1.5）
- モード形（モード選択・固有周期表示、P2 §4）
- **ヒンジ状態**（crack/yield/ultimate 色分け・発生順アニメ、P5 §7.4）
- **性能曲線**（容量曲線・性能点、P12 §5）
- 時刻歴グラフ（節点応答・層せん断・層間変形角の時系列、P6）
- 層レベル（層せん断–層間変形・剛性率・偏心率）
- **検定比色分け（＝断面算定結果）**：緑/黄/赤、NG部材ハイライト、クリックで内訳（設計タブと連動）
- 共通：凡例・カラースケール・断面位置プローブ。

**DoD（§6）:** 各可視化が対応フェーズの結果から描ける。検定比色分けが設計結果と一致し 3D/表が連動。

## 7. 入力支援

- スプレッドシート同等：**コピー&ペースト（Excel往復）・一括編集・フィル・Undo/Redo**。
- **即時バリデーション**：不正セル赤背景＋理由ツールチップ（`sc_core::Model::validate` と同規則、P0 §3.11）。
- 一括生成：通り芯グリッド→節点・部材自動生成、階の複製、断面の一括割当。
- 単位：入力＝日本慣用、内部 N·mm·s、境界変換（P0 §3.1）、表示単位の切替。
- 仮想スクロール（数万行、egui_extras `body.rows`、P3 §3.1）・フィルタ/ソート/検索。
- CSV 入出力（P8 §7）。

**DoD（§7）:** 1万部材でコピペ・一括・スクロールが滑らか。不正入力が即赤表示。

**DoD（§8）:** 章を選んで PDF/Excel を出力。図表が解析・設計結果と一致。出典・条文が明記される。

## 9. 実装スケジュール（いつ・どの順で実装するか）

> UI は単独フェーズではなく、**各フェーズに UI タスクを差し込む**形で段階導入する。下表で
> 「どの UI 項目を・どのフェーズで・何を前提に・どのタスク（UI-x）として・何を満たせば完了か」を確定する。
> 各 UI-x タスクは、対応フェーズの仕様書のタスク一覧に**追加して**実装する（§9.5 の反映先）。

### 9.1 着手の大原則（順序の根拠）

1. **データが先、UI が後**：UI が編集・表示する対象（`SectionShape` 等）を先にデータ層へ入れる。
2. **土台 → 機能拡充**：4ペイン・工程タブ・3D・編集トランザクション・stale は P3 で**土台**を作り、
   以降のフェーズで結果が出るたびに**結果可視化を足す**。
3. **断面タイプは増分**：鋼 H形・RC 矩形＋基本配筋を先行。他タイプは `SectionShape` の enum 追加で順次。

## 11. 用語（UI で増えるもの）
| 用語 | 意味 |
|---|---|
| 工程タブ | 解析フロー順のタブ（モデル→…→レポート）。進行ロックしない |
| 4ペイン | 工程タブ＋ナビゲータ＋中央(3D/表)＋インスペクタ＋下部ステータス |
| ナビゲータ | 階/部材群/ケースのツリー（中央と連動） |
| インスペクタ | 右ペイン。選択要素の諸元・断面編集・検定サマリ |
| SectionShape | パラメトリック断面定義（形状＋寸法＋配筋）。Section を算定 |
| stale | モデル変更で結果/設計が古くなった状態（要再計算） |
| 共有編集 | 断面編集が同断面参照の全部材へ波及すること |
| 検定比色分け | 断面算定結果を 3D 上で緑/黄/赤に可視化 |
