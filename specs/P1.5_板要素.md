## 0. このフェーズについて

### 0.1 目的

床・スラブを扱う **4節点シェル要素（MITC4）** を `sc-element` に追加する。設計書 §6.5 に対応。

- **膜（面内）＋曲げ（面外）＋せん断**を扱う汎用シェル。
- せん断ロッキング回避に **MITC4（仮定ひずみ法 / Dvorkin–Bathe）**。
- 面内は適合要素＋**ドリリング自由度の安定化**（Allman/Cook 流）。
- 床の主用途は「面内剛性（柔床時）」と「鉛直荷重伝達・面外曲げ」。
- **剛床ダイアフラム適用階では面内（膜＋ドリリング）剛性を無効化**（R13。P1 §9.1 から引継ぎ）。

### 0.2 完了像（ゴール）— ★検証哲学が P1 と異なる

> **⚠ 最重要：MITC4 は「厳密一致」ではなく「収束」する。**
> P1 の梁（IIE）は対象問題で*厳密解*だったので相対 1e-6 で一致を要求した。
> **MITC4 板は厳密解ではなく、メッシュを細かくすると参照解に収束する**要素である。
> **正しい実装でも粗メッシュでは 1〜2% 程度の誤差が出るのが正常。**
> したがって板のたわみ DoD は「厳密一致」ではなく**収束テスト**で判定する:
> 1. メッシュ密度と許容帯を指定（例: 16×16 で参照解の **±2% 以内**）。
> 2. **細分化で誤差が単調に減少**して参照解へ近づくこと（粗→密で誤差が縮む）。
> 唯一の*厳密*チェックは **パッチテスト**（機械精度で定数応力を再現。§9.2）。
> この区別を理解しないと、ジュニアは「正しい実装」を「バグ」と誤認する。

判定テスト（詳細 §9）:

1. **パッチテスト**：歪んだ四辺形メッシュで、境界に線形変位を与え、内部が**定数応力を機械精度で再現**
   （膜状態・定曲げ状態の両方）。
2. **単純支持 正方形板**（等分布荷重）：中央たわみが理論係数 `α·q·a⁴/D` に細分化収束（§9.3）。
3. **クランプ（固定）正方形板**：同上（§9.3）。
4. **片持ち板**：梁的サニティチェック（厳密な板閉形式は無いので近似確認。§9.4）。
5. **剛床面内無効化**：剛床階のシェルで膜＋ドリリングが効かず、曲げが残ることを確認（§9.5）。
6. 組立・縮約は **CPU 単一スレッドでビット一致**（決定性。P0/P1 を踏襲）。

## 3. T1: シェル要素の自由度構成

設計書 §6.5。4節点・各節点6自由度に組み込むための骨格を作る。

ファイル: `sc-element/src/shell.rs`（新規）

### 3.1 自由度の構成（★ドリリング自由度の扱い）

- 平面 4節点シェルは**力学的には 5 自由度/節点**（並進3＋面内2回転）で完結する。法線まわりの
  回転（**ドリリング自由度** = 6番目の回転 Rz_local）は本来の剛性を持たない。
- だが本システムは**梁と共有する 6 自由度/節点**へ組み立てる（P0 §3.3）。よって6番目（ドリリング）
  を持たせ、§5 の安定化で剛性を与える（ゼロのままだと特異になる。§3.3）。
- 要素は **4節点 × 6自由度 = 24 自由度**。剛性は 3つの寄与の和:
  - **膜（メンブレン）**: 面内並進 (Ux, Uy) ＋ ドリリング (Rz)
  - **曲げ（プレート）**: 面外並進 (Uz) ＋ 面内回転 (Rx, Ry)
  - **横せん断**: 曲げと連成（MITC4 で扱う。§4）

```rust
// sc-element/src/shell.rs
use sc_core::ids::NodeId;

pub struct ShellElement {
    pub nodes: [NodeId; 4],
    pub t: f64,                 // 板厚
    pub e: f64, pub nu: f64,    // 弾性定数
    pub frame: ShellFrame,      // 要素ローカル座標（面内 x,y と法線 z）
    pub drilling_factor: f64,   // ドリリング安定化係数（§5.2）
    pub membrane_active: bool,  // 剛床階で false（§7）。曲げは常に有効
}

/// 4節点から要素ローカル座標（面内2軸＋法線）を作る。
pub struct ShellFrame { pub e1: [f64;3], pub e2: [f64;3], pub n: [f64;3] }
impl ShellFrame { pub fn from_nodes(p:[[f64;3];4]) -> Self { todo!() } }
```

### 3.2 24自由度 → 全体（4×6）への対応（★明示）

要素ローカルの 24 自由度の並びを固定する（決定性・組立のため）:

```
[node0: Ux Uy Uz Rx Ry Rz]  [node1: ...]  [node2: ...]  [node3: ...]
   0  1  2  3  4  5            6..11          12..17        18..23
```

- 各ローカル6自由度は、`ShellFrame` の回転で**全体6自由度**へ変換してから組み立てる
  （梁の `to_global`（P1 §4.3）と同型の 6×6 ブロック対角回転を4ブロック）。
- `global_dofs`（`ElementBehavior`）は `node_i.index()*6 + d` を 4節点ぶん返す。

### 3.3 ドリリング特異点の注意（★平床で必発）

- すべての要素が**同一平面に並ぶ節点**（＝平らな床。本システムの最頻ケース）では、
  ドリリング自由度に剛性が無いと**全体剛性が特異**（ゼロピボット）になり分解に失敗する。
- これを防ぐのが §5 のドリリング安定化。**平床テスト（§9.5/§9.3）で必ず確認**する。

**DoD（T1）:** `ShellFrame::from_nodes` が直交正規な3軸を返す。24自由度の並びが上記に一致。

## 4. T2: MITC4 仮定横せん断ひずみ（tying points）

設計書 §6.5。**この要素の核心であり、プロジェクト最大の実装リスク**。

**MITC4 の仮定横せん断ひずみ（Dvorkin–Bathe 1984）の具体式を以下に明示する。** 出典は同論文／
Bathe *Finite Element Procedures*。検算は §9.2 パッチテスト（機械精度）で担保する。

### 4.1 構成（具体式）

- 面内（膜・曲げ）の B 行列は通常のアイソパラメトリック双線形補間から作る（標準。記憶で可）。
- **横せん断は「共変成分を辺中点 tying point で評価し補間」**（双線形そのままだとロッキング）:

```
自然座標 (ξ,η)∈[−1,1]²。tying point（辺中点）:
   A=(0,+1),  C=(0,−1)   … ξ方向せん断 e_ξζ を採る点（η=±1 の辺中央）
   B=(−1,0),  D=(+1,0)   … η方向せん断 e_ηζ を採る点（ξ=±1 の辺中央）

共変横せん断の補間（MITC4 の肝）:
   e_ξζ(ξ,η) = ½(1+η)·e_ξζ|_A + ½(1−η)·e_ξζ|_C     （η だけの1次補間）
   e_ηζ(ξ,η) = ½(1+ξ)·e_ηζ|_D + ½(1−ξ)·e_ηζ|_B     （ξ だけの1次補間）

各 tying point の共変ひずみは、標準双線形場（w, θx, θy）から
   e_ξζ = (∂x/∂ξ)·γ_xz-相当 + …  = J 行・（横たわみ勾配＋回転）で計算
   （= 双線形補間で得た γ をその点の共変成分に射影したもの）

物理せん断へ変換:  [γ_xz; γ_yz] = J⁻ᵀ · [e_ξζ; e_ηζ]   （J=ヤコビアン）
```

> ポイントは「**ξζ せん断は η のみで、ηζ せん断は ξ のみで補間する**」こと（交差させない）。
> これで薄板極限 (t→0) のロッキングが消える。`shear_strain_b` はこの補間後の物理 γ を
> 節点自由度で微分した B 行列を返す。
> **正直な但し書き:** 上の**補間規則（tying点・η/ξ補間）は確定**だが、共変ひずみの評価と
> J⁻ᵀ 変換は実装者が組む部分が残る（本要素は本書中で唯一、完全コピペには至らない）。
> **真の合否ゲートは §9.2 パッチテスト（機械精度）**＝ここを通れば実装は正しい。

```rust
// sc-element/src/shell.rs
impl ShellElement {
    /// MITC4 の横せん断ひずみ–変位行列（上の tying-point 補間を実装）。
    fn shear_strain_b(&self, xi: f64, eta: f64) -> ShearB {
        // 1. A,B,C,D の各点で双線形場から共変 e_ξζ,e_ηζ を評価
        // 2. e_ξζ は η で、e_ηζ は ξ で1次補間（上式）
        // 3. J⁻ᵀ で物理 γ_xz,γ_yz へ → 節点 DOF についての B 行列に整理
        todo!("補間規則(A/B/C/D・η/ξ補間)は明示済み。共変ひずみ評価とヤコビアン変換は実装する。\
               真のゲートは §9.2 パッチテスト（機械精度）")
    }
    /// 膜・曲げのひずみ–変位行列（通常の双線形。標準）。
    fn membrane_bending_b(&self, xi: f64, eta: f64) -> MemBendB { todo!() }
}
```

**DoD（T2）:** §9.2 のパッチテスト（定曲げ・定膜）に機械精度で合格。薄板（t/a が小）でロッキング
しない（単純支持板で過小たわみにならない＝§9.3 が収束する）。

## 5. T3: 膜＋ドリリング自由度の安定化

設計書 §6.5（面内は適合要素＋ドリリング安定化、Allman/Cook 流）。

> **⚠ 要検証:** ドリリング自由度付き膜要素の定式化は **Cook**（*Concepts and Applications of FEA*）
> または Allman の原論文を一次資料とし、`// 要検証`。

### 5.1 構成（概略）

- 面内変位を、節点並進＋節点ドリリング回転で補間（Allman 型）。これでドリリング自由度に
  物理的剛性が与えられ、面内回転が梁の Rz と整合する。
- 加えて、純粋なドリリングモードのゼロエネルギーを抑える**安定化項**を足す（Cook 流）。

### 5.2 安定化係数（★結果が不感であること）

```rust
// 既定の安定化係数（要検証・経験値）。代表値 γ_d ≈ 1.0e-3 〜 1.0e-2 程度。
// 重要: 解（変位・応力）が係数の小変化に対して実質不感であることをテストで確認する。
pub const DEFAULT_DRILLING_FACTOR: f64 = 1.0e-3;
```

**DoD（T3）:** 平床（全要素同一平面）の組立 K が**正定値**（分解が成功＝特異でない）。
安定化係数を 10 倍/1/10 にしても主要なたわみ・応力が 1% 以内で変わらない（不感性）。

## 7. T5: 剛床階の面内無効化（R13）＋ factory 登録

設計書 §6.5 / R13。P1 §9.1 で予告した二重計上回避をここで実装する。

> **⚠ 無効化するのは「膜＋ドリリング」ブロックだけ。曲げは残す（★梁のパネル方式A と混同しない）。**
> 剛床ダイアフラムを適用した階では、シェルの面内自由度 (Ux, Uy, **および ドリリング Rz**) は
> 既にマスター節点へ従属（拘束）している。ここに膜・ドリリング剛性を残すと**剛床拘束と二重計上**。
> 一方、**曲げ（Uz, Rx, Ry）は剛床に拘束されない**＝床の面外（鉛直）剛性なので**必ず残す**。
> → 「**要素ごと飛ばす**」のではなく「**膜＋ドリリングのブロックをゼロにし、曲げブロックは組む**」。
> （P1 のパネルゾーン方式A は*要素ごとスキップ*だった。こちらは*部分ブロックのみ無効*。混同禁止。）

```rust
impl ShellElement {
    /// membrane_active=false のとき、24×24 のうち
    /// 膜＋ドリリング（各節点の Ux,Uy,Rz）に関わる行・列をゼロにし、曲げ（Uz,Rx,Ry）は残す。
    pub fn apply_rigid_floor_membrane_off(&self, k: &mut crate::behavior::LocalMat) { todo!() }
}
```

`membrane_active` の決定（前処理）:
- 要素の4節点が属する階に**剛床ダイアフラムが適用**されていれば `false`、柔床なら `true`。
- この判定はモデルの階・拘束情報（sc-core）から行う（factory or アセンブラ前処理）。

factory（P1 §3.3）に Shell を追加:

```rust
// sc-element/src/factory.rs（match に追加）
ElementKind::Shell => (Box::new(crate::shell::ShellElement::new(data, model)), ElemState::default()),
```

**DoD（T5）:** 剛床階のシェルで、水平加力時に面内剛性が効かず（剛床が負担）、鉛直加力で
曲げ剛性は効く。柔床（剛床なし）では面内も効く。両者で二重計上が起きていないこと
（剛床あり/なしの水平剛性が、剛床拘束の有無で説明できる差になる）。

## 8. T6: 応力復元（8 応力合力）

設計書 §6.5。シェルの内力は**単位幅あたりの応力合力**で表す。

```rust
// sc-element/src/shell.rs
/// シェルの応力合力（単位幅あたり）。評価点（積分点 or 指定点）ごと。
pub struct ShellResultants {
    // 膜力 [N/mm]
    pub nx: f64, pub ny: f64, pub nxy: f64,
    // 曲げ・ねじりモーメント [N·mm/mm = N]
    pub mx: f64, pub my: f64, pub mxy: f64,
    // 横せん断 [N/mm]
    pub qx: f64, pub qy: f64,
}

impl ShellElement {
    /// 要素節点変位（全体系 24）から、各評価点の応力合力を復元する。
    /// 手順: u_global → ローカルへ回す → B行列でひずみ → 構成則 D で応力合力。
```

## 9. T7: テスト・DoD（フェーズ全体）

### 9.1 検証哲学（再掲・必読）

- **パッチテストだけが厳密**（機械精度）。
- **板たわみ・モーメントは収束**（粗メッシュ 1〜2% は正常）。メッシュ密度＋許容帯＋単調収束で判定。
- 決定性は単一スレッドでビット一致（P0/P1 踏襲）。

### 9.2 パッチテスト（→ T2/T3）★唯一の厳密チェック

- **歪んだ四辺形メッシュ**（内部節点をわざとずらした不整形メッシュ、最低 2×2 要素＋内部1節点）。
- 境界節点に**線形変位場**を与える。
  - (a) 膜状態：面内一様ひずみに対応する線形変位 → 内部が**定数膜応力**を機械精度で再現。
  - (b) 定曲げ状態：定曲率に対応する変位 → 内部が**定数モーメント**を機械精度で再現。
- 合格条件：内部点の応力が解析値と **相対 1e-10 程度（機械精度）** で一致。

### 9.3 単純支持／クランプ 正方形板（→ T4/T6）★収束テスト

> **係数は本文に明示（ν=0.3）。他の ν のみ出典参照:** 正方形板・等分布荷重 `q` の中央たわみは
> `w_c = α·q·a⁴/D`、
> 板曲げ剛性 `D = E·t³/[12(1−ν²)]`。係数 α は **ν=0.3 の級数解**:
> 単純支持 `α ≈ 0.00406`、クランプ（四辺固定）`α ≈ 0.00126`。
> 出典: **Timoshenko & Woinowsky-Krieger, *Theory of Plates and Shells***。`// 要検証`、ν依存に注意。

**確定値の例**（`E=200000, ν=0.3, t=10, a=1000, q=0.01 N/mm²`）:

| 量 | 式 | 値 |
|---|---|---|
| `D` | `E·t³/[12(1−ν²)]` | `200000·1000/(12·0.91) = 1.8315e7 N·mm` |
| `q·a⁴/D` | — | `0.01·1e12/1.8315e7 = 545.99` |
| 単純支持 中央たわみ | `0.00406·q·a⁴/D` | `≈ 2.217 mm`（参照解） |
| クランプ 中央たわみ | `0.00126·q·a⁴/D` | `≈ 0.688 mm`（参照解） |

判定:
- **16×16 メッシュで参照解の ±2% 以内**（単純支持・クランプ各）。
- **細分化収束**：4×4 → 8×8 → 16×16 で誤差が**単調減少**して参照解へ近づく。

### 9.4 片持ち板（→ サニティ）

- 一辺固定・先端等分布の片持ち板。**厳密な板閉形式は無い**（アンチクラスティック曲率）。
  幅広梁的な近似（`δ ≈ wL⁴/(8D')`, `D'` は実効剛性）と**桁・傾向が合う**ことを確認する程度の
  サニティチェック。精密な板基準値には使わない（§9.3 を主基準とする）。

### 9.5 剛床面内無効化＋平床特異性（→ T3/T5）

- 剛床階のシェルで面内剛性が無効・曲げが有効（§7 の DoD）。

### 9.6 決定性（→ 全体）

## 10. 数式・理論補足（ジュニア向け）

### 10.1 せん断ロッキングと MITC（→ §4）

薄板を双線形要素で素直に離散化すると、曲げ変形に伴う「偽のせん断ひずみ」が消えず、板が
**実際よりはるかに硬く**なる（過小たわみ＝せん断ロッキング）。MITC4 は横せん断ひずみを
要素辺中点（tying point）でサンプルして補間し直すことで、この偽せん断を除く。結果、薄板でも
ロッキングせず、厚板（せん断変形が実在）も扱える。

### 10.2 ドリリング自由度（→ §3, §5）

平らなシェルは法線まわり回転（ドリリング）に物理剛性を持たない。だが梁（6自由度）と節点を
共有して組み立てると、この自由度に剛性ゼロの行・列ができ、**全体行列が特異**になる
（とくに平床で全要素が同一平面のとき）。Allman/Cook 流の定式化はドリリングに物理的な面内回転を
与え、さらに微小な安定化項でゼロエネルギーを抑える。安定化は数値的な逃がしなので、**解が
安定化係数にほとんど依存しない**ことを確認する（§5.2）。

### 10.3 パッチテスト（→ §9.2）

「要素を歪んだ小片（patch）に分割し、定ひずみ（定応力）状態を厳密に再現できるか」を見る
収束性の必要条件。境界に定ひずみ相当の線形変位を与え、内部が**定数応力を機械精度で**返せば合格。
MITC4 は定曲げ・定膜の両方でこれを満たす必要がある。これだけが*厳密*チェックで、
たわみ問題は収束（§10.4）で見る。

### 10.4 なぜ「収束」で見るのか（→ §0.2, §9.1, §9.3）

板曲げの厳密解（級数解）は無限自由度。有限要素は有限自由度なので**必ず離散化誤差**が残り、
メッシュを細かくするほど厳密解へ近づく（収束）。よって正しい MITC4 でも粗メッシュでは数%ずれる。
判定は「ある密度で許容帯に入る」＋「細分化で誤差が単調に縮む」で行う。厳密一致を要求すると
正しい実装を不合格にしてしまう。

### 10.5 板曲げ剛性 D（→ §9.3）

梁の `EI` に相当する板の曲げ剛性が `D = E·t³/[12(1−ν²)]`。`(1−ν²)` は板の二軸拘束効果
（梁と違い横方向に縮めない）による。中央たわみ係数 α はこの D と荷重・寸法から
`w_c = α·q·a⁴/D` で定まり、α は境界条件（単純支持/固定）と ν に依存する（級数解）。
