## 2. T0: V&V フレームワーク

設計書 §17。テスト階層（既存の各フェーズで作ったものを束ねる）:

| レベル | 内容 | ツール |
|---|---|---|
| 単体 | 要素剛性・履歴則・断面算定式 | `cargo test`, `approx` |
| 性質 | 剛性対称性・エネルギー保存・パッチテスト | `proptest` |
| 回帰 | 履歴ループ・スケルトン形状 | `insta`（スナップショット） |
| 数値照合 | 理論解（梁・板・SDOF/MDOF） | 専用ベンチ集 |
| ベンチマーク照合 | 既往実験・商用ソフト（壁・プッシュオーバー） | 検証レポート |
| 性能 | 速度回帰 | `criterion` + CI 閾値 |
| 決定性 | 同一入力ビット一致 | 専用テスト |

### 2.1 V&V レポートの構造（成果物）

各「要素・設計式」につき1エントリ:

```
- 対象（例: ティモシェンコ梁 / パネルゾーン / Ai分布 / プッシュオーバー機構）
- 参照解の出典（理論式 / 実験 / 商用ソフト / 規準例題 / 添付資料）
- 入力モデル（再現可能な定義。テストに対応づけ）
- 許容差（厳密=1e-9 / 収束=±% / 規準例題照合）
- 結果（合否・差分・グラフ）
```

> **検証の性格を明記（各フェーズの規律を集約）:** 厳密一致（IIE 梁・CMQ・σ=M/Z・剛性率）と、
> 収束（MITC4 板・固有値・時刻歴）と、規準例題照合（Ai・Ds・許容応力度）を**区別して**記載する。

> **★一次基準は「手計算／理論解」（実装者が外部参照なしで自走可）。** 各フェーズの DoD は
> 手計算・理論解・告示式の自己整合・添付資料数値例で**合否判定できるよう構成済み**。
> **実測／商用ソフト照合は補助**（入手できれば追加）で、無くてもビルド・単体テスト・一次 V&V は通る。
> **唯一の例外＝壁（壁谷澤）の妥当性確認**は、モデルの性質上 実験照合が本質的に必須（Category B）。
> ここだけは技術リードが実験データを用意する（R4/R23）。それ以外は実装者が自走できる。

**DoD（T0）:** V&V レポートの雛形と、各フェーズのテストをレポート項目へ対応づける索引が整う。

## 4. T2: 性能ベンチ・CI・決定性

設計書 §1.3 / §17 / R28。

- **性能ベンチ（criterion）**：線形静的・固有値・プッシュオーバー1ステップ・時刻歴1ステップを
  **基準モデル・基準機**（リード提供）で計測。CI で**規定%以上の劣化＝ビルド失敗**。
- **決定性テスト**：全解析種別で「CPU・単一スレッドでビット一致」を検証（R28）。並列／Parquet 書込／
  GPU は値一致で検証（ビット一致保証外）。
- **依存方向チェック**：`xtask check-deps`（P0 §2.5）を CI に入れる。循環依存禁止。

```
# CI（例）
cargo test --workspace                 # 全テスト（GPU/ML/GUI feature 無し）
cargo run -p xtask -- check-deps       # 依存方向
cargo bench --bench perf               # 性能（閾値超過で失敗）
# 決定性: 専用テストで 100 回ビット一致
```

**DoD（T2）:** CI が緑。性能基準を満たし、劣化検出が機能（わざと遅くすると失敗する）。決定性テスト合格。

## 8. 補足（ジュニア向け）

### 8.1 Verification と Validation の違い（→ §2）
- **Verification（検証）**：「式を正しく解いているか」。理論解・手計算・規準例題との一致で確認。
- **Validation（妥当性確認）**：「正しい現象を表しているか」。実験・実測との一致で確認（壁・履歴など）。
両方を V&V レポートに分けて記載する。

### 8.2 なぜ決定性を CI で守るか（→ §4）
構造計算は再現性が命（同じ入力＝同じ出力）。並列化や最適化で結果がぶれると検証が無意味になる。
CPU 単一スレッドでのビット一致を保証範囲とし（R28）、並列/GPU は値一致で別途検証する。

### 8.3 責任範囲の明記（→ §5.1）
本ソフトは技術者の判断を支援する道具であり、**算定結果の最終確認・責任は技術者**にある旨を
出力・ドキュメントに明記する（設計書 §20-risk7）。法的トラブルと誤用を防ぐための必須事項。
