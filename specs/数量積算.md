## 0. この機能について

### 0.1 目的

建物モデル（`squid_n_core::model::Model`）だけを入力として、部位別
（柱・大梁・小梁・基礎梁・床・片持ち床・壁・雑壁・ブレース）に

- コンクリート体積 [m³]
- 型枠面積 [m²]
- 鉄筋重量 [t]（用途別: 主筋／フープ／スターラップ／壁筋／床筋…）
- 鉄骨重量 [t]（断面名別の長さ・重量）
- 鉄筋継手個所数（圧接個所数）

を**概算**集計する。解析結果には依存しない（モデル編集のたびに即時再集計できる）。

### 0.2 完了像（ゴール）

1. 算定式の単体テストが手計算値と一致する（`quantity::member::tests`、許容差 1e-6）。
2. モデル走査（分類・スラブ控除・開口控除・集計）がテストモデルで検証済み
   （`quantity::tests`）。
3. GUI（設計タブ「数量積算」）・CSV・MCP から同一の集計結果を参照できる。
4. モデルに情報が無く算定できない項目は結果の注記（`notes`）に必ず現れる。

### 0.3 スコープ境界（含む／含まない）

| 項目 | 本機能で | 備考 |
|---|---|---|
| RC/S/SRC/CFT 線材（柱・梁・ブレース）の数量 | **含む** | §3 |
| 床（スラブパネル）・壁（開口控除）・フレーム外雑壁 | **含む** | §3.5〜3.7 |
| 鉄筋継手個所数（圧接） | **含む** | 梁 0.5+5m毎0.5／柱 1+7m毎1（1本あたり） |
| ハンチ付き梁の算定式 | **式のみ含む** | モデルにハンチ寸法が無いため走査は常にハンチなし |
| 基礎フーチング | **含まない → 将来** | モデルに定義が無い |
| 2断面/3断面配筋・カットオフ筋(+15d) | **含まない → 将来** | 部材断面が全長 1 断面のため |
| 腹筋・幅止筋 | **含まない → 将来** | 段数入力が無い |
| デッキスラブ控除・床ごとの床厚 | **含まない → 将来** | 床厚は `Model::slab_thickness`（建物一律） |
| 施工数量（フック・余長・継手長さ・開口補強筋・接合部材） | **含まない（恒久）** | 概算数量の割り切り |

## 1. 構成とデータフロー

```
Model ──> squid_n_design_jp::quantity::compute_quantity_takeoff(&Model, &QuantityCfg)
              │  （純関数: 走査・分類・単位換算。解析結果に非依存）
              ▼
        QuantityTakeoff { items: Vec<MemberQuantity>, notes: Vec<String> }
              │
   ┌──────────┼──────────────────────┐
   ▼          ▼                      ▼
 GUI       CSV 文字列              MCP JSON
 (設計タブ  summary::              query::quantity_takeoff_json
  quantity_view)  build_quantity_csv   （ツール quantity_takeoff）
```

- **エンジン**: `crates/squid-n-design-jp/src/quantity/`
  - `mod.rs` — 公開型・モデル走査・集計 API
  - `member.rs` — 部位別算定式の**純関数**（mm 単位、ユニットテスト対象）
  - `rebar.rs` — JIS G 3112 呼び径別単位質量 [kg/m] と長さ→重量換算
- **GUI**: `crates/squid-n-app/src/quantity_view.rs`（`DesignView::Quantities`）
- **CSV**: `crates/squid-n-app/src/summary.rs::build_quantity_csv`
  （数量ビューのエクスポートとレポート CSV `[数量積算 *]` セクションで共用）
- **MCP**: `crates/squid-n-mcp/src/query.rs::quantity_takeoff_json`（feature 非依存）＋
  `server.rs` の `#[tool] quantity_takeoff`

単位系: 入力（モデル）は内部単位 N-mm-s、`member.rs` の式も mm 系。
`MemberQuantity` へ格納する時点で m³ / m² / t / m に換算する。

## 2. 公開型（シグネチャ）

```rust
pub struct QuantityCfg {
    pub anchorage_dia_factor: f64, // 定着長さ係数（L2・S = 係数×径。既定 35）
    pub assumed_wall_bar_dia: f64, // 壁筋の仮定径 [mm]（既定 D10）
    pub joist_main_ratio: f64,     // 小梁主筋の鉄筋比（既定 0.008）
    pub joist_stirrup_ratio: f64,  // 小梁スターラップ（既定 0.001）
    pub slab_rebar_ratio: f64,     // 床筋（既定 0.01）
}

pub enum MemberCategory { FoundationGirder, Column, Girder, Joist,
                          Slab, CantileverSlab, Wall, MiscWall, Brace }
pub enum StructureKind  { Rc, S, Src, Cft }
pub enum RebarUsage     { MainBar, Hoop, Stirrup, WallHorizontal, WallVertical,
                          SlabBar, JoistMain, JoistStirrup }

pub struct RebarItem  { usage, dia: Option<f64>, total_length_m, weight_t }
pub struct SteelItem  { section_name, length_m, weight_t }
pub struct MemberQuantity {
    elem: Option<ElemId>, slab: Option<SlabId>, label, story,
    category, structure,
    concrete_m3, formwork_m2, rebar: Vec<RebarItem>,
    steel: Option<SteelItem>, rebar_joints: f64,
}
pub struct QuantityTakeoff { items: Vec<MemberQuantity>, notes: Vec<String> }
// 集計: totals() / totals_by_category() / totals_by_story()
//       / steel_by_section() / rebar_by_dia()

pub fn compute_quantity_takeoff(model: &Model, cfg: &QuantityCfg) -> QuantityTakeoff;
```

## 3. 算定規則（要点）

算定式の全文は [計算根拠 11.1](../docs/calc_basis/11_数量積算/01_数量積算の算定式.md) を
正とする。実装上の要点のみ記す。

### 3.1 部位分類（モデルに用途区分が無いため幾何判定）

1. `ElementKind::Beam` の 2 節点線材で、両端の水平距離 < 1mm → **柱**
   （`is_vertical_pair`。自重算定と同規則）
2. 水平梁のうち、両端節点が最下層レベル（全線材節点の最小標高 ±10mm）かつ
   柱が取り付く → **基礎梁**
3. 端部節点のいずれかに柱が取り付く水平梁 → **大梁**、それ以外 → **小梁**
4. `Wall`/`Shell`（3 節点以上）→ **壁**、`Brace` → **ブレース**、
   `Slab.kind` により **床**／**片持ち床**、`MiscWall`（壁厚あり）→ **雑壁**

構造種別は `SectionShape` バリアント優先（`SrcRect`→SRC、`Cft*`→CFT、`Steel*`→S、
`Rc*`→RC）、形状未設定時は材料名接頭辞 `is_steel_name`（一次設計の検定器選択と同規則）。

### 3.2 事前計算（走査を O(モデル) に保つ）

- `column_nodes: HashSet<節点>` — 柱の取り付く節点（大梁/小梁分類用）
- `beams_at_node: HashMap<節点, Vec<(elem, 方向xy)>>` — 主筋端部の外端/内端判定用
  （当該節点から外向き ±45° に連続する別の水平梁があれば内端）
- `slab_edges: HashMap<(節点対), 隣接数>` — 梁型枠のスラブ厚控除
  （両側=2/片側=1/なし=0）。境界 3 節点未満のスラブは辺登録しない
- `min_z` — 基礎梁レベル

### 3.3 スラブ厚控除（大梁型枠）

側面のせいは隣接数により `(D, D)` / `(D, D−t)` / `(D−t, D−t)`。
`t = model.slab_thickness.clamp(0, D)`、スラブが 1 枚も無いモデルでは控除しない。
基礎梁は「側面 1 面＋底面」で、隣接スラブ（耐圧版）があれば側面から控除。

### 3.4 鉄筋の重量換算

長さ [mm] × `rebar_unit_mass_kg_per_m(dia)`。表に無い径は公称断面積×7.85e-3 で計算。
鉄筋比による概算（小梁・床）は `dia: None`・長さ 0 とし、
体積×比率×7.85 t/m³ を重量に直接計上する。

### 3.5 壁

- 4 節点壁は周辺柱梁の半寸法を控除した内法 (L, H)（`wall_clear_dims`。
  控除相手が無い辺は芯々のまま）。それ以外は多角形面積と高さから等価 L を取る。
- 開口は `WallAttr::total_opening_area()` を控除。
- 壁筋は `RcWall::ps` の等価換算: 横筋体積 = ps·t·H·(L+2S)、縦筋 = ps·t·L·(H+2S)。

### 3.6 空集計の正規化

`Vec<f64>` の空イテレータ `sum()` は `-0.0` を返し「-0.0000」表示になるため、
`rebar_weight_t()` は `+ 0.0` で正の 0 に正規化する（回帰注意点）。

## 4. UI・接続仕様

- 設計タブに `DesignView::Quantities`（「数量積算」）。集計切替は
  `QuantityViewState::grouping`（部位別／階別／明細／鉄骨種類別／鉄筋径別）。
- 明細の ID クリックで `nav.focus_member` を設定（3D ハイライト連動）。
- 表示は毎フレーム `compute_quantity_takeoff` を呼ぶ（走査は O(モデル)。
  undo/stale 管理に依存しないため編集へ即時追従する）。
- CSV は `build_quantity_csv(&Model)`（GUI 非依存）。レポート CSV には
  解析結果の有無に関わらず常時出力する。
- MCP ツール `quantity_takeoff { group_by?: "category"|"story"|"steel"|"rebar"|"detail" }`
  → `{ rows, totals, notes }` の JSON。
