## 2. T0: 時刻積分器（Newmark-β／HHT-α）

設計書 §10.5。運動方程式 `M ü + C u̇ + K u = p(t)`（地震は `p(t) = −M r ẍg(t)`）を数値積分する。

```rust
// sc-solver/src/timehistory.rs
pub struct NewmarkCfg { pub beta: f64, pub gamma: f64, pub dt: f64 }
impl NewmarkCfg {
    pub fn average_accel() -> Self { Self { beta: 0.25, gamma: 0.5, dt: 0.0 } } // 無条件安定
    pub fn linear_accel()  -> Self { Self { beta: 1.0/6.0, gamma: 0.5, dt: 0.0 } } // 条件付安定
}
pub struct HhtCfg { pub alpha: f64 /* -1/3..0 */, pub dt: f64 } // 数値減衰つき

/// 地動（基盤一様加振）。水平1〜2方向（R8）。
pub struct GroundMotion { pub dt: f64, pub accel_x: Vec<f64>, pub accel_y: Option<Vec<f64>> }
```

**Newmark-β の更新式（線形・増分形。標準形をそのまま使える。Chopra *Dynamics of Structures*）。**
`u, v(=u̇), a(=ü)` を既知の現ステップ、`Δt, β, γ` を定数として、各ステップは次の閉形式で進む:

```
定数（dt 固定なら1回だけ計算）:
  c1 = 1/(β·dt²),  c2 = γ/(β·dt)
  c3 = 1/(β·dt),   c4 = 1/(2β) − 1
  c5 = γ/β − 1,    c6 = dt·(γ/(2β) − 1)

有効剛性:    K^ = K + c2·C + c1·M       （線形なら1回分解して再利用）
有効荷重:    p^_{n+1} = p_{n+1}
             + M·(c1·u_n + c3·v_n + c4·a_n)
             + C·(c2·u_n + c5·v_n + c6·a_n)
変位を解く:  K^ · u_{n+1} = p^_{n+1}      （後退代入。地震は p = −M·r·ẍg）
加速度更新:  a_{n+1} = c1·(u_{n+1} − u_n) − c3·v_n − c4·a_n
速度更新:    v_{n+1} = v_n + dt·((1−γ)·a_n + γ·a_{n+1})
```

- 平均加速度法 `β=1/4, γ=1/2`（無条件安定）。線形加速度法 `β=1/6, γ=1/2`。
- HHT-α は上記に α（`-1/3≤α≤0`）の重み付けを足して高次モードに数値減衰を入れる
  （`(1+α)` を慣性・減衰・剛性・荷重項に配分。Hughes *The Finite Element Method*）。
- 非線形（P5 §6）では `K`→接線 `K_t`、`p^` の釣合を Newton 反復で満たす（§4）。

> 上記係数は標準形で確定。実装後に **§8.1 の SDOF 解析解**（`T=0.198692 s` 等）で必ず照合する。

**DoD（T0）:** 線形 SDOF が解析解に収束（Δt 細分で誤差減少）。平均加速度が無条件安定（大 Δt で発散しない）。

## 3. T1: 減衰モデル

設計書 §10.5 / R9。

- **質量比例** `C = a0·M`、**剛性比例** `C = a1·K`、**Rayleigh** `C = a0·M + a1·K`。
- **既定：1次モード剛性比例**（日本の建築慣行）。規定減衰 **RC h=0.03／S h=0.02**（外部設定）。
- 剛性比例は **初期 K／接線 K を選択**（非線形では効果が変わる）。
- 係数 `a0, a1` は固有値（P2 §4）の振動数と目標減衰比 `h` から決める（§9.3）。

```rust
// sc-solver/src/damping.rs
pub enum Damping {
    MassProportional { h: f64, omega: f64 },
    StiffnessProportional { h: f64, omega: f64, basis: StiffnessKind },
    Rayleigh { h1: f64, w1: f64, h2: f64, w2: f64 },
}
/// 剛性比例減衰がどの剛性を基準にするか。
pub enum StiffnessKind { Initial, Tangent }

// 以下 SparseMat = faer::sparse::SparseColMat<usize, f64>（P0/P2 と同じ縮約済み疎行列）
impl Damping { pub fn assemble_c(&self, m: &SparseMat, k: &SparseMat) -> SparseMat { todo!() } }
```

**DoD（T1）:** 各タイプで、対象振動数の応答減衰比が目標 `h` と整合（自由振動の対数減衰率で確認）。

## 4. T2: 非線形時刻歴（各ステップ内部反復）

設計書 §10.5。各時刻ステップ内で、非線形の釣合を Newton 反復で解く。**P5 §6 の commit/rollback
トランザクションをそのまま再利用**。

```
各時刻ステップ:
  snapshot ← 確定状態（P5 §6）
  予測子（Newmark）→ Newton 反復（接線 K_t＋有効項で残差を消す。各反復 update_state(trial)）
  収束 → commit_all／結果1ステップ記録／結果I/O へストリーミング（§6）
  不収束 → restore → dt 細分 or 反復継続（方針を明示）
```

- 接線剛性に**幾何剛性 Kg（P5 §9）**を含められる（ON/OFF）。
- 出力：最大応答・層間変形角・累積塑性率（設計書 §10.5）。

## 5. T3: チェックポイント／再開・スクラッチ（sc-io）

設計書 §12.4。長時間の非線形／時刻歴の**再開・巻き戻し**のため、ジョブ作業領域を定義。

```
<workdir>/job_<id>/
├── checkpoint/   # 解析状態スナップショット（再開用）
├── scratch/      # 行列分解・組立の一時ファイル（再生成可・削除可）
└── log/          # tracing ログ・収束履歴
```

**チェックポイント内容（設計書 §12.4）:** 現在の変位・速度・加速度、全要素の確定履歴状態
（commit 状態＝P5 §6 の `StateSnapshot`）、荷重係数／時刻、ステップ番号、（あれば）乱数状態、スキーマ版。

```rust
// sc-io/src/checkpoint.rs
#[derive(serde::Serialize, serde::Deserialize)]
pub struct Checkpoint {
    pub schema_version: u32,
    pub model_hash: String,       // モデルとの整合性ハッシュ（不正再開の防止）
    pub step: u64, pub time: f64,
    pub disp: Vec<f64>, pub vel: Vec<f64>, pub accel: Vec<f64>,
    pub state: StateBlob,         // 全要素・全材料の確定状態（P5 §6 を直列化）
}
/// P5 §6 の StateSnapshot を直列化したバイト列（全 ElemState・全材料 committed）。
#[derive(serde::Serialize, serde::Deserialize)]
pub struct StateBlob { pub bytes: Vec<u8> }
pub fn save_checkpoint(dir: &std::path::Path, cp: &Checkpoint) -> std::io::Result<()> { todo!() } // 高速バイナリ
pub fn load_checkpoint(dir: &std::path::Path) -> std::io::Result<Checkpoint> { todo!() }
```

- 形式は高速バイナリ（`bincode`／MessagePack）、必要に応じ memmap。
- **再開（restart）:** 直近チェックポイントから継続。プッシュオーバーの任意ステップ分岐再解析にも対応。
- **スクラッチ:** 疎分解の中間・巨大一時行列はメモリ上限超過時のみディスク退避。再生成可能キャッシュ扱い、
  ジョブ終了時に自動掃除。
- **整合性ハッシュ**でモデル変更後の不正な再開を防止（`model_hash` 不一致なら拒否）。

**DoD（T3）:** 解析を途中強制終了→チェックポイントから再開し、無停止実行と同一結果に収束
（ビット一致）。`model_hash` 改竄で再開拒否。

## 9. 数式・理論補足（ジュニア向け）

### 9.1 運動方程式と地震入力（→ §2）
`M ü + C u̇ + K u = p(t)`。基盤一様加振では地動加速度 `ẍg(t)` が `p(t) = −M r ẍg(t)`（`r`＝影響ベクトル、
加振方向の並進を1とする）として効く。相対変位 `u` を解く。

### 9.2 Newmark-β（→ §2）
次ステップの変位・速度を `β, γ` でパラメータ化した更新式で進める陰解法。`β=1/4, γ=1/2`（平均加速度）は
**無条件安定**（大きい Δt でも発散しない）、`β=1/6`（線形加速度）は条件付き安定（精度は高い）。HHT-α は
高次モードに数値減衰を入れて安定化する。

### 9.3 Rayleigh 減衰（→ §3）
`C = a0 M + a1 K`。モード `i` の減衰比は `hi = a0/(2ωi) + a1·ωi/2`。2つの振動数で目標 `h` を与えると
`a0, a1` が決まる。剛性比例（`a0=0`）は高次モードを強く減衰、質量比例（`a1=0`）は低次を強く減衰。
日本の慣行は1次モード剛性比例が既定（高次の過大応答を抑える）。

### 9.4 チェックポイントと経路依存（→ §5）
非線形は経路依存なので、再開には「その時刻の全状態（変位・速度・加速度＋全要素の確定履歴）」が要る。
これは P5 §6 の `StateSnapshot` を直列化したもの。モデルが変わると状態が無意味になるため、`model_hash`
で対応を縛る。
